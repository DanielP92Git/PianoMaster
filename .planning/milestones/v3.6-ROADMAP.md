# Milestone v3.6: Game Screen UI Unification

**Status:** ✅ SHIPPED 2026-06-14
**Phases:** 36
**Total Plans:** 11 (across 7 execution waves)

## Overview

Bring the other game screens up to the polished UI standard set by **NotesRecognitionGame**
by extracting its inline HUD/shell (progress bar, score pill, lives/hearts, combo + on-fire,
timer, speed-bonus, tier-up, nav, feedback) into reusable shared components under
`src/components/games/shared/hud/`, and adopting them across the other games where they fit —
component-based reuse, not forced uniformity. Games with fundamentally different mechanics
(staff-based sight-reading, tile/card grids, fixed-length ear-training) adopt only the suitable
subset. ArcadeRhythmGame's existing inline lives/combo/on-fire were de-duplicated onto the
shared components. No game-mechanics changes; HUD presentation only.
