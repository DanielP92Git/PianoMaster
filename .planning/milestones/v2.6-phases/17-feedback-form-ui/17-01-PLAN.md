Error detection pattern (from RESEARCH.md Pitfall 1):
      ```javascript
      try {
        const { data, error } = await supabase.functions.invoke('send-feedback', { body: { ... } });
        if (error) throw error;
        if (data?.success) {
          // success path: setStatus('success'), startCooldown(), reset form fields
        } else {
          if (data?.error === 'rate_limit') setError({ type: 'rate_limit' });
          else setError({ type: 'server' });
        }
      } catch (err) {
        const status = err?.context?.status;
        if (status === 429) setError({ type: 'rate_limit' });
        else if (status >= 500) setError({ type: 'server' });
        else setError({ type: 'network' });
      }
      ```
    - `startCooldown`: sets cooldownSecondsLeft to COOLDOWN_SECONDS
    - `formatCountdown(seconds)`: returns `${Math.floor(seconds/60)}:${(seconds%60).toString().padStart(2,'0')}`

    **Cooldown effect (per RESEARCH.md Pattern 5):**
    ```javascript
    useEffect(() => {
      if (cooldownSecondsLeft <= 0) {
        if (status === 'success') setStatus('idle');
        return;
      }
      const timer = setTimeout(() => setCooldownSecondsLeft(s => s - 1), 1000);
      return () => clearTimeout(timer);
    }, [cooldownSecondsLeft, status]);
    ```
