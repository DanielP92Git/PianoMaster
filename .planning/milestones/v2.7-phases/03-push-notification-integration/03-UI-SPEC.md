## Interaction Contract

### URL Param Detection Flow (D-11, D-12, D-18)

1. On Dashboard mount, read `?practice_checkin=1` via `useSearchParams` (React Router v7).
2. If param present:
   a. Call `practiceLogService.logPractice(localDate)`.
   b. On success with `inserted: true` → show auto-log success toast. Trigger PracticeLogCard settled state via query cache invalidation.
   c. On success with `inserted: false` (already logged) → show already-logged neutral toast. No state change.
   d. On error → show error toast. No state change.
3. Clean up URL via `window.history.replaceState({}, '', '/')` immediately after detecting the param (before the async call resolves), so a page refresh does not re-trigger auto-log.
4. Detection is idempotent: if param is absent, do nothing.

Source: CONTEXT.md D-11, D-12, D-18; RESEARCH.md architecture patterns.

### PracticeLogCard Animation Trigger

When auto-log succeeds (URL param path):
- Invalidate `['practice-log-today', userId, localDate]` query.
- PracticeLogCard's existing `useEffect` detects `logStatus.logged === true` and transitions to settled state automatically — no new prop or ref needed.
- No separate animation call required; the existing FSM (`idle → settled`) handles the visual update.

### Snooze Scheduling (D-08, D-19)

Inside `sw.js` `notificationclick` handler, when `action === 'not-yet'` and this is the primary (non-snoozed) notification:
- Use `event.waitUntil(new Promise(resolve => setTimeout(resolve, 2 * 60 * 60 * 1000)))` to keep SW alive.
- After timeout fires, call `self.registration.showNotification(title, options)` with snoozed content and `data.isSnoozed: true`.
- When snoozed notification's "Not yet" (`action === 'dismiss-final'`) is tapped: close notification only. No further timeout.

## Accessibility Contract

- All toasts use react-hot-toast's built-in ARIA live region (`role="status"` / `aria-live="polite"`). No additional ARIA work needed.
- URL param processing is silent (no focus changes, no modal interruptions). The PracticeLogCard's existing settled state becomes visible on the next render cycle.
- `PracticeLogCard` existing accessibility attributes (`aria-hidden` on icons, `text-sm font-bold` labels) remain unchanged.
- Reduced-motion: no new animations are added by this phase. The auto-log path reuses the existing PracticeLogCard animation which already respects `useMotionTokens().reduce`.
- RTL: toast messages use react-hot-toast defaults (text direction inherits from `<html dir>` set by i18next). No additional RTL handling needed.
