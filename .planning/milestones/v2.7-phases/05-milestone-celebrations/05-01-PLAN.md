    ```sql
    -- Add milestone tracking column for once-per-streak celebration (D-06)
    ALTER TABLE instrument_practice_streak
      ADD COLUMN IF NOT EXISTS last_milestone_celebrated INTEGER NOT NULL DEFAULT 0;

    COMMENT ON COLUMN instrument_practice_streak.last_milestone_celebrated
      IS 'Highest milestone (5/10/21/30) celebrated in current streak. Resets to 0 on streak break. Prevents re-triggering.';
    ```

    No RLS change needed — existing `"Students can manage own practice streak"` policy covers all columns.

    **Step B: Extend practiceStreakService.js (3 changes).**

    1. **getPracticeStreak**: Change `.select('streak_count, last_practiced_on')` to `.select('streak_count, last_practiced_on, last_milestone_celebrated')`. Add `lastMilestoneCelebrated: data?.last_milestone_celebrated ?? 0` to the return object.

    2. **updatePracticeStreak**: After the existing `.select('streak_count, last_practiced_on')` fetch, also select `last_milestone_celebrated`. When building the upsert payload:
       - When streak resets (`gap > 1` or first-ever log, `newStreakCount = 1`): include `last_milestone_celebrated: 0` in the upsert object (per D-08).
       - When streak increments (`gap === 1`): do NOT include `last_milestone_celebrated` in the upsert (Supabase only updates columns you include).
       - Change the return value from `{ streakCount: newStreakCount }` to `{ streakCount: newStreakCount, lastMilestoneCelebrated: current?.last_milestone_celebrated ?? 0 }`. For the same-day early return (gap === 0), also return `lastMilestoneCelebrated: current.last_milestone_celebrated ?? 0`. For the reset case, return `lastMilestoneCelebrated: 0`.

    3. **Add new method `updateLastMilestoneCelebrated(milestone)`**:
       ```js
       async updateLastMilestoneCelebrated(milestone) {
         const { data: { session } } = await supabase.auth.getSession();
         if (!session) throw new Error('Not authenticated');
         const { error } = await supabase
           .from('instrument_practice_streak')
           .update({
             last_milestone_celebrated: milestone,
             updated_at: new Date().toISOString(),
           })
           .eq('student_id', session.user.id);
         if (error) throw error;
       },
       ```
