## Component Inventory

### New components to build

#### `PracticeHeatmapCard`
- **Location:** `src/components/parent/PracticeHeatmapCard.jsx`
- **Container:** `bg-white/10 backdrop-blur-md border border-white/20 rounded-xl shadow-lg p-6`
- **Composed of:** card header row, summary stats row, heatmap grid, empty state overlay

#### Card header row
- Piano icon (`lucide-react`, `h-5 w-5 text-emerald-400`) + heading text
- Heading: `text-lg font-bold text-white` — copy: "Practice Calendar"
- RTL: `flex-row-reverse` when `isRTL`

#### Summary stats row
Three stat chips displayed horizontally above the heatmap. Each chip:
- Container: `bg-white/5 border border-white/10 rounded-lg px-3 py-2`
- Number: `text-xl font-bold text-white`
- Label: `text-xs text-white/60`
- Chips: [Total days] [Current streak] [Longest streak]
- Responsive: `grid grid-cols-3 gap-2` (all three on one row, no wrapping)

#### Heatmap grid
- Library: `react-activity-calendar` (npm package, no registry vetting required — no shadcn)
- Props: `data`, `theme` (see Color section), `hideColorLegend={true}`, `hideTotalCount={true}`, `showWeekdayLabels={false}`, `blockSize={12}`, `blockRadius={3}`, `blockMargin={3}`
- Responsive: `blockSize={10}` at <sm, `blockSize={12}` at sm+
- RTL: wrap in `<div style={{ direction: 'ltr' }}>` and apply `transform: scaleX(-1)` to the ActivityCalendar element; also apply `scaleX(-1)` to any month labels container so they read left-to-right after flip (D-04)
- Month labels: locale-aware short month names via `Intl.DateTimeFormat(locale, { month: 'short' })` — passed through `labels.months` prop

#### Loading skeleton
- Matches PracticeLogCard skeleton: `bg-white/10 animate-pulse rounded`
- Show three stat chip skeletons (h-12 each) and a heatmap area skeleton (h-20 w-full)

#### Empty state (no data)
- Full 52-week grid shown in all-gray (all cells at level 0)
- Below grid: `text-sm text-white/60 text-center mt-4` — copy per Copywriting Contract below
- Do NOT hide the grid in empty state — D-10 specifies the full grid must show

### Modified components

#### `ParentPortalPage.jsx`
- Import and render `<PracticeHeatmapCard studentId={user?.id} />` directly below the subscription card div, inside the existing `max-w-lg mx-auto px-4 sm:px-6 py-6` container
- Spacing: `mt-6` between subscription card and heatmap card
- No subscription gate — heatmap is always visible (D-02)
- Structure the heatmap card wrapper with a `data-section="practice-heatmap"` attribute to enable future premium gating wrapper without refactoring (D-11 future note)

## Interaction Contract

### States

| State | Trigger | Visual |
|-------|---------|--------|
| Loading | Query in-flight (`isLoading: true`) | Animated pulse skeletons for stats row + heatmap area |
| Empty | Query resolved, 0 practice records | Full 52-week all-gray heatmap + empty copy below |
| Populated | Query resolved, 1+ records | Gray cells for not-practiced days, emerald cells for practiced days |
| Error | Query throws | Error message text below stats area (no skeleton, no toast) |

### No interactive elements on day cells
Per D-09: no tooltips, no tap handlers, no hover effects on individual cells. Day cells are purely decorative data-viz. Pass `interactive={false}` to react-activity-calendar.

### RTL direction (D-04)
- `isRTL = i18n.dir() === 'rtl'` (existing pattern from ParentPortalPage.jsx)
- Card header row: `flex-row-reverse` when RTL
- Stats row: `dir="rtl"` on container (text aligns naturally)
- Heatmap: wrap ActivityCalendar in `<div style={{ direction: 'ltr' }}>` and apply `style={{ transform: 'scaleX(-1)' }}` to ActivityCalendar itself. This mirrors the grid so most-recent week appears rightmost in Hebrew layout. Month labels must also be flipped: apply `scaleX(-1)` via CSS class on the labels container after render.

### Reduced motion
- Loading skeleton: `animate-pulse` is CSS-only; no framer-motion used in this component
- No framer-motion animations in this phase (heatmap is a static visualization)
- `useMotionTokens` is not required here

## Data Contract

New service method `practiceLogService.getHistoricalLogs(studentId, startDate, endDate)`:
- Returns: `Array<{ practiced_on: string }>` (ISO date strings, e.g. `"2026-01-15"`)
- RLS: parent can read their child's logs via existing Supabase RLS on `instrument_practice_logs`

Data transformation for `react-activity-calendar`:
```js
// Transform flat date array into ActivityCalendar data format
// ActivityCalendar expects: { date: "YYYY-MM-DD", count: 0|1, level: 0|1 }[]
// Fill 52 weeks (364 days ending today) — missing dates = count 0, level 0
// Practiced dates = count 1, level 1
```

TanStack Query key: `["practice-history", studentId]` with `staleTime: 5 * 60 * 1000`.

Summary stats computed client-side from the returned array (at most 365 rows — D-12):
- **Total days:** `array.length`
- **Current streak:** delegate to `practiceStreakService.getPracticeStreak()` (already has this logic)
- **Longest streak:** compute from sorted date array by checking consecutive-day chains
