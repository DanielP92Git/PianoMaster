Three targeted changes to TrailNodeModal.jsx:

**1. Skill bubble data source (line 314 area):**
Replace `node.skills.map(...)` with a computed array that prefers `focusNotes` when non-empty:

```javascript
// Determine which skills to show: focusNotes for Discovery nodes, full skills for others
const displaySkills = (node.noteConfig?.focusNotes?.length > 0)
  ? node.noteConfig.focusNotes
  : node.skills;
```

Then change the `.map()` to iterate over `displaySkills` instead of `node.skills`. Place the `displaySkills` computation just above the "Skills 3D bubble badges" section (around line 308), NOT inside the JSX.

**2. Dynamic text sizing for long labels (line 324 area):**
Add dynamic font sizing so Hebrew two-word accidental names fit inside the bubble. Inside the `.map()` callback, after computing `displaySkill`, calculate a size class:

```javascript
const textSizeClass = displaySkill.length > 4
  ? 'text-xs sm:text-sm'    // Long labels (Hebrew accidentals like "פה דיאז")
  : 'text-xl sm:text-2xl';  // Short labels (C, D, F♯)
```

Replace the hardcoded `text-xl sm:text-2xl` in the bubble's className (line 324) with `${textSizeClass}`. Keep all other classes (`w-14 h-14 sm:w-16 sm:h-16 rounded-full flex items-center justify-center font-bold text-white select-none`) unchanged.

**3. Unicode-safe description fallback (line 304 area):**
Add a small inline helper (or compute before the JSX return) that converts ASCII accidentals to Unicode in the fallback description:

```javascript
// Convert ASCII accidentals to Unicode symbols for display
const sanitizeAccidentals = (str) =>
  str?.replace(/([A-G])#/g, '$1♯').replace(/([A-G])b(?![a-z])/g, '$1♭') || str;
```

Then change line 304 from:
```jsx
{t(`descriptions.${node.name}`, { defaultValue: node.description || t('modal.subtitle') })}
```
to:
```jsx
{t(`descriptions.${node.name}`, { defaultValue: sanitizeAccidentals(node.description) || t('modal.subtitle') })}
```

This ensures that even if the i18n lookup falls through to the defaultValue, the user sees F♯ not F#.

**Important constraints:**
- Do NOT change any other component behavior (progress bars, exercise list, action buttons, etc.)
- Do NOT modify node data files — the fix is entirely in the modal rendering
- The `sanitizeAccidentals` regex uses a negative lookahead `(?![a-z])` to avoid converting "Bb" in words like "Bubble" — it only matches single uppercase letter + "b" NOT followed by lowercase (i.e., note name patterns like "Bb4", "Eb", "Db3")
