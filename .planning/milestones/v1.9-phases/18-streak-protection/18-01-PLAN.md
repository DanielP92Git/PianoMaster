**New export: `getStreakState()`**
Returns an object with all streak-related state for UI consumption:
```javascript
{
  streakCount: number,        // current streak
  freezeCount: number,        // 0-3
  weekendPassEnabled: boolean,
  inGraceWindow: boolean,     // true if last practice was >24h but <36h ago
  lastFreezeConsumedAt: string|null, // for "freeze saved your streak" toast
  comebackBonus: {
    active: boolean,
    expiresAt: string|null,   // ISO timestamp
    daysLeft: number           // 0-3
  }
}
```
This queries `current_streak` with all new columns + `last_practiced_date` in a single flow. Uses same in-flight dedup and cooldown pattern as existing getStreak(). Query key for React Query: `["streak-state", userId]`.

**Refactor `updateStreak()`**
Replace the current midnight-cutoff logic with:

1. **Get last practice timestamp** (not calendar date) from `last_practiced_date`.
2. **Calculate hours since last practice**: `(now - lastPractice) / (1000 * 60 * 60)`.
3. **Weekend pass check**: If `weekend_pass_enabled` is true, check if any skipped days between lastPractice and now are exclusively Friday (day 5) and/or Saturday (day 6). If so, those days don't count as missed. Use `getDay()` (0=Sunday, 5=Friday, 6=Saturday). The check: enumerate calendar days between lastPractice and today; if ALL intermediate days are Fri/Sat, treat as consecutive.
4. **Same-day check**: If practiced today (same calendar date), return early — no streak change.
5. **Grace window (36 hours)**: If hoursSinceLastPractice <= 36 AND it's a new calendar day, treat as consecutive — increment streak.
6. **Past grace**: If hoursSinceLastPractice > 36:
   a. Check `streak_freezes > 0`. If yes, decrement `streak_freezes` by 1, set `last_freeze_consumed_at = now`, streak is preserved (no reset).
   b. If no freeze available, streak breaks: set `streak_count = 1`, create comeback bonus (`comeback_bonus_start = now`, `comeback_bonus_expires = now + 3 days`).

7. **Freeze earning**: After updating streak, check if `streak_count` is a multiple of 7 AND `streak_count > 0` AND `streak_freezes < 3`. Also check `last_freeze_earned_at` is not within the current streak milestone window (prevents double-earn if updateStreak is called twice). If eligible, increment `streak_freezes`, set `last_freeze_earned_at = now`. Return `freezeEarned: true` in the result.

8. **Comeback bonus clearing**: If comeback bonus is active AND `comeback_bonus_expires < now`, clear both `comeback_bonus_start` and `comeback_bonus_expires` to NULL.

9. **Return object**: `{ newStreak, freezeEarned, freezeConsumed, streakBroken, comebackBonusActivated }`.

**Refactor `getStreak()`**: Still returns just the streak count number (backward compatible), but now reads from the same query as getStreakState.

**Keep `resetStreak()`**: Also clears freeze-related and comeback columns.

**Keep `getLastPracticeDate()`**: Unchanged.

**Important edge cases**:
- First time practicing (no lastPractice): streak = 1, no freeze/grace logic applies.
- Network failure: existing cooldown/dedup patterns preserved.
- Weekend pass + grace: weekend days are simply skipped in the day enumeration; grace window still applies to the non-weekend gap.
- Timezone: all comparisons use local time via `new Date()` (matches existing pattern — client-side calendar day comparison).

**Do NOT modify** useStreakWithAchievements.js in this plan — it will continue calling updateStreak() which returns { newStreak, ... }. The hook's onSuccess handler will be updated in Plan 02 to handle freezeEarned/comebackBonusActivated.
