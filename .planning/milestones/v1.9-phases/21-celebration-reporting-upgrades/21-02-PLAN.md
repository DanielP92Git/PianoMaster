**2. Create `supabase/functions/send-weekly-report/index.ts`:**

Follow the exact pattern of `send-daily-push/index.ts`:

- **File header comment**: Document purpose, security model, env vars (CRON_SECRET, BREVO_API_KEY, SENDER_EMAIL, SENDER_NAME, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY, WEEKLY_REPORT_HMAC_SECRET)
- **Import**: `createClient` from `https://esm.sh/@supabase/supabase-js@2`
- **HMAC helper function**: `generateUnsubscribeToken(studentId: string, secret: string): string` -- use Web Crypto API (Deno has `crypto.subtle`):
  ```typescript
  async function generateUnsubscribeToken(studentId: string, secret: string): Promise<string> {
    const key = await crypto.subtle.importKey(
      'raw',
      new TextEncoder().encode(secret),
      { name: 'HMAC', hash: 'SHA-256' },
      false,
      ['sign']
    );
    const sig = await crypto.subtle.sign('HMAC', key, new TextEncoder().encode(studentId));
    return Array.from(new Uint8Array(sig)).map(b => b.toString(16).padStart(2, '0')).join('');
  }
  ```
- **Email HTML generator function** `generateWeeklyReportHTML(params)`: Takes `{ childName?, daysPracticed, streakCount, nodesCompleted, currentLevel, unsubscribeUrl }`. Returns table-based HTML matching consent email branding:
  - Same purple gradient header: `background: linear-gradient(135deg, #6366f1 0%, #8b5cf6 50%, #a855f7 100%)`
  - Title: "PianoMaster" with piano emoji
  - Subtitle: "Weekly Progress Report"
  - Warm greeting: "Great news about {childName}'s piano journey this week!"
  - Stats table in a `#f8fafc` bg card with 4 rows:
    - Days practiced: "{daysPracticed}/7 days" with encouraging note
    - Current streak: "{streakCount} days" with fire emoji if > 0
    - Nodes completed: "{nodesCompleted} new lessons" this week
    - Current level: "Level {currentLevel}"
  - Encouraging closing: "Keep encouraging your child to practice! Every session builds confidence and skill."
  - Footer with unsubscribe link: "To stop receiving these weekly updates, <a href='{unsubscribeUrl}'>click here</a>."
  - Copyright footer matching consent email

- **Main handler** `Deno.serve(async (req) => { ... })`:
  1. CORS preflight handler (same as send-daily-push)
  2. POST-only check (405 for other methods)
  3. Verify `x-cron-secret` header against `CRON_SECRET` env var (401 on mismatch)
  4. Get `BREVO_API_KEY`, `SENDER_EMAIL` (default 'noreply@pianomaster.app'), `SENDER_NAME` (default 'PianoMaster'), `WEEKLY_REPORT_HMAC_SECRET` from env
  5. Initialize Supabase service role client
  6. Calculate `sevenDaysAgo` ISO string (7 days before now)
  7. Query eligible parents:
     ```typescript
     const { data: eligibleStudents } = await supabase
       .from('push_subscriptions')
       .select(`
         student_id,
         last_weekly_report_at,
         students!inner (
           parent_email,
           total_xp,
           current_level
         )
       `)
       .eq('parent_consent_granted', true)
       .eq('weekly_report_opted_out', false);
     ```
  8. **Per-student processing loop** with sent/failed/skipped counters:
     - **Skip** if `last_weekly_report_at` is within the last 6 days (prevents duplicate Monday sends)
     - **Skip** if `students.parent_email` is null/empty or fails email regex
     - Normalize student data: `const student = Array.isArray(row.students) ? row.students[0] : row.students;`
     - **Gather 7-day stats**:
       - Days practiced: query `students_score` where `student_id` and `created_at >= sevenDaysAgo`, count distinct dates
       - Nodes completed: query `student_skill_progress` where `last_practiced >= sevenDaysAgo` and `stars > 0`, count distinct `node_id`
       - Streak: query `current_streak` for `streak_count`
     - **Generate unsubscribe token**: `await generateUnsubscribeToken(studentId, hmacSecret)`
     - **Build unsubscribe URL**: `${siteUrl}/functions/v1/unsubscribe-weekly-report?sid=${studentId}&token=${token}`
       - `siteUrl` from `Deno.env.get('SUPABASE_URL')`
     - **Generate email HTML**: call `generateWeeklyReportHTML(...)` with gathered data
     - **Send via Brevo**: same pattern as consent email -- POST to `https://api.brevo.com/v3/smtp/email` with 30s timeout
     - **On success**: update `last_weekly_report_at = now()` for this student's push_subscriptions row
     - **On Brevo error**: log error, increment `failed` counter, continue to next student (do not abort loop)
  9. Log summary and return: `{ sent, failed, skipped, total }`

Note: The function does NOT add `headers` like `List-Unsubscribe` to the Brevo API call because Brevo's transactional API does not support arbitrary email headers. The unsubscribe link is in the HTML body instead.

**Create `supabase/functions/unsubscribe-weekly-report/index.ts`:**

This is a small Edge Function that handles GET requests from the unsubscribe link in weekly emails. It verifies the HMAC token, updates the database, and returns a simple HTML confirmation page.

- **File header comment**: Document purpose, security model (HMAC token verification, no login required), env vars (WEEKLY_REPORT_HMAC_SECRET, SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY)
- **Import**: `createClient` from `https://esm.sh/@supabase/supabase-js@2`
- **HMAC verification function** (same as in send-weekly-report):
  ```typescript
  async function verifyUnsubscribeToken(studentId: string, token: string, secret: string): Promise<boolean> {
    const key = await crypto.subtle.importKey(
      'raw',
      new TextEncoder().encode(secret),
      { name: 'HMAC', hash: 'SHA-256' },
      false,
      ['sign']
    );
    const sig = await crypto.subtle.sign('HMAC', key, new TextEncoder().encode(studentId));
    const expected = Array.from(new Uint8Array(sig)).map(b => b.toString(16).padStart(2, '0')).join('');
    return token === expected;
  }
  ```
- **HTML response generator** `generateResponseHTML(success: boolean): string`:
  - Simple, clean HTML page with PianoMaster branding
  - If success: "You have been unsubscribed from PianoMaster weekly progress reports. You can re-enable these emails from the app's Settings page."
  - If failure: "Invalid or expired unsubscribe link. Please try again from a recent email."
  - Uses the same purple gradient header as the email template
  - Responsive, table-free (this is a web page, not email -- can use modern CSS)
- **Main handler** `Deno.serve(async (req) => { ... })`:
  1. Accept GET requests only (this is a browser click from email). Return 405 for other methods.
  2. Parse URL query params: `sid` (student ID) and `token` (HMAC hex)
  3. Validate both params exist, return 400 HTML if missing
  4. Get `WEEKLY_REPORT_HMAC_SECRET` from env
  5. Verify token: `await verifyUnsubscribeToken(sid, token, secret)` -- return error HTML if invalid
  6. Initialize Supabase service role client
  7. Update: `await supabase.from('push_subscriptions').update({ weekly_report_opted_out: true }).eq('student_id', sid)`
  8. Return success HTML page with Content-Type `text/html`
  9. Log: `unsubscribe-weekly-report: student ${sid} opted out`

**Security considerations:**
- HMAC prevents URL guessing -- only recipients of the email have valid tokens
- Service role client bypasses RLS (parent has no Supabase account)
- No PII in logs (student_id only, no names or emails)
- `verify_jwt = false` is NOT needed since this is a GET handler called by clicking a link (no JWT involved). However, Supabase Edge Functions require JWT by default. Add a note that config.toml should set `verify_jwt = false` for this function, similar to send-daily-push. The HMAC token provides authentication instead.
