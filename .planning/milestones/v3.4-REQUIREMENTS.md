# Requirements: PianoApp v3.4 — Rhythm Games Responsive UX (ARCHIVED)

**Status:** ✅ SHIPPED 2026-05-12 — all 18 requirements Complete
**Archived:** 2026-05-12

**Defined:** 2026-05-07
**Core Value:** Children's data must be protected and inaccessible to unauthorized users.

**Milestone goal:** Drop the route-based landscape lock for rhythm games and make each renderer responsive across phone-portrait, phone-landscape, tablet-portrait, and tablet-landscape — so vertical-card-stack games (dictation) stop scrolling, and tablets get layouts that use the extra space.

## v3.4 Requirements

### Infrastructure

- [x] **INFRA-01**: Rhythm routes (7 total: metronome-trainer, rhythm-reading, rhythm-dictation, arcade-rhythm, visual-recognition, syllable-matching, mixed-lesson) removed from `LANDSCAPE_ROUTES` in `src/App.jsx`; remain in `gameRoutes` (sidebar/header still hidden)
- [x] **INFRA-02**: New `NeedsLandscapeContext` provider lets renderers declare `needsLandscape` via `useDeclareNeedsLandscape(boolean)` hook
- [x] **INFRA-03**: `RotatePromptOverlay` shows only when `(context.needsLandscape) AND (viewport < 768px) AND (orientation === portrait)`
- [x] **INFRA-04**: Phone vs. tablet cutoff = Tailwind `md` breakpoint (768px); tablets never see rotate prompt regardless of orientation

### Core Renderers — Non-Notation

- [x] **CORE-01**: `DiscoveryIntroQuestion` renders without scroll on iPhone SE portrait (375×667); tablet-portrait variant added (single column with larger SVG)
- [x] **CORE-02**: `SyllableMatchingQuestion` + `DictationChoiceCard` render as 2×2 card grid on phone-portrait, 1×4 row on wider viewports
- [x] **CORE-03**: `PulseQuestion` tap zone fills viewport across all orientations and sizes
- [x] **CORE-04**: `VisualRecognitionQuestion` renders 2×2 on phone-portrait, 1×4 on wider viewports
- [x] **CORE-05**: `RhythmDictationQuestion` staff at top, choice cards in 2×2 grid on phone-portrait, 1×N on wider viewports — no scroll trap

### Notation Renderers — Content-Driven Prompt

- [x] **NOTATION-01**: `RhythmReadingQuestion` computes `needsLandscape` from pattern length × time-signature factor; short patterns render in phone-portrait without prompt, long patterns trigger prompt on phone only
- [x] **NOTATION-02**: `RhythmTapQuestion` uses the same `needsLandscape` heuristic — long-pattern variants trigger prompt on phone, all patterns render inline on tablet
- [x] **NOTATION-03**: `needsLandscape` heuristic helper extracted to `src/components/games/rhythm-games/utils/needsLandscape.js` with unit test of the threshold

### Wrappers + Setup Audit

- [x] **WRAPPER-01**: 6 standalone game wrappers (`RhythmDictationGame`, `RhythmReadingGame`, `MetronomeTrainer`, `VisualRecognitionGame`, `SyllableMatchingGame`, `MixedLessonGame`) audited for hardcoded landscape assumptions; fixes applied where needed
- [x] **WRAPPER-02**: `RhythmGameSetup` and `RhythmGameSettings` setup screens render correctly across all 4 quadrants (phone/tablet × portrait/landscape) — RhythmGameSettings marked @deprecated (dead code, no UI consumer)
- [x] **WRAPPER-03**: Supporting components (`CountdownOverlay`, `BossIntroOverlay`, `FloatingFeedback`, `MetronomeDisplay`, `TapArea`) audited for responsive sanity

### Tablet 2-Column Optimization

- [x] **TABLET-01**: Cards-based renderers (dictation, syllable-matching, visual-recognition) use real 2-col layout on tablet-landscape — not just centered with whitespace; uses Tailwind `md:`/`lg:` breakpoints

### ArcadeRhythmGame Portrait — Separate Phase

- [x] **ARCADE-01**: Spike vertical lanes (top-down scrolling) on phone-portrait — ~10-min throwaway prototype to feel-test timing/visual experience
- [x] **ARCADE-02**: Based on spike outcome, either ship vertical-lane redesign (with horizontal lanes preserved on tablet-landscape) OR declare `needsLandscape=true` and rely on rotate prompt from INFRA-02/03 — shipped ROTATE-PROMPT path per 35-SPIKE.md verdict
