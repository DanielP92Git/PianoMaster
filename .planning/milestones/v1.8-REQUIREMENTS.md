# Requirements: PianoApp v1.8 App Monetization

**Defined:** 2026-02-25
**Status:** SHIPPED (2026-03-01)
**Core Value:** Children's data must be protected and inaccessible to unauthorized users

## v1.8 Requirements

Requirements for freemium monetization. Each maps to roadmap phases.

### Subscription Infrastructure

- [x] **SUB-01**: Subscription database table exists with RLS restricting client to SELECT-only
- [x] **SUB-02**: Subscription plans table seeded with monthly and yearly prices in ILS and USD
- [x] **SUB-03**: Free tier boundary defined in a single config file (Unit 1 per path = free)
- [x] **SUB-04**: Subscription writes restricted to webhook Edge Function via service role key

### Payment Integration

- [x] **PAY-01**: Payment processor account registered with sandbox verified and payout path to Israeli bank confirmed
- [x] **PAY-02**: Webhook Edge Function receives and processes subscription lifecycle events (created, updated, canceled)
- [x] **PAY-03**: Webhook handler is idempotent and handles duplicate or out-of-order events
- [x] **PAY-04**: Webhook verifies cryptographic signature before processing any event

### Content Gating

- [x] **GATE-01**: Trail nodes beyond Unit 1 in each path show as locked for free-tier users
- [x] **GATE-02**: Two visually distinct lock states exist: prerequisite lock (gray) vs subscription lock (gold)
- [x] **GATE-03**: Content gate enforced at database RLS layer, not only client-side React checks

### Child-Facing UI

- [x] **CHILD-01**: Tapping a subscription-locked node shows a friendly modal with zero pricing and zero buy buttons
- [x] **CHILD-02**: No purchase interface, pricing, or payment form is reachable from any screen a logged-in student can access

### Parent-Facing Pages

- [x] **PARENT-01**: Pricing page displays monthly and yearly plans with prices in user's currency (ILS or USD)
- [x] **PARENT-02**: Parent can initiate checkout via the selected payment processor from the pricing page
- [x] **PARENT-03**: Success page confirms subscription activation after webhook processes the payment
- [x] **PARENT-04**: Parent can cancel subscription from within the app without leaving to an external portal
- [x] **PARENT-05**: Cancellation preserves full access until current billing period ends (no immediate revocation)

### Service Layer

- [x] **SVC-01**: SubscriptionContext provides isPremium status globally to all components
- [x] **SVC-02**: Subscription status query uses staleTime: 0 to prevent stale gate decisions
- [x] **SVC-03**: Subscription status updates propagate to client in real-time after webhook processes

### Compliance

- [x] **COMP-01**: Service worker excludes subscription status endpoints from cache
- [x] **COMP-02**: Service worker cache version bumped to force refresh on monetization deploy
- [x] **COMP-03**: Parental consent email template updated to disclose payment processor name
- [x] **COMP-04**: No child personal data is shared with the payment processor (parent email only for billing)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Virtual currency / coins / gems | COPPA 2025 prohibits financial incentives targeting children; FTC enforcement risk |
| FOMO countdown timers or urgency language | FTC dark patterns enforcement; inappropriate for children's app |
| In-app purchase buttons visible to children | FTC requires parental consent for children's purchases (Genshin $20M settlement) |
| Ads of any kind | COPPA prohibits behavioral ads to children; school districts reject ad-supported apps |
| Feature gating (locking features not content) | Gate units/content only; all users get all app features |
| Leaderboard with names as paid perk | COPPA-03 already anonymizes usernames; reversing creates compliance regression |
| Streak-based access lock | FTC targets anxiety-inducing mechanics for children |
| Separate pricing tiers for features | Content gating only; feature tiers create UX confusion |
