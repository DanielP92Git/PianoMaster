3. **Add free play XP awarding in processTrailCompletion:**
   In the `else` branch (line ~545, "Not a trail node, no processing needed"), replace the simple `setIsProcessingTrail(false)` with:
   ```javascript
   // Free play: award XP based on score
   if (user?.id && scorePercentage > 0) {
     try {
       const freePlayXP = calculateFreePlayXP(scorePercentage, comebackActive ? 2 : 1);
       if (freePlayXP > 0) {
         const xpResult = await awardXP(user.id, freePlayXP);
         setXpData({ totalXP: freePlayXP, ...xpResult });
         queryClient.invalidateQueries({ queryKey: ["student-xp", user.id] });
       }
     } catch (error) {
       console.error("Error awarding free play XP:", error);
     }
   }
   setIsProcessingTrail(false);
   ```
   Also set `setIsProcessingTrail(true)` at the start for free play too (it's currently `!!nodeId` which is false for free play — change to always true so the loading state works).

4. **Update the achievement bonus in streak callback:**
   In the `updateStreakWithAchievements.mutate` onSuccess callback (lines ~378-396): the `achievementBonus` state is now gone. The achievement XP is already awarded by achievementService via awardXP (Plan 01 change). Remove the `setAchievementBonus(bonus)` call. The achievement bonus no longer needs to be tracked separately because it flows through the XP system directly.

5. **Remove percentile state and effect:**
   - Delete `percentileMessage` state (line ~251)
   - Delete `setPercentileMessage` (same)
   - Delete the percentile calculation useEffect (lines ~594-608)

6. **Clean up the return object:**
   - Remove: `animatedTotal`, `actualGain`, `totalPointsData`, `percentileMessage`
   - Keep: everything else (xpData, animatedXPGain, stars, celebrationData, etc.)

7. **Keep `usePointBalance` and related accessory unlock detection** — this is still needed for detecting accessory unlocks based on the new XP-based balance. Update the `initialProgressRef` and `currentProgress` to use `totalXP` instead of `totalPoints`:
   - In setBaselineProgress (line ~339): change `totalPoints: (pointsBalance?.earned || 0) - predictedPointsEarned` to `totalXP: pointsBalance?.earned || 0` (the "earned" field from apiAccessories now returns total_xp per Plan 01)
