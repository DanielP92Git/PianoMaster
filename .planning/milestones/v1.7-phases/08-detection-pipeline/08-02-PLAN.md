**Step 1: Compute shortest note duration in beats**

Add a `useMemo` that computes the shortest note in the current pattern, expressed in beats. Place it near the existing `currentPattern` / `gameSettings` logic (around line 500-600 area where pattern state is managed).

```javascript
const shortestNoteBeats = useMemo(() => {
  if (!currentPattern?.notes) return 1; // default to quarter note
  const secondsPerBeat = 60 / gameSettings.tempo;
  const durations = currentPattern.notes
    .filter(n => n.type !== 'rest' && typeof n.duration === 'number')
    .map(n => n.duration / secondsPerBeat);
  return durations.length > 0 ? Math.min(...durations) : 1;
}, [currentPattern, gameSettings.tempo]);
```

**Step 2: Pass bpm and shortestNoteDuration to useMicNoteInput**

Update the `useMicNoteInput` call (around line 841-847) to include `bpm` and `shortestNoteDuration`:

Change from:
```javascript
const { audioLevel, isListening, startListening, stopListening, debug } =
  useMicNoteInput({
    isActive: false,
    noteFrequencies,
    ...MIC_INPUT_PRESETS.sightReading,
    onNoteEvent: handleNoteEvent,
  });
```

To:
```javascript
const { audioLevel, isListening, startListening, stopListening, debug } =
  useMicNoteInput({
    isActive: false,
    noteFrequencies,
    rmsThreshold: MIC_INPUT_PRESETS.sightReading.rmsThreshold,
    tolerance: MIC_INPUT_PRESETS.sightReading.tolerance,
    bpm: gameSettings.tempo,
    shortestNoteDuration: shortestNoteBeats,
    onNoteEvent: handleNoteEvent,
  });
```

Important: Do NOT spread `...MIC_INPUT_PRESETS.sightReading` anymore because it would set `onFrames`, `offMs`, `minInterOnMs`, `changeFrames` as explicit props which would be used as fallback values when bpm is null. Instead, only pass the non-timing preset values (`rmsThreshold`, `tolerance`). The timing values will be computed dynamically by `computeTimingParams()` inside `useMicNoteInput` since `bpm` is now provided.

**Step 3: Remove fixed DEBOUNCE_MS from handleNoteDetected**

Find the `DEBOUNCE_MS = 80` constant around line 1593 and the per-note debounce check that uses it:

```javascript
const DEBOUNCE_MS = 80;
const lastTime = lastDetectionTimesRef.current[matchingNoteIndex] ?? -Infinity;
if (elapsedTimeMs - lastTime < DEBOUNCE_MS) {
  // ...skip
}
```

Remove the DEBOUNCE_MS constant and the debounce check entirely. The state machine's COOLDOWN state in useMicNoteInput now handles inter-onset debouncing with BPM-aware timing. Having both creates a double-penalty that swallows fast notes at high tempos (Pitfall 5 from research).

Keep `lastDetectionTimesRef` and its reset calls in place — they may still be useful for debug logging — but remove the early-return guard that uses `DEBOUNCE_MS`.
