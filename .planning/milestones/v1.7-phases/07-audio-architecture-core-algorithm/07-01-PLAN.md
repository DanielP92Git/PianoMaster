  <action>
1. Install pitchy: `npm install pitchy@4.1.0`

2. Create `src/contexts/AudioContextProvider.jsx` following the pattern from the RESEARCH.md:

- Create React Context (`AudioCtx`) with `createContext(null)`
- Export `AudioContextProvider` component and `useAudioContext` hook
- The provider manages:
  - `audioContextRef` (useRef) — single AudioContext instance
  - `analyserRef` (useRef) — single AnalyserNode instance
  - `streamRef` (useRef) — MediaStream from getUserMedia
  - `sourceRef` (useRef) — MediaStreamSourceNode (store to prevent GC)
  - `isReady` state — whether analyser is set up and ready
  - `micPermission` state — 'prompt' | 'granted' | 'denied'

- `getOrCreateAudioContext()` callback:
  - If audioContextRef.current exists and not 'closed', resume if suspended, return it
  - Otherwise create `new (window.AudioContext || window.webkitAudioContext)()`
  - Store in audioContextRef.current and return

- `requestMic()` async callback — called by game components when they need mic:
  - AUDIO-01: `navigator.mediaDevices.getUserMedia({ audio: { echoCancellation: false, noiseSuppression: false, autoGainControl: false } })`
  - Store stream in streamRef
  - Call getOrCreateAudioContext()
  - Create MediaStreamSource, create AnalyserNode
  - AUDIO-02: `analyser.smoothingTimeConstant = 0.0`
  - AUDIO-03: `analyser.fftSize = 4096`
  - Connect source → analyser
  - Store analyser in analyserRef, source in sourceRef
  - Set micPermission to 'granted', isReady to true
  - Return `{ audioContext: ctx, analyser }`
  - On catch: if error is NotAllowedError, set micPermission to 'denied'; rethrow

- `releaseMic()` callback:
  - Stop all tracks on streamRef.current
  - Null out streamRef, analyserRef, sourceRef
  - Set isReady to false
  - Do NOT close audioContextRef — that's for unmount only

- ARCH-05: `suspendAudio()` / `resumeAudio()` callbacks:
  - suspendAudio: if audioContext state is 'running', call suspend()
  - resumeAudio: if audioContext state is 'suspended', call resume()

- Cleanup effect on unmount:
  - Call releaseMic()
  - Close audioContextRef if not already closed, null it out

- Basic `visibilitychange` listener:
  - On 'hidden': call suspendAudio() (per context decision: "pause and resume when app goes to background")
  - On 'visible': call resumeAudio()
  - Phase 09 will build iOS-specific recovery on top of this

- Context value exposes: `audioContextRef, analyserRef, isReady, micPermission, requestMic, releaseMic, suspendAudio, resumeAudio, getOrCreateAudioContext`

- `useAudioContext()` hook: `useContext(AudioCtx)` with error if null ("useAudioContext must be used inside AudioContextProvider")

Note: Do NOT import pitchy in this file — that happens in usePitchDetection (Plan 02). This plan only installs the package.

  <action>
1. Import `AudioContextProvider` from `../contexts/AudioContextProvider` in App.jsx

2. Wrap each game route element with AudioContextProvider. Use Option A from research (minimal diff — wrap individual game elements):

```jsx
<Route
  path="/notes-master-mode/notes-recognition-game"
  element={<AudioContextProvider><NotesRecognitionGame /></AudioContextProvider>}
/>
<Route
  path="/notes-master-mode/sight-reading-game"
  element={<AudioContextProvider><SightReadingGame /></AudioContextProvider>}
/>
<Route
  path="/rhythm-mode/metronome-trainer"
  element={<AudioContextProvider><MetronomeTrainer /></AudioContextProvider>}
/>
```

3. Do NOT wrap non-game routes (dashboard, trail, settings, etc.). Per locked decision: "game routes (not app root)" — mic permission never requested on non-game pages.

4. Do NOT wrap MemoryGame or NotesMasterMode or RhythmMasterMode — those are menu/selection pages, not mic-using games.

5. The key is that `AudioContextProvider` is mounted fresh when entering a game route and unmounted when leaving — the cleanup effect releases the mic automatically. When navigating between two game routes, React Router unmounts the old element and mounts the new one, giving each game a fresh provider. This matches the "seamless trail transitions" requirement.
  </action>
