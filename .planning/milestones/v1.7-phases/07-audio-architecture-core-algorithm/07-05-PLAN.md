3. **Pass requestMic() analyser to startListening at call time:** SightReadingGame uses `startListeningSync` which wraps `startListening` from useMicNoteInput. The key change is in `startListeningSync` (around line 855). Currently:
   ```javascript
   const startListeningSync = useCallback(async () => {
     micIsListeningRef.current = true;
     try {
       await startListening();
     } catch (err) {
       micIsListeningRef.current = false;
       throw err;
     }
   }, [startListening]);
   ```
   Change to:
   ```javascript
   const startListeningSync = useCallback(async () => {
     micIsListeningRef.current = true;
     try {
       const { analyser, audioContext: ctx } = await requestMic();
       await startListening({ analyserNode: analyser, sampleRate: ctx.sampleRate });
     } catch (err) {
       micIsListeningRef.current = false;
       throw err;
     }
   }, [startListening, requestMic]);
   ```
   This follows the exact same call-time analyser passthrough pattern used in NotesRecognitionGame.jsx (lines 1671-1672 of that file). The `startListening` function from useMicNoteInput already supports an overrides object that gets forwarded to usePitchDetection's `startListening({ analyserNode, sampleRate })`.

**Do NOT modify `stopListeningSync`** — it calls `stopListening()` which does not need mic/analyser args.
