## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Custom onboarding (not react-joyride) | App's glassmorphism design hard to theme with library; 4-step overlay is simple enough |
| Plausible/Umami for analytics (not GA4) | Cookie-free, COPPA-safe by design, no consent dialog needed |
| Sentry is the only new npm dependency | Everything else uses existing libs or script tags |
| Daily challenges via DB table (not client-only) | Completion must be tamper-proof for XP rewards; mirrors daily_goals pattern |
| Ship Phase 06 with 2 challenge types only | Reduces risk; Speed Round + Review Challenge cover the core loop |
| location.state for challenge config | Reuses existing trail game launch pattern; minimal game component changes |
| Legal body text hardcoded English | Legal documents need exact wording; Hebrew headings translated via i18n |
