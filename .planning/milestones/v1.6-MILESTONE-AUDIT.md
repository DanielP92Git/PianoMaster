## E2E Flow Verification

### Flow 1: iOS Mobile User - Portrait Entry
User opens game in portrait on iOS -> useOrientation detects portrait -> useRotatePrompt shows prompt (not Android PWA) -> RotatePromptOverlay renders with translated text + ARIA -> User rotates or dismisses -> Game displays with landscape CSS modifiers -> VexFlow resizes via debounced hook.
**Status: COMPLETE**

### Flow 2: Android PWA User - Game Entry
User opens game on Android PWA -> useLandscapeLock enters fullscreen + locks landscape -> useRotatePrompt suppressed (isAndroidPWA=true) -> Game displays in landscape -> Escape exits fullscreen + unlocks -> Navigation cleanup fires.
**Status: COMPLETE**

### Flow 3: Desktop User - Game Entry
User opens game on desktop -> useRotatePrompt: isMobile=false (no prompt) -> useLandscapeLock: no-op -> Game displays normally with CSS modifiers based on viewport.
**Status: COMPLETE**

### Flow 4: Hebrew RTL User - Portrait Entry
Language=Hebrew, portrait mobile -> RotatePromptOverlay renders with dir="rtl" -> Hebrew text from he/common.json -> ARIA announces in Hebrew -> RTL layout mirrors -> User dismisses -> Game uses existing Hebrew i18n.
**Status: COMPLETE**
