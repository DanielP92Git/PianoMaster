B-01 root cause: The `send-consent-email` Edge Function returns 500 because `BREVO_API_KEY` is not set in the Supabase Edge Function environment. The code at line 221-223 of `send-consent-email/index.ts` checks `Deno.env.get('BREVO_API_KEY')` and returns 500 with "Email service configuration error" if missing.

Other Edge Functions (`send-weekly-report`, `process-account-deletions`) use the same env vars and were likely set up via `supabase secrets set` during their respective phases. The `send-consent-email` function may have been deployed before these secrets existed.

The user must set these secrets in the Supabase project. Steps:

1. Go to Supabase Dashboard -> your project -> Project Settings -> Edge Functions -> Secrets
   OR use CLI: `supabase secrets list` to check if BREVO_API_KEY exists
2. If BREVO_API_KEY is NOT listed, set it via CLI:
   `supabase secrets set BREVO_API_KEY=your-brevo-api-key SENDER_EMAIL=your-verified-sender@domain.com SENDER_NAME=PianoMaster`
   Or set them in the Dashboard UI under Edge Function Secrets.
3. Verify by testing the function:
   - Go to Supabase Dashboard -> Edge Functions -> send-consent-email -> Test
   - Send body: `{"parentEmail": "your-test-email@example.com", "consentUrl": "https://testpianomaster.netlify.app/consent/verify?token=test&student=test"}`
   - Expected: 200 response with `{"success": true, "messageId": "..."}` (or a Brevo error about invalid email, but NOT a 500 "Email service configuration error")
4. If the function returns 200 and the email arrives in your inbox, B-01 is resolved.
