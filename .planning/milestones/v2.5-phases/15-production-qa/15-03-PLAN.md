  <action>
In the fetch event listener in `public/sw.js`, the block at lines 178-192 skips ALL `.js` files, ALL `script` destination requests, and ALL `module` destination requests. This prevents Vite-built production bundles (`/assets/*.js`) from ever reaching the cache-first handler at lines 209-211.

Fix by restructuring the skip logic to only skip development-related patterns while allowing production `/assets/*.js` through:

1. REMOVE lines 187-189 (the three blanket skip lines):
   ```
   url.pathname.endsWith(".js") || // Skip all JS files - let browser handle them
   event.request.destination === "script" || // Skip script requests
   event.request.destination === "module" // Skip module requests
   ```

2. REPLACE the entire conditional block (lines 178-192) with this logic that keeps dev skips but allows production assets:
   ```javascript
   const url = new URL(event.request.url);
   const isDevRequest =
     url.pathname.includes("@vite") ||
     url.pathname.includes("@react-refresh") ||
     url.pathname.includes("@id") ||
     url.pathname.includes("node_modules") ||
     url.searchParams.has("t") || // Vite timestamp query param
     url.pathname.endsWith(".jsx") ||
     url.pathname.endsWith(".tsx") ||
     url.pathname.endsWith(".ts");
   if (isDevRequest) {
     return; // Let browser handle dev server requests directly
   }
   ```

This preserves all Vite dev server exclusions (HMR, module resolution, source files) but removes the blanket `.js` / `script` / `module` skip. Production `/assets/*.js` files (content-hashed by Vite) will now reach the existing cache-first handler on line 209 (`if (isSameOrigin && isAsset)`).

Also bump the cache version from `pianomaster-v7` to `pianomaster-v8` on line 4 to force existing SWs to refresh their cache on next visit.
