**Fix strategies (in order of preference):**

1. **Unused import** — Remove the entire import line or remove the unused named import from a destructured import. Example: if `import { Foo, Bar } from 'lib'` and only `Bar` is unused, change to `import { Foo } from 'lib'`. If the entire import is unused, delete the line.

2. **Unused variable assignment** — If a variable is assigned but never read, remove the assignment. If the variable is destructured from something (e.g., `const { used, unused } = obj`), remove `unused` from the destructuring. If it's a standalone `const x = someCall()` where the return value is never used but the call has side effects, keep the call but remove the assignment: `someCall()`.

3. **Unused function parameter** — For callback parameters that must exist for positional reasons (e.g., `(req, res, next)` where only `next` is used), prefix with underscore: `(_req, _res, next)`. For parameters at the end of the argument list that are simply not needed, remove them entirely.

4. **Unused catch variable** — Change `catch (error)` to `catch (_error)` (the underscore prefix is configured to be ignored by Plan 01's config change).

5. **Unused rest pattern** — For `const { used, ...rest } = obj` where `rest` is unused, change to `const { used, ..._rest } = obj` or remove the rest pattern entirely if feasible.

**Critical rules:**
- NEVER delete a variable that IS used somewhere else in the file — always verify by searching the file for other occurrences before deleting
- NEVER delete an import that is used as a JSX component (these look unused to grep but are used in JSX)
- For re-exported values (`export { something }`) that appear unused locally but are consumed by other files — do NOT remove these; they are not truly unused
- When removing an import from a destructured import, preserve the remaining imports on the same line or reformat as needed
- For the high-count files (MetronomeTrainer: 18, NotificationCenter: 10, TrailMap: 8, UnifiedGameSettings: 7, SightReadingGame: 7), be especially careful to read the full file and confirm each variable is truly unused before removal
