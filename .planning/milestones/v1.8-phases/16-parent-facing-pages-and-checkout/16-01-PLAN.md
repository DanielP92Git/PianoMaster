**create-checkout/index.ts** — Deno Edge Function that creates a Lemon Squeezy checkout URL:

1. Set `verify_jwt = true` (Supabase auto-verifies JWT; only authenticated users can call).
2. Reject non-POST requests (405).
3. Parse request body: `{ planId: string, studentId: string }`.
4. **Defense in depth:** Extract `auth.uid()` from the JWT (via `Authorization` header decoding or Supabase client `auth.getUser()`). Verify `studentId === auth.uid()`. Reject with 403 if mismatch.
5. Look up the plan in `subscription_plans` table: `SELECT lemon_squeezy_variant_id FROM subscription_plans WHERE id = planId AND is_active = true`. Return 400 if not found or variant ID is NULL.
6. Call `POST https://api.lemonsqueezy.com/v1/checkouts` with:
   - Headers: `Accept: application/vnd.api+json`, `Content-Type: application/vnd.api+json`, `Authorization: Bearer ${LS_API_KEY}`
   - Body: JSON:API structure with `checkout_options.embed: true`, `checkout_data.custom.student_id: studentId`, relationships to store (`LS_STORE_ID`) and variant (`variantId`).
7. Extract `checkoutUrl` from response `data.attributes.url`.
8. Return `{ checkoutUrl }` as JSON with 200.
9. On LS API error: return `{ error: 'Checkout creation failed' }` with 500. Log the LS API error for debugging.
10. Read env vars: `LS_API_KEY` (Lemon Squeezy API key), `LS_STORE_ID` (Lemon Squeezy store ID), `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` (auto-injected).
11. Create a Supabase client using the anon key from the request's Authorization header for the plan lookup (respects RLS), NOT the service role key. Alternative: Use service role client for simplicity since plan lookup is public anyway (RLS = USING true). Use service role for consistency with webhook pattern.
12. Add CORS headers for same-origin requests: `Access-Control-Allow-Origin: *`, `Access-Control-Allow-Headers: authorization, x-client-info, apikey, content-type`. Handle OPTIONS preflight.

**cancel-subscription/index.ts** — Deno Edge Function that cancels a subscription:

1. Set `verify_jwt = true`.
2. Reject non-POST requests (405).
3. Extract `auth.uid()` from JWT. No request body needed — the student ID comes from the JWT.
4. Create service role Supabase client (need to bypass RLS to read parent_subscriptions for the student).
5. Fetch `ls_subscription_id` and `current_period_end` from `parent_subscriptions WHERE student_id = auth.uid()`. Use `maybeSingle()`. Return 404 if no subscription found.
6. Call `DELETE https://api.lemonsqueezy.com/v1/subscriptions/{ls_subscription_id}` with:
   - Headers: `Accept: application/vnd.api+json`, `Content-Type: application/vnd.api+json`, `Authorization: Bearer ${LS_API_KEY}`
7. Extract `ends_at` from response `data.attributes.ends_at` — return this to the client for optimistic UI display.
8. Return `{ ok: true, endsAt }` as JSON with 200.
9. On error: return `{ error: 'Cancellation failed' }` with 500.
10. Add CORS headers (same pattern as create-checkout).

**IMPORTANT — extracting auth.uid() in Deno Edge Functions:**
Use the Supabase client initialized with the user's JWT (from `Authorization` header):
```typescript
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const authHeader = req.headers.get('Authorization')!;
const supabaseUser = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_ANON_KEY')!,
  { global: { headers: { Authorization: authHeader } } }
);
const { data: { user } } = await supabaseUser.auth.getUser();
```
