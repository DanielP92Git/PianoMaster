**ParentPortalPage.jsx** — `/parent-portal` subscription management:

1. **Imports:** React, useState, useEffect, useNavigate, Link, useTranslation, useUser, useSubscription, useQuery, useQueryClient, fetchSubscriptionDetail from subscriptionService, supabase from services/supabase, toast from react-hot-toast, Lucide icons (CreditCard, Calendar, XCircle, RefreshCw, Loader2, Crown, ArrowLeft, AlertTriangle).

2. **Fetch subscription detail:** Use `useQuery` with key `['subscription-detail', user?.id]` and `queryFn: () => fetchSubscriptionDetail(user?.id)`. Enabled when `user?.id` is truthy. `staleTime: 0` (matches subscription context pattern).

3. **No subscription state:** If query returns null and not loading, show:
   - Glass card with message: "No active subscription"
   - "Unlock Full Access" button → navigates to `/subscribe`

4. **Active subscription display:** Glass card showing:
   - Crown icon + "Premium Subscription" header (or plan name from detail)
   - Status badge: green for active/on_trial, amber for cancelled, red for past_due/expired
   - Plan details: billing period (Monthly/Yearly), currency and amount formatted with `Intl.NumberFormat`
   - Next renewal/expiry date: formatted with `Intl.DateTimeFormat` using the user's locale
   - For cancelled status: show "Access until [date]" instead of "Next renewal"

5. **Cancel button and confirmation dialog:**
   - "Cancel Subscription" button: `bg-red-500/20 border border-red-400/30 text-red-300 hover:bg-red-500/30 rounded-xl py-2.5 px-6`
   - Only show when status is 'active' or 'on_trial' (don't show for already cancelled, expired, etc.)
   - On click: set `showCancelDialog = true`
   - **Confirmation dialog** — modal overlay:
     - Glass card centered: `bg-slate-900/95 backdrop-blur-md border border-white/20 rounded-xl p-6 max-w-sm`
     - AlertTriangle icon in amber
     - Title: t('parentPortal.cancelConfirmTitle') = "Cancel Subscription?"
     - Body: t('parentPortal.cancelConfirmBody', { date: formattedEndDate }) = "You'll keep full access until {{date}}. After that, you'll return to the free plan."
     - Two buttons: "Keep Subscription" (secondary, closes dialog) and "Yes, Cancel" (red, triggers cancel)

6. **Cancel handler (`handleCancel`):**
   - Set `isCancelling = true`
   - Call `supabase.functions.invoke('cancel-subscription')` (no body needed — JWT provides student ID)
   - On success: extract `endsAt` from response. Set local optimistic state: `setOptimisticCancel({ cancelled: true, endsAt })`. Close dialog. Toast: "Subscription cancelled. You'll keep access until [date]."
   - On error: toast.error. Close dialog.
   - Finally: `isCancelling = false`
   - Invalidate queries: `queryClient.invalidateQueries({ queryKey: ['subscription-detail', user?.id] })` and `queryClient.invalidateQueries({ queryKey: ['subscription', user?.id] })` (the latter updates SubscriptionContext)

7. **Cancelled state display:**
   - When status is 'cancelled' (from DB or optimistic):
   - Show "Cancelled" amber badge
   - Show "Access until [end date]" prominently
   - Show "Re-subscribe" button → navigates to `/subscribe`
   - Hide the cancel button

8. **Back navigation:** Link back to `/settings` or `/` (dashboard) at the top.

9. **Layout:** Centered glass card.
   - `max-w-lg mx-auto p-4 sm:p-6`
   - Card: `bg-white/10 backdrop-blur-md border border-white/20 rounded-xl shadow-lg p-6`

10. **RTL support:** Standard `isRTL` pattern. Date formatting respects locale automatically via `Intl.DateTimeFormat`.
