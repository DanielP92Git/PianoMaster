Output: A deployed-ready Supabase Edge Function with pure business logic modules testable by Vitest, plus config.toml update to disable JWT verification for the webhook endpoint.
</objective>

    Create the directory `supabase/functions/lemon-squeezy-webhook/lib/`.

    **File 1: `lib/verifySignature.ts`** — Pure function, no Deno-specific APIs except `crypto.subtle` (global WebCrypto).

    ```typescript
    export async function verifySignature(
      rawBody: string,
      receivedHex: string,
      secret: string
    ): Promise<boolean>
    ```

    Implementation:
    - Guard: return `false` if `receivedHex` or `secret` is empty/falsy
    - Use `crypto.subtle.importKey('raw', ...)` with HMAC/SHA-256
    - Use `crypto.subtle.sign('HMAC', key, encodedBody)` to compute HMAC
    - Convert computed HMAC buffer to lowercase hex string
    - Length-check: if computed and received hex strings differ in length, return `false`
    - Use timing-safe comparison: import `timingSafeEqual` from `https://deno.land/std@0.224.0/crypto/timing_safe_equal.ts` (pinned version per 13-RESEARCH.md recommendation to avoid version ambiguity)
    - Encode both hex strings as Uint8Array via TextEncoder, compare with `timingSafeEqual`

    **File 2: `lib/extractPayload.ts`** — Pure function, zero imports.

    ```typescript
    export interface WebhookPayload {
      event_name: string;
      student_id: string | undefined;
      ls_subscription_id: string;
      ls_customer_id: string;
      ls_variant_id: string;
      status: string;
      parent_email: string | undefined;
      current_period_end: string | undefined;
    }

    export function extractPayload(body: unknown): WebhookPayload
    ```

    Implementation:
    - Cast body to `{ meta: any; data: any }`
    - Extract fields per the whitelist in 13-RESEARCH.md Pattern 5:
      - `event_name` from `meta.event_name`
      - `student_id` from `meta.custom_data?.student_id`
      - `ls_subscription_id` from `data.id` (as string)
      - `ls_customer_id` from `String(data.attributes?.customer_id ?? '')`
      - `ls_variant_id` from `String(data.attributes?.variant_id ?? '')`
      - `status` from `data.attributes?.status`
      - `parent_email` from `data.attributes?.user_email`
      - `current_period_end` from `data.attributes?.renews_at`
    - Return only these fields — ignore all other payload data (data sanitization per CONTEXT.md)

    **File 3: `lib/upsertSubscription.ts`** — Takes a Supabase client instance (injected, not created here).

    ```typescript
    import type { WebhookPayload } from './extractPayload.ts';

    export async function upsertSubscription(
      supabase: any,
      payload: WebhookPayload
    ): Promise<void>
    ```

    Implementation:
    - Call `supabase.from('parent_subscriptions').upsert({...}, { onConflict: 'ls_subscription_id' })`
    - Map fields:
      - `student_id: payload.student_id`
      - `ls_subscription_id: payload.ls_subscription_id`
      - `ls_customer_id: payload.ls_customer_id`
      - `ls_variant_id: payload.ls_variant_id`
      - `plan_id: null` (Phase 16 will map variant_id to plan_id)
      - `status: payload.status`
      - `current_period_end: payload.current_period_end || null`
      - `parent_email: payload.parent_email || null`
    - If `error` is returned from the upsert, throw the error (caller handles HTTP response)

    **File 4: `index.ts`** — Thin entry point, HTTP in/out only.

    ```typescript
    import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';
    import { verifySignature } from './lib/verifySignature.ts';
    import { extractPayload } from './lib/extractPayload.ts';
    import { upsertSubscription } from './lib/upsertSubscription.ts';
    ```

    Flow (follows 13-RESEARCH.md Pattern 1 — verify before parse):
    1. Reject non-POST with 405
    2. `const rawBody = await req.text()` — read raw body FIRST (critical: stream consumed once)
    3. Get `X-Signature` header and `LS_SIGNING_SECRET` env var
    4. Call `verifySignature(rawBody, signature, secret)` — return 400 if invalid
    5. `JSON.parse(rawBody)` — return 400 if invalid JSON
    6. Call `extractPayload(body)`
    7. Check event_name against handled set: `subscription_created`, `subscription_updated`, `subscription_cancelled`, `subscription_expired` — return 200 for unhandled events
    8. Guard: if `!payload.student_id`, log error (redacted email format: `j***@example.com`) and return 200 (stops LS retries per discretion decision in CONTEXT.md)
    9. Create Supabase client with `SUPABASE_URL` + `SUPABASE_SERVICE_ROLE_KEY` (auto-injected by Supabase, per 13-RESEARCH.md)
    10. Call `upsertSubscription(supabase, payload)` in try/catch — on error return 500 (LS retries); on success return 200
    11. Log successful processing with redacted email

    Add a `redactEmail` helper inline in index.ts:
    ```typescript
    function redactEmail(email: string): string {
      const at = email.indexOf('@');
      if (at < 0) return '***';
      return email[0] + '***@' + email.slice(at + 1);
    }
    ```

    **File 5: `supabase/config.toml`** — Add at the end (before `[analytics]` section or after `[edge_runtime]`):

    ```toml
    [functions.lemon-squeezy-webhook]
    ```

    This tells Supabase to skip JWT verification for this function (it does its own HMAC verification). Place it right after the `[edge_runtime]` section's closing lines (before `[analytics]`).

    **IMPORTANT:** Do NOT add CORS headers to this function — Lemon Squeezy sends server-to-server POSTs, not browser requests.
