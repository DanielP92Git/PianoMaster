  <action>
Create a single migration file that does the following, in order:

**1. subscription_plans table:**
```sql
CREATE TABLE subscription_plans (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  billing_period TEXT NOT NULL CHECK (billing_period IN ('monthly', 'yearly')),
  currency TEXT NOT NULL CHECK (currency IN ('ILS', 'USD')),
  amount_cents INTEGER NOT NULL CHECK (amount_cents > 0),
  lemon_squeezy_variant_id TEXT,
  is_active BOOLEAN DEFAULT true NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
- Enable RLS
- Add public SELECT policy: `USING (true)` -- no PII, read-only reference data

**2. parent_subscriptions table:**
```sql
CREATE TABLE parent_subscriptions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE,
  ls_subscription_id TEXT NOT NULL UNIQUE,
  ls_customer_id TEXT,
  ls_variant_id TEXT,
  plan_id TEXT REFERENCES subscription_plans(id),
  status TEXT NOT NULL CHECK (status IN ('on_trial','active','paused','past_due','unpaid','cancelled','expired')),
  current_period_end TIMESTAMPTZ,
  parent_email TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```
- Add indexes: `idx_parent_subscriptions_student` on `student_id`, `idx_parent_subscriptions_status` on `(status, current_period_end)`
- Enable RLS
- Add SELECT policy for students: `USING (student_id = (SELECT auth.uid()))` -- students can read their own subscription status
- NO INSERT/UPDATE/DELETE policies for authenticated role -- all writes go through service_role (webhook)

**3. is_free_node() function:**
- LANGUAGE sql, IMMUTABLE, SECURITY INVOKER, SET search_path = public
- Returns true if p_node_id is in the hardcoded array of 19 free node IDs matching `subscriptionConfig.js`:
  treble_1_1..treble_1_7, bass_1_1..bass_1_6, rhythm_1_1..rhythm_1_6
- Also returns true for NULL node_id (non-trail games are always free)

**4. has_active_subscription() function:**
- LANGUAGE sql, STABLE, SECURITY DEFINER, SET search_path = public
- Accepts p_student_id UUID
- Returns true if a row exists in parent_subscriptions WHERE student_id = p_student_id AND:
  - status = 'active', OR
  - status = 'cancelled' AND current_period_end > NOW() (preserve access until period ends), OR
  - status = 'past_due' AND current_period_end > NOW() - INTERVAL '3 days' (3-day grace period)

Use `mcp__supabase__apply_migration` to apply the migration.
