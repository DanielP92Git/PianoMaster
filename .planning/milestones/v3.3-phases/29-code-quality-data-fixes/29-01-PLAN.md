<task type="auto">
  <name>Task 2: Fix ArcadeRhythmGame score exceeding 100% (CODE-02)</name>
  <files>src/components/games/rhythm-games/ArcadeRhythmGame.jsx</files>
  <read_first>
    - src/components/games/rhythm-games/ArcadeRhythmGame.jsx (see scoredRef usage at ~line 182, score calculation at ~line 620-630, miss detection at ~line 458-466, tap scoring at ~line 700-706)
  </read_first>
  <action>
The bug: `scoredRef.current` is a Set of tile indices. It gets entries added in two places:
1. When a tile passes the hit zone untapped (line ~458-459) — both rest AND non-rest tiles are added.
2. When the user taps and matches a non-rest tile (line ~706).

At score-tally time (line ~623): `const hitCount = scoredRef.current.size` counts ALL entries including rests. `const nonRestCount = beatTimes.length` only counts non-rest beats (since `buildBeatTimes` skips rests). So `hitCount > nonRestCount` when rest tiles are in `scoredRef`, making score > 100%.

Fix:

1. At line ~623, replace:

   ```
   const hitCount = scoredRef.current.size;
   ```

   with:

   ```
   const hitCount = [...scoredRef.current].filter(
     (idx) => idx < tilesRef.current.length && !tilesRef.current[idx]?.isRest
   ).length;
   ```

2. Also add a `Math.min(100, ...)` cap on the final score as a safety net at line ~626-630:
   ```
   finishPattern(
     Math.min(
       100,
       Math.max(
         0,
         Math.round((hitCount / Math.max(nonRestCount, 1)) * 100)
       )
     )
   );
   ```

This ensures:

- Rest tiles in scoredRef do not inflate the hit count.
- Even with edge cases, score never exceeds 100%.
  </action>
