# Milestone v2.1: Forgot Password Recovery

**Status:** SHIPPED 2026-03-10
**Phases:** 01
**Total Plans:** 2

## Overview

Adds "Forgot password?" inline recovery flow to the login page and a dedicated /reset-password page, using Supabase built-in password reset with EN+HE i18n support and COPPA-appropriate security practices.

## Phases

### Phase 01: Add Forgot Password Recovery on Login

**Goal:** Add forgot password inline recovery flow to LoginForm and a dedicated /reset-password page
**Depends on:** None
**Plans:** 2 plans

Plans:
- [x] 01-01: Password reset API functions, TanStack Query hooks, and EN+HE translations (206ec97, ef3a351)
- [x] 01-02: LoginForm inline forgot password flow + ResetPasswordPage + route wiring (28818d7, 75ca47e, b3d29d5)

**Details:**
- Supabase resetPassword() and updatePassword() API functions with module-level siteUrl
- useResetPassword and useUpdatePassword TanStack Query hooks with retry:0 (no duplicate emails)
- Anti-enumeration: generic error messages only on reset request
- LoginForm three-state view (login / forgotPassword / resetSent) — inline, not modal
- 60-second cooldown timer prevents rapid re-sends
- ResetPasswordPage (239 lines) with glass card UI matching login aesthetic
- Dual password fields with independent Eye/EyeOff visibility toggles
- Supabase PASSWORD_RECOVERY event listener for PKCE session detection
- Smart expired-link detection: URL hash param check for immediate error display
- 10-second timeout fallback for legitimate reset links
- RTL-aware back arrow with rtl:rotate-180 Tailwind class
- Public /reset-password route wired outside ProtectedRoute in App.jsx
- Complete EN (23 keys) and HE (23 keys) translations under auth.forgotPassword.* and auth.resetPassword.*

## Milestone Summary

**Key Decisions:**
- Inline view switching in LoginForm rather than separate modal or page
- RTL-aware back arrow using rtl:rotate-180 for Hebrew layout correctness
- Smart session detection: sniff URL hash for auth params instead of always waiting for timeout
- 10-second timeout for legitimate reset links, immediate failure for missing URL params
- No retry on password reset mutations to prevent duplicate emails
- Generic error messages only on reset request to prevent email enumeration

**Issues Resolved:**
- No password recovery option existed — users had no way to regain access
- RTL back arrow pointed wrong direction in Hebrew mode (fixed post-checkpoint)
- Slow expired-link detection (3s → immediate for missing params, 10s for real links)

**Issues Deferred:**
- None
