<investigation>
## Root Cause

On iOS Safari, AudioContext is created in `suspended` state and requires a user gesture to resume.

**RhythmReadingGame** (BROKEN): Auto-start effect (lines 155-167) checks `ctx.state === 'suspended'` and silently returns with NO fallback:

```javascript
if (ctx && (ctx.state === "suspended" || ctx.state === "interrupted")) {
  return; // ← Just returns, no UI feedback, game stuck in SETUP
}
```

**RhythmDictationGame** (BROKEN): Auto-start effect (lines 353-365) does NOT check AudioContext state at all — it just proceeds. Audio may not play on iOS.

**MetronomeTrainer** (WORKING — reference): Has the complete pattern:

1. `const [needsGestureToStart, setNeedsGestureToStart] = useState(false);` (line 94)
2. Auto-start checks ctx.state → calls `setNeedsGestureToStart(true)` → returns (lines 175-177)
3. `handleGestureStart` callback: resumes context, sets settings, starts game (lines 795-813)
4. Renders `AudioInterruptedOverlay` when `needsGestureToStart` is true (lines 1359-1365)

Both RhythmReadingGame and RhythmDictationGame already import AudioInterruptedOverlay but only use it for `isInterrupted` (runtime audio interruption), not for the initial gesture gate.
</investigation>
