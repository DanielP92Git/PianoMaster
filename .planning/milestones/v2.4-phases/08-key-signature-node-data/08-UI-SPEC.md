## Phase Classification: Data-Authoring Only

Phase 08 creates four JavaScript unit data files. It introduces **zero new UI components**, **zero new screens**, and **zero new routes**. All visual rendering of the new trail nodes is handled by existing infrastructure from prior phases (TrailNodeModal, TrailNode, SightReadingGame, MemoryGame, VexFlowStaffDisplay).

The design contract for this phase is therefore a **data field contract** — the executor must produce node field values that the existing rendering pipeline will display correctly. The sections below specify the exact string values, field names, and structural requirements that drive the existing UI.

## Interaction Contract

No new interactions. The trail node tap-to-open and exercise-launch interactions are fully handled by existing TrailNodeModal and TrailNode components.

The only interaction-adjacent concern is the **boss multi-exercise flow**: the existing VictoryScreen "Next Exercise" mechanic handles the 3-exercise boss progression. Node data must correctly set `totalExercises: 3` in the nav state — this is driven by `exercises.length` in the node definition, which the executor controls by authoring exactly 3 exercise objects in the boss node.
