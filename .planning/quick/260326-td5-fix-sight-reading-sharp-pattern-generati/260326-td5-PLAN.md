<objective>
Fix sight-reading sharp practice nodes (Unit 4) to follow key signature logic instead of explicit accidentals.

**Problem**: Unit 4 sharp practice nodes (treble_4_4, treble_4_5, boss_treble_4, bass_4_4, bass_4_5, boss_bass_4) use SIGHT_READING exercises with explicit sharp+natural note pools but no key signature. This causes two issues:
1. F natural can appear alongside F# in patterns (musically incorrect when practicing sharps)
2. Every sharp note gets a manual accidental glyph even if the same sharp appeared earlier in the bar (no carry-through)

**Solution**: Add `keySignature: 'A'` to all SIGHT_READING exercise configs in Unit 4 nodes. A major has exactly F#, C#, G# -- the three sharps these nodes practice. This leverages the existing key signature pipeline:
- `filterNotesToKey` + `mapNoteToKey` in patternBuilder.js automatically maps F->F#, C->C#, G->G# and deduplicates
- VexFlow `Accidental.applyAccidentals()` handles accidental carry-through within bars
- The key signature glyph renders on the staff, making context clear

The note pools should switch to natural-only names (like Unit 6 does) since the key signature handles the sharp transformation. This is a data-only change -- the pattern generation and rendering infrastructure already supports key signatures fully.

Purpose: Make sharp practice nodes follow real music theory rules for a pedagogically correct experience.
Output: Updated trail node data files + tests verifying behavior.
</objective>
