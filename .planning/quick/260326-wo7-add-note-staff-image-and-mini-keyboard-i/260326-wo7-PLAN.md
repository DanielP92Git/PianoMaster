**MiniKeyboard.jsx** — A static, purely visual CSS/Tailwind piano keyboard component.

Props: `{ focusNotes, clef }`

Implementation:
- Render a single octave of piano keys (C through B + next C) using div elements. White keys are `div` rectangles, black keys are smaller `div` positioned absolutely on top.
- The octave to display is determined from the focusNotes pitches: parse the octave number from the first focusNote (e.g., 'F#4' -> octave 4, so show C4-C5; 'Bb3' -> octave 3, show C3-C4).
- Piano key layout for one octave (13 keys total): C, C#, D, D#, E, F, F#, G, G#, A, A#, B, C(next octave).
- White keys (C, D, E, F, G, A, B, nextC): `bg-white/90 border border-white/20 rounded-b-md` — approximately 28px wide, 80px tall.
- Black keys (C#, D#, F#, G#, A#): `bg-slate-900 border border-white/10 rounded-b-sm` — approximately 18px wide, 50px tall, positioned absolutely overlapping between white keys.
- Focus note keys get a bright highlight: use a category-aware highlight color. For treble: `bg-blue-400`, bass: `bg-purple-400`. Apply as a CSS class replacing the default white/black bg. Add a subtle glow: `shadow-[0_0_8px_rgba(color,0.5)]`.
- The whole keyboard container: `flex justify-center` with `dir="ltr"` attribute forced (critical for RTL Hebrew).
- Height constrained to about 80-90px total so it fits compactly in the modal.
- Each key should have a small label underneath (the note name, e.g., "C", "F#") visible only for focus notes — use `text-[9px] text-center mt-0.5 font-semibold` in the category accent color.
- No interactivity, no hover states, no audio — purely visual.

**NotePreview.jsx** — Wraps the staff SVG images and MiniKeyboard into a single preview section.

Props: `{ node }` (the full trail node object)

Implementation:
- Only render if `node.noteConfig?.focusNotes?.length > 0` (Discovery nodes). Return `null` otherwise.
- Use a `useEffect` + `useState` pattern to dynamically `import()` gameSettings.js and extract TREBLE_NOTES/BASS_NOTES. Store the loaded notes array in state. Show a small loading placeholder until loaded.
- Once loaded, for each pitch in `node.noteConfig.focusNotes`:
  - Find the matching note object from TREBLE_NOTES (if clef='treble') or BASS_NOTES (if clef='bass') by matching `note.pitch === focusPitch`.
  - Render the `ImageComponent` SVG if found. Size: `w-20 h-28` with `[&_svg]:fill-white [&_svg]:stroke-white` to ensure SVGs render visibly on dark background (test this — SVGs may already be styled).
- Layout: A `bg-white/5 border border-white/10 rounded-xl p-3 mb-5` card containing:
  - A translated heading: use i18n key `trail:modal.notePreview` ("On the Staff" / Hebrew equivalent).
  - The SVG images in a `flex justify-center gap-4` row.
  - A second heading: use i18n key `trail:modal.keyboardPreview` ("On the Piano" / Hebrew equivalent).
  - Below the second heading, render `<MiniKeyboard focusNotes={node.noteConfig.focusNotes} clef={node.noteConfig.clef} />`.
- Use `useTranslation` for the heading text.
