| xl | 32px (`--spacing-xl`) | Layout gaps between HUD and conveyor zone |
| 2xl | 48px (`--spacing-2xl`) | Major section breaks (HUD → conveyor → tap hint) |

- Note card: minimum height 200px on mobile to ensure staff is legible for 8-year-olds.

## Interaction Contract

### Conveyor animation (D-06 from CONTEXT.md)

- Card enters: `initial={{ x: '100%', opacity: 0 }}` → `animate={{ x: '0%', opacity: 1 }}`
- Card exits: `exit={{ x: '-100%', opacity: 0 }}`
- Transition: `useMotionTokens()` `soft` spring config (respects `prefers-reduced-motion`)
- `AnimatePresence mode="wait"` — exit completes before next card enters
- Card `key` is unique per card (index-based) to force framer-motion remount

### Feedback flash (D-08, D-09)

- Correct catch: `bg-green-400/40` overlay, `opacity: 0.8 → 0` over 300ms
- Wrong tap: `bg-red-400/40` overlay, `opacity: 0.8 → 0` over 300ms
- Missed target: amber banner `bg-amber-400/20 text-amber-300 text-sm font-bold` at card exit position, 200ms duration — NOT a full-screen flash

### Tap zone (D-07)

- Entire game area below HUD row is the tap surface: `onClick` and `onTouchStart` on the outer container div
- `touch-manipulation` class on tap zone div to prevent 300ms tap delay on iOS
- `cursor-pointer` on the tap zone for desktop

### Speed tiers (from RESEARCH.md session design)

| Cards | Interval | Visual label |
|-------|----------|-------------|
| 1–5 | 2000ms | Learning |
| 6–10 | 1500ms | Warming up |
| 11–15 | 1200ms | Challenge |
| 16–20 | 1000ms | Fast! |
| Bonus per correct catch | −100ms (floor 700ms) | — |

Speed indicator in HUD uses the tier label text above (i18n key: `noteSpeedCards.speedLabel.*`).

### Reduced motion

All framer-motion transitions use `useMotionTokens()`. When `prefers-reduced-motion: reduce` is active, the hook returns zero-duration tokens — cards snap rather than slide, flash overlays appear/disappear without animation. No special branching needed in the component.

## Animation Tokens Reference

Source: `src/utils/useMotionTokens.js` (confirmed in RESEARCH.md). The executor should import `useMotionTokens` and destructure the needed token:

| Token name | Usage |
|------------|-------|
| `soft` | Card enter/exit transition |
| `snappy` | Feedback flash overlay |
| `spring` | Combo number scale bounce on increment |

If `useMotionTokens` does not export `snappy`, use `{ duration: 0.3, ease: 'easeOut' }` directly with a `reducedMotion` guard.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Tap target size | Full screen below HUD — always exceeds 44px minimum |
| Screen reader | `aria-live="polite"` region for feedback state ("Caught it!" / "Not quite!") |
| Reduced motion | `useMotionTokens()` returns zero-duration configs — no extra code needed |
| High contrast | `useAccessibility()` — when `highContrast` is true, replace `bg-white/10` with `bg-white/30`, `text-white/70` with `text-white` |
| Focus state | No keyboard-focusable interactive elements during gameplay (tap-only) — pre-game "Let's Go!" button uses `focus:ring-2 focus:ring-white/50` |
| RTL (Hebrew) | Card slide direction reverses: `initial={{ x: '-100%' }}` → `exit={{ x: '100%' }}` when `dir="rtl"` |
