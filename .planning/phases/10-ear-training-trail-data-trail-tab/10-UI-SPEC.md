## Node Data Contract

### Node Shape

All 13 nodes must conform to the existing trail node structure. Reference: `trebleUnit1Redesigned.js`.

Required fields per node:

| Field | Type | Notes |
|-------|------|-------|
| `id` | string | Pattern: `ear_{unit}_{orderInUnit}` (e.g. `ear_1_1`) |
| `name` | string | Child-friendly, max 3 words |
| `description` | string | One sentence, max 8 words |
| `category` | string | `'ear_training'` (NODE_CATEGORIES.EAR_TRAINING) |
| `unit` | number | 1 or 2 |
| `unitName` | string | Unit 1: "Sound Direction" / Unit 2: "Interval Explorer" |
| `order` | number | Global order in EAR_TRAINING category (1–13) |
| `orderInUnit` | number | Position within unit |
| `prerequisites` | array | Linear chain within unit; boss requires last non-boss node |
| `nodeType` | string | NODE_TYPES value |
| `exercises` | array | Exercise config objects |
| `xpReward` | number | 40–50 for regular nodes, 100 for boss nodes |
| `isBoss` | boolean | true for boss_ear_1 and boss_ear_2 |

### Unit Structure

| Unit | Name | Nodes | Game Type | Free/Premium |
|------|------|-------|-----------|--------------|
| 1 | Sound Direction | 6 nodes + boss_ear_1 (7 total) | PITCH_COMPARISON | All free |
| 2 | Interval Explorer | 6 nodes + boss_ear_2 (6 regular + 1 boss = 7 total) | INTERVAL_ID + mixed | All premium |

Wait — total: Unit 1 has 6 regular + 1 boss = 7. Unit 2 has 6 regular + 1 boss = 7. Grand total = 14. Per D-05, "13 nodes total" with "6 pitch comparison nodes" and "7 interval nodes (including some mixed review)" suggests boss nodes are included in the count. The executor should target 13 nodes total: Unit 1 = 6 (including boss_ear_1), Unit 2 = 7 (including boss_ear_2), per D-05.

### Exercise Config Shape

Pitch comparison exercises:

```javascript
{
  type: EXERCISE_TYPES.PITCH_COMPARISON,
  config: {
    notePool: ['C3', 'D3', ...],   // notes available for random selection
    questionCount: 10,
    ascendingRatio: 0.6,           // default; override per node for variety
    intervalRange: { min: N, max: M }  // semitone distances constraining pairs
  }
}
```

Interval identification exercises:

```javascript
{
  type: EXERCISE_TYPES.INTERVAL_ID,
  config: {
    notePool: ['C4', 'D4', ...],
    questionCount: 10,
    allowedCategories: ['step'],   // start with steps only (D-04)
    ascendingRatio: 0.8            // ascending first per INTV-03
  }
}
```

## Interaction Contract

### Trail Tab Switching

The Ear Training tab appears as the 4th tab in the TrailMap tab bar. Tab activation already handled by `searchParams.get('path')` + `setSearchParams`. No new interaction logic needed.

Active state: `bg-gradient-to-br from-cyan-400 to-teal-500 border-cyan-300 shadow-[0_0_15px_rgba(34,211,238,0.4)]`

Inactive state: `bg-white/10 border-white/20 text-white/70`

### Node Tap

Tap a node → opens `TrailNodeModal`. Behavior identical to existing tabs. No new routing required.

### Progressive Unlock Visual

Each node in Unit 1 has a prerequisite chain. Locked nodes display gray state. As the child completes each node, the next unlocks with the cyan gradient. This animation/state transition is handled by existing `getNodeStateConfig()` + React re-render on progress fetch.
