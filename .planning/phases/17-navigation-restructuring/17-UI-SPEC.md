## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                          |
| ----- | ----- | -------------------------------------------------------------- |
| xs    | 4px   | Icon gaps, inline padding between icon and badge               |
| sm    | 8px   | Compact nav item internal padding (`px-3 py-2` on mobile tabs) |
| md    | 16px  | Default element spacing, section padding                       |
| lg    | 24px  | Section padding in sidebar (`px-4 py-3` nav items)             |
| xl    | 32px  | Layout gaps                                                    |
| 2xl   | 48px  | Major section breaks                                           |
| 3xl   | 64px  | Page-level spacing                                             |

Exceptions:

- Mobile tab bar height: `calc(4rem + env(safe-area-inset-bottom, 0px))` — fixed by `BottomNavigation` component (source: `Navigation.jsx` line 273)
- Touch target minimum: 44px (`min-h-touch` / `min-w-touch` tokens in tailwind.config.js) — applies to all nav tab items
- Desktop sidebar width: `19rem` (304px) — fixed by existing `AppLayout` `xl:pl-[19rem]` offset
- Jump-to-Top FAB vertical offset: `bottom-24` on mobile (to clear tab bar + safe area), `xl:bottom-6` on desktop

Source: `CLAUDE.md` design system section + `tailwind.config.js` + `AppLayout.jsx`.

Inactive nav item hover: `hover:bg-white/5 hover:text-white` (mobile tabs) / `lg:hover:bg-white/10 lg:hover:text-white` (sidebar).

Trail page background class is owned by `AppLayout` via `backgroundClass` (line 56-58), NOT by `TrailMapPage` directly. After route swap, `isTrailPage` uses `location.pathname === "/"`.

## Layout Contracts

### TrailMapPage Layout Conversion (D-07)

| Property                 | Before                                           | After                                             | Reason                                           |
| ------------------------ | ------------------------------------------------ | ------------------------------------------------- | ------------------------------------------------ |
| Root div position        | `fixed inset-0 overflow-y-auto`                  | `min-h-full` (no fixed/inset/overflow-y)          | Trail is a normal page within AppLayout shell    |
| Scroll container         | Internal `scrollRef` div                         | Browser window (natural document scroll)          | AppLayout controls scroll context                |
| Jump-to-Top FAB vertical | `fixed bottom-6 right-6`                         | `fixed bottom-24 right-6 xl:bottom-6 right-6`     | Must clear mobile tab bar (~4rem) + safe area    |
| Jump-to-Top FAB action   | `scrollRef.current?.scrollTo(...)`               | `window.scrollTo({ top: 0, behavior: "smooth" })` | scrollRef removed; window is scroll container    |
| BackButton               | `<BackButton to="/" name="Dashboard" />` present | Removed entirely                                  | Trail IS home; back button makes no sense (D-11) |

### AppLayout Route Detection (D-07, D-08)

| Boolean       | Before                           | After                                |
| ------------- | -------------------------------- | ------------------------------------ |
| `isDashboard` | `location.pathname === "/"`      | `location.pathname === "/dashboard"` |
| `isTrailPage` | `location.pathname === "/trail"` | `location.pathname === "/"`          |

After this change, trail page behavior from AppLayout:

- Header: hidden (same as dashboard — `!isGameRoute && !isDashboard && !isTrailPage`)
- Sidebar: visible (remove `!isTrailPage` guard from sidebar block)
- Mobile tabs: visible (remove `!isTrailPage` guard from `<MobileTabsNav />` render)
- Main padding: `xl:pl-[19rem]` sidebar offset applied; `pt-0` (same as dashboard, no header); `pb-20 xl:pb-0` tab clearance applied
- Background: `bg-[#1a1040]` via `backgroundClass` (no change to logic, only path from `/trail` to `/`)

## Accessibility Contracts

- All nav items retain accessible labels via `aria-label` on NavLink (BottomNavigation) and visible text in Sidebar.
- Screen reader label for Trail mobile tab: rendered via `<span className="sr-only">{item.label}</span>` (labels hidden visually but available to SR — existing pattern from Navigation.jsx line 309).
- RTL support: `isRTL` check in Sidebar applies `direction-rtl text-right` class to nav links. Trail item participates in this automatically.
- Reduced motion: No animations introduced by this phase. Existing `transition-all duration-200` on nav items is covered by the existing `AccessibilityContext` reduced-motion pattern.
- Focus management: Sidebar's existing focus trap (Sidebar.jsx line 70-78) requires no changes.
