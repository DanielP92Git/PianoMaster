## Spacing Scale

Declared values from `src/index.css` CSS custom properties:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, divider spacing |
| sm | 8px | Compact element spacing, icon-to-label gaps |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, card-to-card gaps |
| xl | 32px | Layout gaps between portal sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level padding bottom (scroll clearance) |

Exceptions:
- Touch targets (buttons, toggles): minimum 44px height (`min-h-touch` Tailwind token)
- Math gate overlay input field: minimum 48px height for easy touch input (`min-h-touch-lg`)
- Quick stats card grid gap: 12px (between sibling stat cards, tighter than md)

## Typography

All type is rendered on the AppLayout dark purple gradient background. Use white-spectrum text only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.6 | Subscription detail rows, setting descriptions, dates |
| Label | 16px (text-base) | 600 (semibold) | 1.7 | Card section labels, toggle labels, stat value labels |
| Heading | 20px (text-xl) | 700 (bold) | 1.8 | Section headings (Quick Stats, Practice Heatmap, etc.) |
| Display | 28px (text-3xl approx) | 900 (black) | 1.9 | Math problem expression in gate overlay (`text-4xl font-black` per existing `ParentGateMath`) |

**Source:** `tailwind.config.js` fontSize scale + existing `ParentPortalPage.jsx` and `ParentGateMath.jsx` patterns.

Note: The math problem display uses `text-4xl font-black` (36px/900) — this intentionally breaks the 3-size limit because it is a single decorative element in the gate overlay, not body content.

## Color

The portal lives on the AppLayout background: `bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | indigo-900/purple-900 gradient | App background (AppLayout, not authored by this phase) |
| Secondary (30%) | `bg-white/10 border-white/20` (glass cards) | All portal section cards, gate overlay card, stat cards |
| Accent (10%) | `#6366f1` indigo-400/indigo-300 | Submit button in gate, numerical stat values, price amounts, active focus rings |
| Destructive | `#ef4444` red-500 / `bg-red-500/20 border-red-400/30 text-red-300` | Cancel subscription button and cancel confirmation dialog only |

Accent reserved for:
1. Gate submit button background (`bg-indigo-600 hover:bg-indigo-700`)
2. Numerical stat values in Quick Stats cards (`text-indigo-300`)
3. Price/billing amounts in subscription card (`text-indigo-300`)
4. Input focus ring in gate overlay (`focus:border-indigo-400/70 focus:ring-indigo-400/40`)

Secondary semantic colors (non-destructive):
- `text-amber-300` / `bg-amber-400/10 border-amber-400/30` — cancelled subscription warning state and re-subscribe CTA
- `bg-amber-400 text-amber-900` — "Unlock Access" / re-subscribe primary CTA button (amber gradient, not indigo accent)
- `text-green-300 bg-green-400/20 border-green-400/30` — active subscription status badge
- `text-white/70` — secondary body text throughout portal
- `text-white/60` — tertiary label text (field labels in plan details grid)
- `text-white/50` — muted text (gate cancel link, hint text)

**Source:** `ParentPortalPage.jsx` StatusBadge + card patterns, `ParentGateMath.jsx` overlay.

## Interaction States

### Gate overlay
| State | Behavior |
|-------|----------|
| Initial | Gate shown, input focused, submit disabled |
| Typing | Submit enabled when input non-empty |
| Wrong answer | Input clears, new problem generated, red error message below input, attempt counter increments |
| 3+ wrong attempts | Hint text appears below error in muted white/50 |
| Correct answer | Gate closes with fadeIn on portal content (250ms ease-in-out) |
| Cancel (X or cancel link) | Navigates back (BackButton destination: history.back() or `/`) |

### Cancel subscription dialog
| State | Behavior |
|-------|----------|
| Trigger | "Cancel Subscription" button (red-500/20 style) |
| Dialog | Fixed overlay z-50, `bg-slate-900/95 backdrop-blur-md border-white/20` |
| Confirm | Red-500 full-width button, spinner during API call |
| Keep | White/10 ghost button |
| Success | Toast success, optimistic status update to 'cancelled' |
| Error | Toast error, dialog closes |

### Quick Stats cards
| State | Behavior |
|-------|----------|
| Loading | `animate-pulse bg-white/10 rounded-xl h-20` skeleton per card |
| Data | Static display, no hover interaction, no click target |

## RTL Support

All components must pass `isRTL` derived from `i18n.dir() === 'rtl'`:
- Flex rows use `flex-row-reverse` when `isRTL`
- Text alignment: `text-right` when `isRTL`
- `dir={isRTL ? 'rtl' : 'ltr'}` on the outer container
- `ParentGateMath` already handles RTL natively — pass `isRTL` prop

---

## Accessibility

- Gate input: `autoFocus` on mount, `inputMode="numeric"`, `pattern="[0-9]*"`
- Cancel button in gate: `aria-label` (existing pattern in `ParentGateMath`)
- Dialogs: `role="dialog"`, `aria-modal="true"`, focus trap on open
- Reduced motion: gate open transition (`animate-fadeIn`) must check `prefers-reduced-motion`; if reduced motion, skip animation (render immediately)
- High contrast: all glass borders (`border-white/20`) must be visible; existing high-contrast CSS class in DESIGN_SYSTEM.md handles globally
