    **`binaryToVexDurations(binary, durations, timeSignature = '4/4')` — exported function:**

    Algorithm (greedy longest-fit, prefer sustain per D-06, D-07):
    1. Build sorted candidates from `durations` — map each duration to its slot size, sort descending by slot size. Include only note durations (not rests) in the primary candidates.
    2. Build rest candidates — for each duration slot size, find the rest with matching or smaller slot size. Sort descending.
    3. Walk through the binary array at each onset position (where `binary[i] === 1`):
       a. Compute `gap` = distance to next onset, or to end of array if last onset
       b. Find longest candidate whose slot size <= gap. This is the note duration.
       c. Advance by that candidate's slot size.
       d. If remaining gap exists (candidate didn't fill it), fill with the largest fitting rest, then continue filling until gap is consumed. Use rest candidates sorted descending.
    4. Handle leading rest: if `binary[0] === 0`, fill from position 0 to first onset with rests.
    5. Handle trailing space: after last onset's note+rests, if position < binary.length, fill remaining with rests.
    6. Return the string array of VexFlow duration codes.

    **`resolveByTags(tags, durations, options = {})` — exported function per D-08, D-10:**
    1. Filter RHYTHM_PATTERNS where pattern.tags contains ALL requested tags (AND logic).
    2. If `options.timeSignature` provided, further filter by `pattern.timeSignature === options.timeSignature`.
    3. If no matches, return `null`.
    4. Select one randomly: `matching[Math.floor(Math.random() * matching.length)]`.
    5. Render: `const vexDurations = binaryToVexDurations(selected.pattern, durations, selected.timeSignature)`.
    6. Return: `{ patternId: selected.id, binary: selected.pattern, timeSignature: selected.timeSignature, vexDurations, tags: selected.tags }`.

    **`resolveByIds(ids, durations)` — exported function per D-10:**
    1. Find pattern where `RHYTHM_PATTERNS.find(p => ids.includes(p.id))`.
    2. If not found, return `null`.
    3. Render and return same shape as resolveByTags.

    **Edge cases:**
    - When only one duration is available (e.g., `['q']`), gaps longer than 4 slots must be filled with 'qr' rests.
    - When binary pattern starts with 0 (rest at beginning), fill with appropriate rests before first onset.
    - Dotted notes: `hd=12`, `qd=6` — these are valid candidates when in the durations set.
