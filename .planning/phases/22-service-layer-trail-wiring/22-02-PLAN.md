
    **Props:**
    - `question.rhythmConfig.tempo` — BPM (default 65 per D-04)
    - `question.rhythmConfig.timeSignature` — time sig (default "4/4")
    - `isLandscape` — layout flag
    - `onComplete(onTimeTaps, totalExpectedTaps)` — callback matching handleRhythmTapComplete signature
    - `disabled` — disables interaction

    **Design per D-03:**
    - Glass card background: `bg-white/10 backdrop-blur-md border border-white/20 rounded-xl`
    - Large centered pulsing circle that scales/glows on each metronome beat
    - NO staff lines, NO notation, NO VexFlow — purely visual pulsing circle
    - Instruction text: "Tap with the beat!" (i18n key: `game.pulse.instruction`)
    - Beat counter showing current beat number (e.g., "Beat 4 of 16")

    **Session per D-04:**
    - 4 bars of 4/4 time = 16 total beats at 65 BPM (~15 seconds)
    - Calculate total beats from timeSignature: beatsPerMeasure * 4 bars
    - Score = percentage of beats tapped within timing threshold

    **Internal state machine (simplified from RhythmTapQuestion):**
    ```
    WAITING → COUNT_IN (4 beats) → PLAYING (16 beats, user taps) → EVALUATING → DONE
    ```

    - WAITING: Show instruction, pulse circle idle
    - COUNT_IN: 4 preparatory beats with metronome clicks (same as RhythmTapQuestion pattern). Circle pulses on each count-in beat. Show "1... 2... 3... 4..." or "Ready... Set... Go!"
    - PLAYING: Metronome continues clicking. Circle pulses on each beat. User taps anywhere on the screen. Record tap timestamps relative to beat positions.
    - EVALUATING: Compare each user tap to nearest expected beat time. Count taps within timing threshold (use same calculateTimingThresholds from RhythmTapQuestion — base PERFECT=50ms at 120BPM, scaled by tempo).
    - DONE: Call `onComplete(onTimeTaps, totalExpectedTaps)` where `totalExpectedTaps = 16`.

    **Audio:** Use `useAudioEngine(tempo)` from `src/hooks/useAudioEngine.js` and `useAudioContext()` from AudioContextProvider, same pattern as RhythmTapQuestion. Schedule metronome clicks for count-in + playing phases.

    **Pulsing circle animation:**
    - Use CSS animation keyed to beat duration (not setInterval per anti-pattern in RESEARCH.md)
    - Circle: `w-32 h-32 rounded-full bg-gradient-to-br from-blue-400 to-purple-500`
    - Pulse: CSS `@keyframes pulse-beat` that scales to 1.15x and adds box-shadow glow, duration = beat interval (60/tempo seconds)
    - Use `animation-play-state: paused/running` to sync with count-in start
    - Respect reduced motion: if `reducedMotion`, use opacity change instead of scale

    **Tap area:** Full-screen tap target (same as TapArea from components.jsx if applicable, or a large div with onClick/onTouchStart). Visual feedback on tap: brief white flash/ripple.
