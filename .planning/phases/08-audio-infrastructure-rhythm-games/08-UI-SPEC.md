## Spacing Scale

Standard 8-point scale inherited from project design system (`docs/DESIGN_SYSTEM.md`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding |
| sm | 8px | Compact element spacing, beat circle gaps (`gap-2`) |
| md | 16px | Default card internal padding, button padding |
| lg | 24px | Section padding, game area vertical spacing |
| xl | 32px | Layout section gaps |
| 2xl | 48px | Major section breaks between game zones |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Touch targets (tap area, choice cards, replay button): minimum 44px height (`min-h-touch` token = 44px, defined in tailwind.config.js)
- Cursor overlay: 0px spacing — `position: absolute` overlay on VexFlow SVG container, no box model spacing applies
- Beat circles (MetronomeDisplay pattern): 36px (accented h-9/w-9) and 48-56px (strong beat h-12/w-12) — inherited from MetronomeDisplay — not a new value

Source: `tailwind.config.js` spacing tokens, `MetronomeDisplay.jsx` measured values.

## Typography

Game components use Nunito (`font-rounded`) throughout. Hebrew RTL layout uses Heebo (`font-hebrew`).

Strictly 2 weights: **regular (400)** for body/label/heading, **bold (700)** for display feedback text and section headers.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (`text-base`) | 400 (regular) | 1.7 | Instructions, question text, score labels |
| Label | 14px (`text-sm`) | 400 (regular) | 1.6 | Beat numbers, notation card sub-labels, i18n strings |
| Heading | 20px (`text-xl`) | 700 (bold) | 1.8 | Game title bar, section headers ("Listen", "Choose") |
| Display | 30px (`text-3xl`) | 700 (bold) | 1.9 | Countdown numbers (3, 2, 1, GO!), PERFECT/GOOD/MISS floating text |

Source: `tailwind.config.js` fontSize tokens, existing `TapArea.jsx` (`text-5xl font-bold` for feedback — matched and normalized to `text-3xl` per D-04 "floats up and fades"), `MetronomeDisplay.jsx`.

## Color

Game screen background is the project-wide purple gradient. All game UI uses the glassmorphism pattern on top of it.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900` | Full-screen game background (identical to MetronomeTrainer, SightReadingGame) |
| Secondary (30%) | `bg-white/10 backdrop-blur-md border border-white/20` | Game card containers, notation display area, choice card backgrounds |
| Accent (10%) | `bg-indigo-500` / `border-indigo-400` | Replay button, active beat circle highlight, cursor glow color |
| Destructive | `#ef4444` (`bg-red-500` / `text-red-400`) | MISS feedback text, wrong-answer card flash only |

Accent reserved for: replay button fill, scrolling cursor line color, active-state beat circle background, correct-answer card glow border.

**Feedback color palette** (game-state semantic, not design accent):
- PERFECT: `text-green-400` / `border-green-400` (correct tap within tight window; correct dictation card glow)
- GOOD: `text-yellow-400` (tap within generous window)
- MISS: `text-red-400` / border flash `border-red-500` (tap too late/early; wrong dictation card flash)

These map directly to D-04 (CONTEXT.md) and the existing `TapArea.jsx` color convention.

Source: `AppLayout.jsx` (`from-indigo-900 via-purple-900 to-violet-900`), `MetronomeDisplay.jsx`, `TapArea.jsx`, CLAUDE.md Glass Card Pattern.

### Cursor overlay (RhythmReadingGame)

- `position: absolute` div inside VexFlow container (which uses `position: relative`)
- Width: 2px
- Height: 100% of staff container
- Color: `bg-indigo-400` with `opacity-80`
- Box shadow: `shadow-[0_0_8px_rgba(129,140,248,0.8)]` — soft indigo glow (matches project `dropShadow.glow-cyan` convention, adapted for indigo)
- Animation: `left` CSS property updated via `requestAnimationFrame` (not CSS transition — D-01 requires sync to `audioContext.currentTime`)

### Choice card states (RhythmDictationGame)

| State | Class |
|-------|-------|
| Default | `bg-white/10 border border-white/20 rounded-xl` |
| Hover (pre-answer) | `hover:bg-white/20 hover:border-white/40` |
| Correct (revealed) | `bg-green-500/20 border-2 border-green-400 shadow-[0_0_12px_rgba(74,222,128,0.4)]` |
| Wrong selected (flash) | `bg-red-500/20 border-2 border-red-400` — then dims to `opacity-40` after 300ms |
| Dimmed (other wrong cards) | `opacity-40 pointer-events-none` |

D-09 (CONTEXT.md) is the source of truth for feedback sequence timing.

### Floating feedback text (RhythmReadingGame, D-04)

- Position: `position: absolute`, centered horizontally above tap point
- Initial: `translateY(0) opacity-1`
- Animation: `translateY(-40px) opacity-0` over 800ms, `ease-out`, triggered on each tap
- Sizes: PERFECT = `text-3xl font-bold`, GOOD = `text-3xl font-bold`, MISS = `text-3xl font-bold`
- Colors: PERFECT = `text-green-400`, GOOD = `text-yellow-400`, MISS = `text-red-400`
- Note color change: VexFlow note SVG fill updated via `data-note-index` attribute lookup — green/yellow/red per accuracy

## Animation Inventory

All animations respect `prefers-reduced-motion`. When `reducedMotion` is true (from `AccessibilityContext`):
- Floating feedback text: skip translateY, only fade opacity
- Cursor: still moves (required for gameplay), but no glow shadow
- Countdown pulse: skip `animate-pulse`, show static number

| Animation | Duration | Easing | Implementation |
|-----------|----------|--------|----------------|
| Floating feedback float-up | 800ms | ease-out | Inline style `transform + opacity`, JS-driven |
| Count-in number pulse | 200ms | ease-in-out | `animate-pulse` Tailwind class |
| Choice card correct glow | 300ms | ease-out | Tailwind `transition-all duration-300` on border + shadow |
| Choice card wrong flash | 300ms flash then 200ms dim | — | Two-step state: `flashing` → `dimmed` |
| Cursor sweep | Continuous | linear | `requestAnimationFrame` updates `left` style property |
| Tap area active:scale | 80ms | ease-out | `active:scale-95 transition-transform duration-75` |

## RTL Layout Notes

Both games must render correctly in Hebrew (`dir="rtl"`). Specific rules:

- Beat circle row (`MetronomeDisplay`): Force `dir="ltr"` — musical beat order (1, 2, 3, 4) is not RTL (existing component already has `dir="ltr"`)
- VexFlow staff: Force `dir="ltr"` — music notation is always left-to-right
- Score / progress bar: `dir="ltr"` inner container
- Choice card stack: natural vertical flow, no dir override needed
- Replay button icon (`Volume2`): does not flip in RTL — use `style={{ transform: 'none' }}` if needed
- Floating feedback text: centered above tap point regardless of dir

## Accessibility

Minimum requirements matching existing game components:

- Tap area `<button>` with `aria-label="Tap"` (i18n key)
- Replay button `aria-label="Play rhythm again"` (i18n key)
- Choice cards: `role="button"` or `<button>` with `aria-label="Choice {{n}}: {{duration description}}"`
- Feedback text: `aria-live="polite"` region for PERFECT/GOOD/MISS announcements
- Cursor: `aria-hidden="true"` — decorative, non-interactive
- High contrast mode (`AccessibilityContext`): border-based differentiation (borders increase from `border-white/20` to `border-white/60`)
