    Build the complete RhythmDictationGame component following MetronomeTrainer as the structural template. This is the aural recognition game per RDICT-01 through RDICT-06.

    **GAME_PHASES FSM:**
    ```javascript
    const GAME_PHASES = {
      SETUP: 'setup',               // Config from trail or defaults
      LISTENING: 'listening',        // Auto-play pattern audio (RDICT-01, D-08)
      CHOOSING: 'choosing',          // 3 cards visible, waiting for selection
      FEEDBACK: 'feedback',          // Show correct/wrong, replay correct if wrong (D-09)
      SESSION_COMPLETE: 'session-complete',  // VictoryScreen
    };
    ```

    **Component structure (follow MetronomeTrainer exactly):**
    1. Trail state from `location.state`: `nodeId, nodeConfig, exerciseIndex, totalExercises, exerciseType`
    2. `useRotatePrompt()` (landscape optional for this game per UI-SPEC: "portrait-primary, landscape-compatible")
       — Do NOT call `useLandscapeLock()` for this game. It works in portrait.
    3. `useSessionTimeout()` — `pauseTimer()` during LISTENING/CHOOSING, `resumeTimer()` during SETUP/FEEDBACK
    4. `useAudioContext()` — `audioContextRef, isInterrupted, handleTapToResume`
    5. `usePianoSampler()` — `playNote`
    6. `useSounds()` — correct/wrong sound effects
    7. `hasAutoStartedRef = useRef(false)` — auto-start guard
    8. `useTranslation('common')` for i18n

    **State variables:**
    ```javascript
    const [gamePhase, setGamePhase] = useState(GAME_PHASES.SETUP);
    const [currentQuestion, setCurrentQuestion] = useState(0);
    const [totalQuestions] = useState(10);
    const [correctBeats, setCorrectBeats] = useState(null);   // [{durationUnits, isRest}]
    const [choices, setChoices] = useState([]);                // [beats, beats, beats] — shuffled
    const [correctIndex, setCorrectIndex] = useState(-1);      // Which choice is correct
    const [selectedIndex, setSelectedIndex] = useState(null);   // User's selection
    const [cardStates, setCardStates] = useState(['default', 'default', 'default']);
    const [isPlaying, setIsPlaying] = useState(false);          // Audio playing flag (D-08)
    const [questionScores, setQuestionScores] = useState([]);   // 1 (correct) or 0 per question
    const isPlayingRef = useRef(false);                         // Prevent double-start
    ```

    **Question generation flow:**
    1. Call `getPattern(timeSignature, difficulty)` to get correct pattern
    2. Convert: `correctBeats = binaryPatternToBeats(result.pattern)`
    3. Generate 2 distractors: `distractors = generateDistractors(correctBeats, 2)` (RDICT-04)
    4. Shuffle choices: `[correctBeats, ...distractors]` shuffled randomly, track `correctIndex`
    5. Set state and transition to LISTENING

    **LISTENING phase (D-08 auto-play + RDICT-01):**
    1. Auto-play pattern via `schedulePatternPlayback(correctBeats, tempo, audioContextRef.current, playNote)`
    2. Set `isPlaying = true`, show "Listen..." status text (`t('games.rhythmDictation.listening')`)
    3. After pattern duration completes, set `isPlaying = false`
    4. Transition to CHOOSING phase
    5. Use `setTimeout` based on `totalDuration` returned from `schedulePatternPlayback` + 300ms buffer

    **Replay button (RDICT-02, D-08):**
    - Visible during CHOOSING and LISTENING phases
    - Label: `t('games.rhythmDictation.playAgain')` with `Volume2` icon from lucide-react (20px)
    - When `isPlaying`: `opacity-50 cursor-not-allowed` — disabled
    - When available: `bg-indigo-500 hover:bg-indigo-400 text-white rounded-xl px-4 py-2`
    - On click: if `!isPlayingRef.current`, call `schedulePatternPlayback` again
    - `aria-label={t('games.rhythmDictation.playAgain')}`

    **CHOOSING phase (RDICT-03):**
    - Render 3 `DictationChoiceCard` components in vertical stack (D-06, D-07)
    - Gap: `gap-3` (12px per spacing scale)
    - Each card receives its beats array, timeSignature, cardIndex, state from cardStates, onSelect handler

    **Selection + FEEDBACK (D-09, RDICT-05):**
    On card selection:
    1. Set `selectedIndex`, determine if correct
    2. Play correct or wrong sound via `useSounds()`
    3. **If correct:**
       - Set card state: `correctIndex → 'correct'`, others → 'dimmed'
       - Show `t('games.rhythmDictation.correct')` text
       - Score: push 1 to questionScores
       - After 1.5s: advance to next question or SESSION_COMPLETE
    4. **If wrong:**
       - Set card state: `selectedIndex → 'wrong'`
       - After 300ms: set `selectedIndex → 'dimmed'`, `correctIndex → 'correct'`, others → 'dimmed'
       - Auto-replay correct pattern via `schedulePatternPlayback(correctBeats, ...)` (RDICT-05)
       - Show `t('games.rhythmDictation.wrong')` text
       - Score: push 0 to questionScores
       - After 2s (from initial selection): advance to next question or SESSION_COMPLETE
    5. During feedback: all cards `disabled={true}`

    **SESSION_COMPLETE:**
    - Total score: `sum(questionScores)` correct out of `totalQuestions`
    - `totalPossibleScore = totalQuestions` (each question worth 1 point)
    - Actually for VictoryScreen star rating calculation, use percentage: `score = correctCount`, `totalPossibleScore = totalQuestions`
    - Render VictoryScreen with all trail props per MetronomeTrainer pattern
    - `handleNextExercise` follows MetronomeTrainer switch pattern for exercise type routing
