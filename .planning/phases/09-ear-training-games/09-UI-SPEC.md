## Spacing Scale

Standard 8-point scale inherited from project design system. Identical to Phase 8.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label padding, note name badge padding |
| sm | 8px | Compact element spacing, gap between answer buttons |
| md | 16px | Default card internal padding, button horizontal padding |
| lg | 24px | Section padding, vertical gap between game zones |
| xl | 32px | Layout section gaps |
| 2xl | 48px | Major section breaks (keyboard reveal area separation) |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Answer buttons (HIGHER/LOWER, Step/Skip/Leap): minimum 48px height (`min-h-[48px]` or `min-h-touch-lg`) — D-04 (CONTEXT.md) requires 48px+ touch targets
- Piano keyboard SVG keys: black keys 14px wide × 72px tall; white keys 28px wide × 120px tall — dictated by 1-octave legibility constraint (D-01), not spacing scale
- Direction arrow SVG: 24px icon size (matches lucide-react default)

Source: `tailwind.config.js` `minHeight.touch-lg = 48px`, CONTEXT.md D-01, D-04, Phase 8 UI-SPEC exceptions.

## Typography

Game components use Nunito (`font-rounded`) throughout. Hebrew RTL uses Heebo (`font-hebrew`). Identical font stack to Phase 8.

Strictly 2 weights: **regular (400)** for body/label, **bold (700)** for display feedback and headings.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (`text-base`) | 400 (regular) | 1.7 | Instructions, replay prompt, note name labels below keyboard keys |
| Label | 14px (`text-sm`) | 400 (regular) | 1.6 | Interval sub-labels ("next door", "jump one", "far apart"), progress counter, "Jumped over D4" sublabel |
| Heading | 20px (`text-xl`) | 700 (bold) | 1.8 | Game title bar, section headers ("Listen", "Which is higher?"), interval reveal label ("SKIP — C4 to E4") |
| Display | 30px (`text-3xl`) | 700 (bold) | 1.9 | "HIGHER ▲" / "LOWER ▼" directional reveal text, "Correct!" / "Try again!" feedback text |

Source: Phase 8 UI-SPEC (identical role/size/weight table), `tailwind.config.js` fontSize tokens, CONTEXT.md D-05, D-08.

## Color

Game screen background is the project-wide purple gradient. All game UI uses the glassmorphism pattern on top of it. Identical base palette to Phase 8.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900` | Full-screen game background — identical to all existing games |

## Component Inventory

### Shared: PianoKeyboardReveal

SVG component used by both NoteComparisonGame and IntervalGame. Renders a 1-octave view centered around the two played notes (D-01, D-02, D-03 from CONTEXT.md).

**Key dimensions:**
- White keys: 28px wide × 120px tall, `fill: white`, `stroke: rgba(255,255,255,0.3)`, `rx="3"`
- Black keys: 14px wide × 72px tall, `fill: #1e1b4b` (deep indigo — visible against dark bg), `stroke: rgba(255,255,255,0.3)`, `rx="2"`
- Total SVG width: 7 white keys × 28px = 196px canvas (can flex to container)
- Container: `bg-white/10 backdrop-blur-md border border-white/20 rounded-xl p-4`

**Highlight states:**
- Note 1 key: white key → `fill: #60a5fa` (blue-400), black key → `fill: #3b82f6` (blue-500)
- Note 2 key: white key → `fill: #fb923c` (orange-400), black key → `fill: #ea580c` (orange-600)
- In-between keys (IntervalGame only): `fill: rgba(255,255,255,0.2)` (dim white overlay)
- Normal/unhighlighted: white key → `fill: white`, black key → `fill: #1e1b4b`

**Note name labels** (appear below highlighted keys):
- Font: 14px (`text-sm`), weight 400, color `text-white`
- Note 1 label: `text-blue-300`
- Note 2 label: `text-orange-300`

**Direction arrow** (NoteComparisonGame only — appears between Note 1 and Note 2 labels):
- SVG arrow pointing up or down, 24px, `stroke: #818cf8` (indigo-400)
- Animate: `translateX` slide from Note 1 position to Note 2 position over 400ms, `ease-out`

**IntervalGame interval label** (appears below keyboard):
- Heading size (20px, bold): "SKIP — C4 to E4"
- Body size (16px, regular): "Jumped over D4" (only when exactly 1 note skipped)
- Color: `text-white`

**Slide-in animation** (both games — D-05 step 2):
- Container starts at `translateY(100%)` then animates to `translateY(0)` over 300ms, `ease-out`
- Respects `reducedMotion`: skip translate, only fade `opacity-0` → `opacity-100`

Answer buttons (D-04):
- Layout: `grid grid-cols-2 gap-3 w-full`
- Each button: `w-full min-h-[64px] flex flex-col items-center justify-center gap-1 rounded-xl`
- Default state: `bg-white/10 backdrop-blur-md border border-white/20 hover:bg-white/20 hover:border-white/40 cursor-pointer transition-colors duration-150`

**Answer reveal sequence** (D-05, CONTEXT.md):
1. 0ms: Selected button glows green/red
2. 100ms: PianoKeyboardReveal slides up from bottom (`translateY(100%)` → `translateY(0)`, 300ms ease-out)
3. 400ms: Note 1 key highlights blue
4. 600ms: Direction arrow animates from Note 1 to Note 2, Note 2 key highlights orange
5. 800ms: Direction label ("HIGHER ▲" or "LOWER ▼") appears at display size, `animate-floatUp`
6. Correct: pause 1500ms then advance; Wrong: pause 2000ms then advance

## Interaction States

### Answer Buttons (both games)

| State | Trigger | Visual |
|-------|---------|--------|
| Idle (awaiting answer) | game phase = `choosing` | Default glass style, hover active |
| Selected correct | user taps correct answer | Correct state (green glow), others dim immediately |
| Selected wrong | user taps wrong answer | Wrong state (red flash 300ms), then dims; correct button reveals green |
| Disabled (audio playing) | game phase = `listening` | `opacity-60 cursor-not-allowed` — all buttons non-interactive |
| Post-reveal pause | keyboard sliding in | All buttons remain in final feedback state, non-interactive |

### Replay Button

| State | Visual |
|-------|--------|
| Available | `bg-indigo-500 hover:bg-indigo-400` |
| Playing (locked) | `opacity-50 cursor-not-allowed` — prevents double-start |

### Piano Keyboard Reveal

| State | Visual |
|-------|--------|
| Hidden (pre-answer) | not rendered / `display: none` |
| Sliding in | `translateY(100%) → translateY(0)`, 300ms ease-out |
| Fully visible | Static; Note 1 blue, Note 2 orange, in-between dim (IntervalGame) |

### iOS Audio Interruption

`AudioInterruptedOverlay` component required in both games — identical pattern to MetronomeTrainer and RhythmDictationGame. No new visual design needed.

## Animation Inventory

All animations respect `prefers-reduced-motion`. When `reducedMotion` is true (from `AccessibilityContext`): skip all translateY/translateX animations; fade opacity only.

| Animation | Duration | Easing | Implementation |
|-----------|----------|--------|----------------|
| Keyboard reveal slide-up | 300ms | ease-out | Inline style `transform: translateY` toggled by state |
| Note 1 key highlight | 150ms | ease-out | SVG `fill` attribute update via ref or state |
| Direction arrow slide | 400ms | ease-out | Inline style `transform: translateX` from Note 1 x to Note 2 x position |
| Note 2 key highlight | 150ms after arrow reaches Note 2 | ease-out | SVG `fill` attribute update |
| Direction label appear | 200ms | ease-out | `animate-floatUp` Tailwind animation (defined in `tailwind.config.js`) |
| Answer button correct glow | 300ms | ease-out | `transition-all duration-300` class swap |
| Answer button wrong flash | 300ms flash then 200ms dim | — | Two-step state: `wrong` → `dimmed` |
| Correct/wrong sound | immediate | — | `useSounds` hook `playCorrectSound()` / `playWrongSound()` |

Source: CONTEXT.md D-05 (timing specs); `tailwind.config.js` `animation.floatUp`; Phase 8 DictationChoiceCard precedent.

## RTL Layout Notes

Both games must render correctly in Hebrew (`dir="rtl"`).

- Answer button row (NoteComparison — 2 columns): natural `grid-cols-2` flow — no dir override needed
- Answer button stack (IntervalGame — 3 rows): natural vertical flow — no dir override needed
- Piano keyboard SVG: force `dir="ltr"` — musical pitch order (C to B) is always left-to-right
- Direction arrow SVG: render as explicit `←` or `→` glyph (not `ArrowLeft`/`ArrowRight`), or keep `ArrowUp`/`ArrowDown` which do not flip
- Hint text inside buttons: natural inline RTL — no override needed
- Score / progress counter: `dir="ltr"` inner span for "3 / 10" display
- Note name labels (below keyboard keys): center-aligned, no RTL effect on single note names

## Accessibility

Minimum requirements matching Phase 8 and existing game components:

- Answer buttons: `<button>` elements with `aria-label` values ("Higher", "Lower", "Step — next door", "Skip — jump one", "Leap — far apart")
- Replay button: `aria-label="Play notes again"` (i18n key)
- Piano keyboard SVG: `aria-hidden="true"` — decorative visual reveal, not interactive
- Feedback text: `aria-live="polite"` region for "Correct!" / "Try again!" announcements
- Direction label: `aria-live="assertive"` for "HIGHER" / "LOWER" result announcement
- High contrast mode: borders increase from `border-white/20` to `border-white/60`; note highlights use stronger saturation (`blue-600`, `orange-600` instead of `blue-400`, `orange-400`)
- Reduced motion: skip all translate animations; fade only
