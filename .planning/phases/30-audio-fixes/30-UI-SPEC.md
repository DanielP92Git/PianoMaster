## Component Inventory

No new components. The two affected components and their relevant interactive elements:

### RhythmDictationQuestion.jsx — Listen / Replay button

- **Element:** Single `<button>` that switches between "Listen" (LISTEN_PROMPT phase) and "Replay" (CHOOSING phase)
- **Icon:** `<Volume2 className="h-5 w-5" />` in LISTEN_PROMPT; `<RotateCcw className="h-5 w-5" />` in CHOOSING
- **Label text:** "Listen" (idle) → "Playing..." (active) / "Replay" (idle in CHOOSING) → "Playing..." (active)
- **Disabled condition:** `isPlaying === true` — apply `opacity-60 cursor-default`
- **Fix impact (AUDIO-02):** `handleListen` gains an `await audioEngine.initializeAudioContext()` call before `resumeAudioContext()`. No visual change. The button's `isPlaying` state continues to gate the disabled appearance as before.

### DiscoveryIntroQuestion.jsx — Listen demo button

- **Element:** Single `<button>` for audio demo playback
- **Icon:** `<Volume2 className="h-5 w-5" />`
- **Label text:** "Listen" (idle) → "Playing..." (active)
- **Disabled condition:** `isPlaying === true` — apply `opacity-60 cursor-default`
- **Fix impact (AUDIO-03):** For `8_pair` focusDuration, `playDemo` now schedules 8 eighth notes (4 pairs) instead of 2. Button disabled duration increases from ~0.5s to ~2.0s (4 pairs at 80 BPM = 2 beats). The `isPlaying` state already handles this correctly — no visual change needed.

## Interaction States

### Audio button lifecycle (applies to both affected components)

```
[Idle]
  → user taps button
  → isPlaying = true (button shows "Playing...", opacity-60, cursor-default)
  → audio engine initializes (AUDIO-02 fix: initializeAudioContext awaited)
  → audio context resumes
  → pattern scheduled and plays
  → playback completes + 300ms buffer
  → isPlaying = false (button returns to idle state)
```

**For AUDIO-03 specifically (8_pair discovery demo):**

```
[Idle: "Listen"]
  → tap
  → isPlaying = true → "Playing..."
  → 8 eighth notes scheduled: high-low high-low high-low high-low
    (pitchShift alternates: 0 semitones for even index, -7 semitones for odd index)
  → total demo duration: ~2.0s at 80 BPM (8 × eighth-note = 2 beats)
  → isPlaying = false → "Listen"
```

**Reduced motion:** Both components consume `useMotionTokens()`. No new animations introduced. Existing `animate-fadeIn` on the SVG icon in `DiscoveryIntroQuestion` is unaffected.

**iOS Safari constraint (documented, not a visual concern):** `ctx.resume()` must be called synchronously within the user-gesture handler before any `await`. This is an implementation constraint, not a UI contract item. See RESEARCH.md Pitfall 1.
