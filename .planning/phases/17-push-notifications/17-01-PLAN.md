  <action>
Create the migration file `supabase/migrations/20260304000001_add_push_subscriptions.sql`:

1. Create `push_subscriptions` table:
   - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - `student_id` UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE, UNIQUE constraint
   - `subscription` JSONB — the full PushSubscription object from the browser (endpoint, keys.p256dh, keys.auth)
   - `is_enabled` BOOLEAN NOT NULL DEFAULT false
   - `parent_consent_granted` BOOLEAN NOT NULL DEFAULT false
   - `parent_consent_at` TIMESTAMPTZ — when parent solved the math problem
   - `last_notified_at` TIMESTAMPTZ — for 1/day rate limit enforcement
   - `created_at` TIMESTAMPTZ NOT NULL DEFAULT now()
   - `updated_at` TIMESTAMPTZ NOT NULL DEFAULT now()

2. Enable RLS: `ALTER TABLE push_subscriptions ENABLE ROW LEVEL SECURITY;`

3. RLS policies (following existing subscription pattern — client SELECT + own-row UPSERT, service role writes for Edge Function):
   - SELECT: `auth.uid() = student_id` (students can read their own row)
   - INSERT: `auth.uid() = student_id` (students can insert their own row)
   - UPDATE: `auth.uid() = student_id` (students can update their own row — for toggling is_enabled, updating subscription JSON)
   - DELETE: `auth.uid() = student_id` (students can delete their own row)
   Note: The Edge Function uses service_role key which bypasses RLS for writing last_notified_at and disabling expired subscriptions.

4. Create index: `CREATE INDEX idx_push_subscriptions_enabled ON push_subscriptions(is_enabled) WHERE is_enabled = true;` — for efficient Edge Function queries.

5. Add `[functions.send-daily-push]` block to `supabase/config.toml`:
   ```toml

   verify_jwt = false
   ```
   Place it after the existing `[functions.cancel-subscription]` block. The function uses `verify_jwt = false` because it is invoked by pg_cron (no user JWT). Security is enforced via a shared `x-cron-secret` header check inside the function.

<task type="auto">
  <name>Task 2: Create send-daily-push Edge Function</name>
  <files>supabase/functions/send-daily-push/index.ts</files>
  <action>
Create `supabase/functions/send-daily-push/index.ts` following the existing Edge Function patterns (lemon-squeezy-webhook style: header comment block, service role client, clear responsibilities).

The function:

1. **Security:** Check `x-cron-secret` header against `Deno.env.get('CRON_SECRET')`. Return 401 if mismatch. Handle OPTIONS for CORS (not needed for cron but good practice). Only accept POST.

2. **Initialize Web Push:**
   ```typescript
   import * as webpush from "jsr:@negrel/webpush";
   ```
   Import VAPID keys from environment: `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT` (e.g., "mailto:admin@pianomaster.app").
   Create ApplicationServer instance.

3. **Query eligible students** — use a single query with JOINs to minimize round trips:
   ```typescript
   const supabase = createClient(SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY);
   const today = new Date().toISOString().split('T')[0]; // YYYY-MM-DD

   const { data: eligibleStudents } = await supabase
     .from('push_subscriptions')
     .select(`
       student_id,
       subscription,
       last_notified_at,
       students!inner (
         total_xp,
         current_level
       )
     `)
     .eq('is_enabled', true)
     .not('subscription', 'is', null);
   ```
   Then filter in code: skip if `last_notified_at` is already today (UTC).

4. **Per-student processing loop:**
   a. Check if student practiced today: query `students_score` for `created_at >= today UTC midnight`, count only, head: true. If count > 0, skip.
   b. Get streak: query `current_streak` for `student_id`, get `streak_count`.
   c. Check daily goals: query `student_daily_goals` where `goal_date = today`, check if `completed_goals` array length < `goals` array length.
   d. Calculate XP to next level: inline XP_LEVELS array (same thresholds as client: [0, 100, 250, 450, 700, 1000, 1400, 1900, 2500, 3200, 4000, 5000, 6200, 7500, 9000]). Find next level threshold minus current total_xp.
   e. Call `selectNotification()` with streak, xp_to_next_level, has_incomplete_goals.

5. **selectNotification() function** — pure function, returns `{ title, body }`:
   Priority order per CONTEXT.md decisions:
   - Priority 1: Streak at risk (streak > 0) — 2-3 variants with musical emojis
   - Priority 2: XP near level-up (within 50 XP) — 2-3 variants
   - Priority 3: Daily goals waiting (has incomplete goals) — 2-3 variants
   - Priority 4: Generic encouragement (fallback) — 3 variants
   Messages should be fun, encouraging, written for an 8-year-old. Use musical emojis in titles and bodies.
   Random variant selection: `variants[Math.floor(Math.random() * variants.length)]`.

6. **Send push notification:**
   ```typescript
   const subscriber = appServer.subscribe(row.subscription);
   await subscriber.pushTextMessage(JSON.stringify({
     title: notification.title,
     body: notification.body,
     tag: 'daily-practice',
     data: { url: '/trail', type: 'daily-practice' }
   }), {});
   ```

7. **After successful send:** Update `last_notified_at` to now.

8. **Error handling:**
   - Catch per-student errors (don't let one failure stop the loop).
   - On 410 Gone status: set `is_enabled = false, subscription = null` for that student (expired subscription cleanup).
   - Log errors but continue processing.

9. **Response:** Return JSON `{ sent: N, failed: N, skipped: N }` with 200 status. Include total eligible count for monitoring.

Keep the function in a single file (no lib/ subdirectory) since the logic is straightforward and doesn't need unit testing of sub-modules.
