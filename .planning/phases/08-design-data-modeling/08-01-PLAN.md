**In src/hooks/micInputPresets.js ‚Äî add `calcMicTimingFromBpm` function (PIPE-01, PIPE-02):**

Add a named export function `calcMicTimingFromBpm(bpm, shortestNoteDuration)` that returns `{ onFrames, offMs, changeFrames, minInterOnMs }`.

Parameters:
- `bpm` (number) ‚Äî tempo in beats per minute (60-200 range expected)
- `shortestNoteDuration` (string, default `'q'`) ‚Äî VexFlow duration code: `'q'` (quarter), `'8'` (eighth), `'16'` (sixteenth), `'h'` (half), `'w'` (whole)

Calculation logic:
1. Compute `beatMs = 60000 / bpm` (ms per quarter note)
2. Compute `noteMs` from shortestNoteDuration:
   - `'w'` = beatMs * 4
   - `'h'` = beatMs * 2
   - `'q'` = beatMs
   - `'8'` = beatMs / 2
   - `'16'` = beatMs / 4
3. Derive timing params (assume ~60fps, so 1 frame ~ 16.7ms):
   - `onFrames` = Math.max(2, Math.round(noteMs * 0.15 / 16.7)) ‚Äî 15% of note duration for onset detection, minimum 2 frames
   - `offMs` = Math.max(60, Math.round(noteMs * 0.4)) ‚Äî 40% of note duration for note-off, minimum 60ms
   - `changeFrames` = Math.max(3, onFrames + 1) ‚Äî one more than onFrames for note-change stability
   - `minInterOnMs` = Math.max(40, Math.round(noteMs * 0.25)) ‚Äî 25% of note duration between triggers, minimum 40ms
4. Return `{ onFrames, offMs, changeFrames, minInterOnMs }`

Keep the existing `MIC_INPUT_PRESETS` object unchanged ‚Äî it serves as fallback when no BPM is available.

Add a JSDoc comment documenting the function, its parameters, and example output for 120 BPM eighth notes.

**Refactor the note detection state machine in useMicNoteInput (PIPE-03):**

Replace the ad-hoc `candidateNote` / `candidateFrames` counting with a formal three-state FSM.

1. Add state enum at the top of the file (outside the hook):
```js
const FSM = { IDLE: 'IDLE', ARMED: 'ARMED', ACTIVE: 'ACTIVE' };
```

2. In `stateRef.current`, replace `candidateNote`, `candidateFrames`, `candidateStartedAt` with:
   - `fsmState: FSM.IDLE` ‚Äî current FSM state
   - `candidateNote: null` ‚Äî pitch being evaluated (used in ARMED)
   - `candidateFrames: 0` ‚Äî frames accumulated for candidate (used in ARMED)
   - `candidateStartedAt: -Infinity` ‚Äî when candidate tracking began
   Keep existing fields: `currentNote`, `lastPitchAt`, `prevPitchAt`, `lastEmitAt`, `lastFrequency`, `lastAudioLevel`.

3. Rewrite `handlePitchDetected` using FSM transitions:

   **IDLE state** (no note held, no candidate):
   - If pitch detected: transition to ARMED, set candidateNote = note, candidateFrames = 1, candidateStartedAt = now

   **ARMED state** (building onset confidence):
   - If same note as candidate: increment candidateFrames
     - If candidateFrames >= onFrames AND now - lastEmitAt >= minInterOnMs: transition to ACTIVE, set currentNote = note, emit noteOn
   - If different note: reset candidateNote to new note, candidateFrames = 1, candidateStartedAt = now (stay ARMED)

   **ACTIVE state** (note currently held):
   - If same note as currentNote: do nothing (sustain)
   - If different note: track as candidate for note-change
     - If candidateNote === note: increment candidateFrames
       - If candidateFrames >= changeFrames AND now - lastEmitAt >= minInterOnMs: emit noteOff for old, transition to ACTIVE with new note, emit noteOn
     - If candidateNote !== note: reset candidate tracking (candidateNote = note, candidateFrames = 1)

4. Rewrite `handleLevelChange` (silence detection):
   - If fsmState is ACTIVE and now - lastPitchAt >= offMs: emit noteOff, transition to IDLE, reset all candidate fields
   - If fsmState is ARMED and now - lastPitchAt >= offMs: transition to IDLE, reset candidate fields (no emission needed)

5. Update `resetInternalState` to set `fsmState: FSM.IDLE` and clear all candidate fields.

**IMPORTANT:** The external API (props, return value) must NOT change. The hook still accepts onFrames, offMs, changeFrames, minInterOnMs as props. The FSM is purely internal ‚Äî callers see the same noteOn/noteOff events.

**IMPORTANT:** Keep the `debug` useMemo returning the same shape. Add `fsmState` to the debug object for observability.
