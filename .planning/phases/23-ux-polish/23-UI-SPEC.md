## Color

Source: CLAUDE.md glassmorphism design system + `tailwind.config.js` + existing game components.

| Role            | Value                                                            | Usage                                                    |
| --------------- | ---------------------------------------------------------------- | -------------------------------------------------------- |
| Dominant (60%)  | `bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900` | Full-screen game background — all rhythm game routes     |
| Secondary (30%) | `bg-white/10 backdrop-blur-md border border-white/20`            | Glass cards: staff container, tap area, header bar       |
| Accent (10%)    | `bg-indigo-500` / `text-indigo-300`                              | See reserved list below                                  |
| Destructive     | `text-red-400`                                                   | "Almost!" FloatingFeedback text only (replaces MISS red) |

Accent reserved for:

1. Primary action buttons: "Start Game", "Next" (`bg-indigo-500 hover:bg-indigo-400`)

3. Syllable toggle button active/on state: `bg-indigo-500/30 border-indigo-400/40 text-indigo-300`
4. Beat indicator dots when active (MetronomeDisplay)

Feedback color mapping (source: `FloatingFeedback.jsx` lines 52-57 — preserve exactly):

- PERFECT: `text-green-400`
- GOOD: `text-yellow-400`
- MISS ("Almost!"): `text-red-400`

Syllable annotation color (VexFlow SVG text): `rgba(255,255,255,0.75)` (white/75) — readable on dark staff background without competing with note heads. Chosen as Claude's discretion (D-20).

## Component Inventory

### Syllable Toggle Button (new — UX-05)

Position: Settings bar area above the staff, right-aligned. Sits in the existing header row or a dedicated settings strip below the header bar (same row as exercise counter).

Appearance — OFF state (syllables hidden):

```
bg-white/10 border border-white/20 text-white/60
rounded-xl px-3 py-2 text-sm font-medium
min-h-[44px] transition-colors hover:bg-white/20
```

Appearance — ON state (syllables visible):

```
bg-indigo-500/30 border border-indigo-400/40 text-indigo-300
rounded-xl px-3 py-2 text-sm font-medium
min-h-[44px] transition-colors hover:bg-indigo-500/40
```

Hidden state (Discovery nodes — enforced-on, D-14):

- Button is not rendered. Syllables render without toggle chrome.

Label (EN): "Do-Re" or a music note icon + "Syllables" — use "Syllables" text label with a music note icon prefix (Unicode ♩ or inline SVG). Final label: `♩ Syllables`.
Label (HE): `הברות ♩` (RTL — icon suffix).

Accessibility: `aria-pressed={syllablesEnabled}`, `aria-label="Toggle rhythm syllables"` (EN) / `aria-label="הצג הברות קצב"` (HE).

### VexFlow Syllable Annotations (new — UX-05)

Rendered via VexFlow `Annotation` API inside `beatsToVexNotes()` before `formatter.format()`.

Font spec (passed to `annotation.setFont()`):

```javascript
{ family: "sans-serif", size: 10, weight: "normal" }
```

This keeps syllables at 10px — readable but clearly subordinate to the 14px+ note duration numerals. Source: RESEARCH.md UX-05 section, D-20 discretion.

Vertical justification: `AnnotationVerticalJustify.BOTTOM` (index 3).

Duration-to-syllable mapping:

| VexFlow dur | EN syllable | HE syllable (Nikud)                             |
| ----------- | ----------- | ----------------------------------------------- |
| `'w'`       | `ta-a-a-a`  | `טָה-אָה-אָה-אָה`                               |
| `'h'`       | `ta-a`      | `טָה-אָה`                                       |
| `'q'`       | `ta`        | `טָה`                                           |
| `'8'`       | `ti`        | `טִי`                                           |
| any rest    | `sh`        | `הס` (exact Nikud TBD by user — see note below) |

Note: Hebrew Nikud for "Almost!" HE i18n value and rest syllable `הס` with Nikud are flagged as user-confirmed TBD in D-18, D-19, and RESEARCH.md. Planner must include a user-confirm step for these two Hebrew strings before implementation. Do not hardcode placeholder Nikud.

Per-note vs per-pair eighth notes: Each individual eighth note gets "ti" (not "ti-ti"). "ti-ti" describes the rhythmic pair as a unit, but VexFlow annotates per note. Source: RESEARCH.md UX-05 note.

### FloatingFeedback — MISS → "Almost!" (UX-03)

No visual change to the component. Only the i18n string values change:

- `games.metronomeTrainer.tapArea.accuracy.miss`: `"Miss"` → `"Almost!"`
- `games.rhythmReading.tapArea.accuracy.miss`: `"MISS"` → `"Almost!"`

Color remains `text-red-400` (existing — softened by copy change alone, not by color change).
Animation: unchanged — float-up 800ms, opacity fade.

### Multi-Stave RhythmStaffDisplay (UX-04)

When `measures` prop is provided (array of beat arrays):

- Render one `Stave` per measure in horizontal sequence
- Each stave: same height as current single stave
- Stave width: divide available container width equally across measures (e.g. 2 staves = 50% each, 4 staves = 25% each)
- Bar lines between measures: VexFlow `stave.setBegBarType(Barline.type.NONE)` on staves after the first; end barline on final stave only
- Cursor logic: cursor position maps to current measure + beat-within-measure

Backward compatibility: existing `beats` prop (flat array) continues to render as single stave — no breaking change. Source: RESEARCH.md Pitfall 4.

## Interaction States

### Syllable Toggle

| State         | Trigger                              | Visual                                            |
| ------------- | ------------------------------------ | ------------------------------------------------- |
| Hidden        | nodeType === 'discovery'             | Button not rendered; syllables always visible     |
| Off (default) | User preference not set, or false    | `bg-white/10 text-white/60` (muted glass)         |
| On            | User toggled on OR localStorage true | `bg-indigo-500/30 text-indigo-300` (accent glass) |
| Press         | Pointer down                         | `scale-95` (100ms transition)                     |

localStorage key: `pianomaster_kodaly_syllables` (string `"true"` / `"false"` or absent).

### Tap Area Feedback

| Quality | Color             | Copy (EN) | Copy (HE)   |
| ------- | ----------------- | --------- | ----------- |
| PERFECT | `text-green-400`  | PERFECT   | (existing)  |
| GOOD    | `text-yellow-400` | GOOD      | (existing)  |
| MISS    | `text-red-400`    | Almost!   | TBD (Nikud) |

### Reduced Motion

All animations respect `reducedMotion` from `AccessibilityContext`:

- FloatingFeedback: no translateY when reducedMotion=true (existing)
- Syllable toggle: no scale-95 press animation when reducedMotion=true
- Cursor sweep: existing reduced-motion handling preserved
