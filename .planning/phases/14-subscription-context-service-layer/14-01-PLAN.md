**File 1: `src/services/subscriptionService.js`**

Create a pure async function `fetchSubscriptionStatus(studentId)` that:
1. Returns `{ isPremium: false }` if `studentId` is falsy (guard)
2. Queries `parent_subscriptions` table: `.select('status, current_period_end').eq('student_id', studentId).maybeSingle()` (use `.maybeSingle()` not `.single()` — no row for unsubscribed users should return null, not throw)
3. Returns `{ isPremium: false }` if error or no data
4. Mirrors `has_active_subscription()` Postgres logic exactly:
   - `status === 'active' || status === 'on_trial'` -> `{ isPremium: true }`
   - `status === 'cancelled' && current_period_end && current_period_end > now` -> `{ isPremium: true }` (grace: cancelled but period not ended)
   - `status === 'past_due' && current_period_end` -> compute `threeDaysLater = new Date(new Date(current_period_end).getTime() + 3 * 24 * 60 * 60 * 1000).toISOString()` -> if `threeDaysLater > now` -> `{ isPremium: true }`
   - All other statuses -> `{ isPremium: false }`
5. Use `const now = new Date().toISOString()` for timestamp comparisons
6. Follow project convention: named export + default export object, JSDoc comment on function, `import supabase from './supabase'`

**File 2: `src/contexts/SubscriptionContext.jsx`**

Create `SubscriptionProvider` component and `useSubscription()` hook:

1. Import: `createContext, useContext, useEffect, useRef` from react; `useQuery, useQueryClient` from @tanstack/react-query; `fetchSubscriptionStatus` from services; `useUser` from features/authentication; `supabase` from services; `toast` from react-hot-toast
2. Create context: `const SubscriptionContext = createContext()`
3. `SubscriptionProvider({ children })`:
   - Call `useUser()` to get `{ user, isAuthenticated }`; extract `userId = user?.id`
   - Call `useQueryClient()` for invalidation
   - Set up `useQuery`:
     - `queryKey: ['subscription', userId]`
     - `queryFn: () => fetchSubscriptionStatus(userId)`
     - `enabled: isAuthenticated && !!userId` (no fetch until auth ready)
     - `staleTime: 0` (SVC-02 requirement — override global 5min default)
     - `refetchOnWindowFocus: false` (Realtime handles push; avoid redundant fetches)
     - `retry: 1`
   - Set up Realtime channel in `useEffect` with `[userId, queryClient]` deps:
     - Guard: `if (!userId) return`
     - Create channel: `supabase.channel(`subscription-changes-${userId}`)`
     - Subscribe to `postgres_changes` with `event: '*', schema: 'public', table: 'parent_subscriptions', filter: `student_id=eq.${userId}``
     - Callback: `queryClient.invalidateQueries({ queryKey: ['subscription', userId] })` (v5 object syntax)
     - Add subscribe status log gated by `process.env.NODE_ENV === 'development'`
     - Cleanup: `return () => { supabase.removeChannel(channel) }`
   - **Toast de-duplication** (Pitfall 2 from RESEARCH.md):
     - Create `prevIsPremiumRef = useRef(null)`
     - Add a `useEffect` watching `data`:
       ```
       if (prevIsPremiumRef.current === false && data?.isPremium === true) {
         toast.success('Premium unlocked!', { duration: 4000 });
       }
       prevIsPremiumRef.current = data?.isPremium ?? false;
       ```
     - Do NOT show toast in the Realtime callback — only in the useEffect watching query data. This prevents duplicate toasts on Realtime reconnect.
   - Derive: `const isPremium = data?.isPremium ?? false` (defaults to false per CONTEXT.md)
   - Return `<SubscriptionContext.Provider value={{ isPremium, isLoading }}>` wrapping children
4. `useSubscription()` hook:
   - `useContext(SubscriptionContext)`
   - Throw if context is undefined: `throw new Error('useSubscription must be used within a SubscriptionProvider')`
   - Return context value

Follow project conventions: double quotes, 2-space indent, JSDoc on exports, named exports for SubscriptionProvider and useSubscription.
