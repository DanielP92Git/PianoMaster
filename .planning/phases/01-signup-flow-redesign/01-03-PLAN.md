    **Architecture decisions implemented:**

    1. **Step state** (D-01): `useState("role")` with values `'role' | 'birth-year' | 'parent-email' | 'credentials'`. Step dots use `STUDENT_STEPS` / `TEACHER_STEPS` constants.

    2. **Role selection is Step 1** (D-02, D-03): Clicking Student/Teacher immediately navigates to next step (no separate Continue button — direct selection triggers navigation per the card-click pattern from existing `RoleSelection.jsx`).

    3. **Teacher shortcut** (D-03): Teachers go directly from role to credentials. `TEACHER_STEPS` has only 2 entries so dots show 2.

    4. **Back navigation** (D-04):
       - `birth-year` -> back to `role`
       - `parent-email` -> back to `birth-year`
       - `credentials` (teacher) -> back to `role`
       - `credentials` (student, under-13) -> back to `parent-email`
       - `credentials` (student, 13+) -> back to `birth-year`

    5. **Reset downstream state on role change** (Pitfall 1 from RESEARCH.md): When role is selected in Step 1, `birthYear` and `parentEmail` are reset to null to prevent stale state when navigating back.

    6. **Google OAuth** (D-08, D-09): `SocialLogin` is rendered on the credentials step with `role={role || "student"}`. Per D-15 (Claude's discretion): Google OAuth students skip birth year collection (role is already known from Step 1; no birth year intercept needed for OAuth flow).

    7. **Role selection card design**: Vertical card layout (like `RoleSelection.jsx`) with icon + label + description. Glass card hover states. "Already have an account? Log in" link below cards.

    8. **Removed**: Role selection from the credentials step (it was in the old Step 3). Role is now set in Step 1 and used throughout.

    9. **Preserved**: All form field IDs (`signup-firstName`, `signup-lastName`, `signup-email`, `signup-password`), submit button text pattern, Terms/Privacy links, SocialLogin placement, glass styling classes, responsive layout (flex-col on mobile, flex-row on lg).
