## Typography

All sizes use the Nunito (`font-rounded`) font family, matching existing dashboard card text.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.6 (from tailwind fontSize config) |
| Label | 12px (text-xs) | 700 (bold) | 1.5 |
| Heading | 14px (text-sm) | 700 (bold) | 1.6 |
| Button text | 14px (text-sm) | 700 (bold) | 1 (single line) |

**Rationale:** Matches DailyChallengeCard.jsx headings: `text-sm font-bold text-white`. Button text uses the same bold weight for visual consistency and legibility on the dark gradient background. Audience is 8-year-olds — no smaller than 12px (text-xs). Two declared weights: 400 (regular) and 700 (bold).

## Color

The dashboard renders on `bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900` (AppLayout.jsx).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | indigo-900/purple-900/violet-900 gradient | App background, behind all cards |
| Secondary (30%) | `bg-white/10 backdrop-blur-md border border-white/20` | Card containers (glass card pattern) |
| Accent (10%) | emerald-400 / green-500 / green-300 | Practice card only — see reserved list |
| Destructive | red-400 / kidsError-400 | No destructive actions in this phase |

**Accent (emerald/green) reserved for:**
- Practice card border glow: `border-emerald-400/30` (active state) / `border-white/20` (not-yet state)
- Log button idle: `bg-emerald-600 hover:bg-emerald-700` (pre-log CTA)
- Log button logged: `bg-green-500/20 border border-green-400/30` (completed state)
- Streak count text: `text-green-300`
- Practice icon (`Piano`): `text-emerald-400`
- XP badge: `bg-emerald-400/20 text-emerald-300` ("+25 XP" pill in button)
- Checkmark on logged state: `text-green-400` (CheckCircle icon)

**Non-accent colors (already owned by other elements — do NOT reuse for practice):**
- amber/orange: app-usage streak (Flame icon in StreakDisplay)
- indigo-600/indigo-400: XP ring, existing CTA buttons (DailyChallengeCard)
- amber-400: Zap/challenge icons (DailyChallengeCard header)

## Interaction Contract

### Log button tap sequence
(source: CONTEXT.md D-06, D-07)

1. **Idle:** Button shows "Yes, I practiced!" — `bg-emerald-600`
2. **Tap (0ms):** Instant optimistic update. Button transitions to "Logged! +25 XP" — `bg-emerald-700`, disabled. CheckCircle icon scales in over 150ms ease-out.
3. **Hold (2000ms):** Button remains in "Logged! +25 XP" state.
4. **Settle (2001ms+):** Card transitions to State 4 (practiced today) with full completed styling.

**Reduced-motion fallback:** Skip CheckCircle scale-in animation. Instant opacity change instead. Gate with `useMotionTokens` hook (existing: `src/utils/useMotionTokens.js`).
(source: CONTEXT.md D-06, D-17)

### No destructive actions in this phase
No confirmation dialogs, destructive buttons, or irreversible actions exist in Phase 2 UI.

### Already-logged on page load
If the student returns to the dashboard after logging, the card renders directly in State 4. No button is shown in active state. Streak count updates to reflect the new streak.
(source: CONTEXT.md D-08)

## Copywriting Contract

All strings must have EN and HE translation keys under the `practice` namespace in `locales/en/common.json` and `locales/he/common.json`.
(source: REQUIREMENTS.md INFRA-05, CONTEXT.md D-15)

| Element | English Copy | i18n Key |
|---------|-------------|----------|
| Card title | "Practice Instrument" | `practice.card.title` |
| Active prompt | "Did you practice today?" | `practice.card.prompt` |
| Log button (idle) | "Yes, I practiced!" | `practice.card.logButton` |
| Log button (logging) | "Logged! +25 XP" | `practice.card.loggingButton` (split: `practice.card.loggingText` + `practice.card.xpBadge`) |
| Log button (settled) | "Practiced today!" | N/A — State 4 replaces button |
| Completed heading | "Practiced today!" | `practice.card.completedHeading` |
| Completed XP line | "+25 XP earned" | `practice.card.xpEarned` |
| Streak label (singular) | "day practice streak" | `practice.streak.dayLabel` (count: 1) |
| Streak label (plural) | "day practice streak" | `practice.streak.dayLabel` (count: N) |
| Loading text | (skeleton — no text shown) | N/A |

**Tone:** Kid-friendly and celebratory. Matches "Challenge Complete!" and "New challenge tomorrow!" in DailyChallengeCard. Short, positive, no exclamation overload (max 1 per string).

**Empty state:** No zero-state empty card shown. At streak 0, the active prompt ("Did you practice today?") is shown. Zero-day streak count row is hidden until day 1 is logged (avoid showing "0-day practice streak" which is demotivating). Per REQUIREMENTS.md, `LOG-F02` (zero-state CTA) is deferred to v3+.

**Error state:**
- Network failure logging: Button reverts to idle state. No error toast. Silent retry on next tap.
- If Supabase RPC returns non-idempotent XP error: Log is still shown as succeeded (DB constraint prevents double-award silently).

## Accessibility Contract

- Log button must have `aria-label` when icon-only context is ambiguous. Full text label "Yes, I practiced!" satisfies this naturally.
- Logged state button: `disabled` attribute + `aria-disabled="true"` + `cursor-not-allowed`.
- Reduced-motion: All animations gated via `useMotionTokens`. Zero-duration fallback for scale-in and any pulse effects.
- Piano icon is decorative (`aria-hidden="true"`) — text label carries the semantic meaning.
- Minimum touch target: 44px height on log button (`min-h-touch` Tailwind token).
