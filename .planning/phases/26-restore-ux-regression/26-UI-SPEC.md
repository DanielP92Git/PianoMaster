## Typography

Source: `tailwind.config.js` fontSize scale (locked).

| Role                | Size                              | Weight        | Line Height           |
| ------------------- | --------------------------------- | ------------- | --------------------- |
| Body / label        | 14px (sm)                         | 400 (regular) | 1.6                   |
| UI text / settings  | 16px (base)                       | 400 (regular) | 1.7                   |
| Feedback label      | 30px (3xl / `text-3xl`)           | 700 (bold)    | 1.9                   |
| Syllable annotation | 11px (custom, VexFlow Annotation) | 400 (regular) | n/a (VexFlow-managed) |

Weights in use: regular (400) and bold (700). Two weights only.

Font family for syllable annotations:

- English: `sans-serif` (VexFlow default is acceptable; do not override)
- Hebrew (Nikud): `Heebo` or `Noto Sans Hebrew` — must be a font that renders diacritics. Pass font family via VexFlow Annotation `setFont()`.

### New UI element: syllable toggle button

Placement: in the settings/controls bar above the rhythm staff, same row as existing controls.

Visual spec:

- Inactive state: `text-white/50 hover:text-white/80` icon button, 44px touch target
- Active state: `text-indigo-300` with `bg-white/10 rounded-lg` background tint
- Icon: music note or syllable label (Claude's discretion — suggested: text `"ta"` label or a note-with-text SVG)
- Hidden entirely on Discovery nodes (syllables enforced, no toggle visible)
- Present but respects `reducedMotion` — no animation on toggle state change if reduced-motion

### New UI element: multi-stave layout

When `measures > 1`, `RhythmStaffDisplay` renders multiple staves stacked vertically.

Layout spec:

- Each stave is full container width
- Vertical gap between staves: 8px (`--spacing-sm`)
- Container scrolls vertically if staves overflow viewport — `overflow-y-auto`
- Cursor overlay spans only the active measure stave

## Interaction Contract

### UX-01: Timing forgiveness

- `scoreTap(tapTime, expectedTime, tempo, nodeType)` — new `nodeType` parameter
- When `nodeType` is in `EASY_NODE_TYPES` (Discovery, Practice, MIX_UP, REVIEW): use `BASE_TIMING_THRESHOLDS_EASY` (PERFECT=100ms base)
- When `nodeType` is in hard set (CHALLENGE, SPEED_ROUND, MINI_BOSS, BOSS): use existing `BASE_TIMING_THRESHOLDS` (PERFECT=50ms base)
- Tempo scaling is applied on top of the base threshold (existing behavior preserved)
- No visible UI change — entirely logic-layer

### UX-02: Game rename

- `useDocumentTitle` in MetronomeTrainer defaults to `t('games.metronomeTrainer.headerTitle')`
- Header `<h1>` / title bar renders via i18n key — resolves to "Listen & Tap" / "חזור אחריי"
- The literal string "MetronomeTrainer" must not appear in any rendered label, heading, or page title

### UX-03: Almost! feedback

- `FloatingFeedback` renders `t('games.rhythmReading.tapArea.accuracy.miss')` — resolves to "Almost!"
- Color for MISS quality remains `text-red-400` (warm encouraging red — unchanged per design)
- Animation: identical float-up behavior (800ms ease-out, 850ms total) — unchanged
- `aria-live="polite"` region already in place — no change needed

### UX-04: Progressive measure length

- `RhythmStaffDisplay` `measures` prop: integer 1–4, defaults to 1
- When `measures > 1`: renders `measures` staves vertically stacked; each stave is independent VexFlow render
- `trailMeasureCount` derived from node config `measureCount` field and passed down via `RhythmReadingGame`
- `validateMeasureCountPolicy()` in `validateTrail.mjs`: enforces table from D-12, fails build with descriptive error on violation

### UX-05: Kodaly syllable toggle

- Toggle button: appears in settings bar above staff in `RhythmReadingGame` and `RhythmDictationGame`
- Discovery nodes: toggle absent, syllables always rendered (enforced)
- All other nodes: toggle present, initial state from `localStorage.getItem('syllablesEnabled')`, default `true` if not set
- `localStorage` key: `'syllablesEnabled'` (string `'true'`/`'false'`)
- VexFlow Annotation: attached below note head, positioned with `setVerticalJustification(Annotation.VerticalJustify.BOTTOM)`
- Syllable font size: 11px via VexFlow `setFont('Heebo', 11)` for HE, `setFont('sans-serif', 11)` for EN
- REST syllables: "sh" (EN) / "הָס" (HE) — shown below rest note elements

### PAT-04: resolveByTags

- `RhythmReadingGame.jsx` imports `{ resolveByTags }` from `../utils/RhythmPatternGenerator`
- Trail-mode pattern selection calls `resolveByTags(patternTags, measureCount)` — not legacy `getPattern()`
- No UI change

## Accessibility Contract

- Syllable toggle: `aria-label` reads the inverse action ("Show syllables" when off, "Hide syllables" when on)
- FloatingFeedback already has `aria-live="polite"` — no change
- RTL support: Hebrew i18n already handled by `i18next`; VexFlow syllables render LTR regardless of page direction (acceptable — musical notation is always LTR)
- Reduced-motion: syllable toggle has no animation; floating feedback uses opacity-only fallback (already implemented)
