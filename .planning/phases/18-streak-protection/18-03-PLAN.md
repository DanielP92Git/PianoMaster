  <action>
Add a "Streak Settings" section to AppSettings between the Subscription section and the Install section.

1. **Import additions**: Import `Flame` from lucide-react (for the section icon). Import `streakService` from services. Import `ParentGateMath` from settings components. Import `useQuery` and `useMutation` from tanstack (already imported by other components in the file — check if useQuery is available, otherwise add).

2. **State management**:
   - Use `useQuery` with key `["streak-state", user?.id]` to fetch `streakService.getStreakState()`, enabled when `user?.id` exists.
   - Track `showParentGate` state (boolean, default false).
   - Track `pendingToggleValue` state (the value the user wants to set, stored while gate is showing).

3. **Parent consent check**:
   - Query `push_subscriptions` for `parent_consent_granted` flag using the existing pattern from NotificationPermissionCard. The simplest approach: query `push_subscriptions` for consent status (it's already in the DB). If `parent_consent_granted === true` on push_subscriptions, skip the gate for weekend pass too. This makes COPPA consent a single flag per student.
   - If no push_subscriptions row exists (student hasn't interacted with push), treat as no consent — show gate.

4. **Toggle flow**:
   - When user taps the weekend pass toggle:
     a. If consent already granted → directly call `streakService.setWeekendPass(!currentValue)` and invalidate `["streak-state"]`.
     b. If consent NOT granted → set `pendingToggleValue = !currentValue`, `showParentGate = true`.
   - When ParentGateMath `onConsent` fires:
     a. Update `push_subscriptions.parent_consent_granted = true` (upsert pattern).
     b. Call `streakService.setWeekendPass(pendingToggleValue)`.
     c. Close gate, invalidate queries.
   - When ParentGateMath `onCancel` fires: close gate, clear pending value.

5. **Add `setWeekendPass` to streakService** if not already exported from Plan 01:
   - This is a simple upsert: `supabase.from('current_streak').upsert({ student_id, weekend_pass_enabled: value }, { onConflict: 'student_id' })`.
   - If Plan 01 already includes this, just use it. If not, add it to streakService.js in this task (minor addition, same file pattern).

6. **Section UI** (between Subscription and Install sections):
   ```jsx
   <SettingsSection
     isRTL={isRTL}
     title={t("streak.weekendPassLabel")}
     description={t("streak.weekendPassDescription")}
     icon={Flame}
     defaultOpen={false}
   >
     <div className="mt-4">
       <ToggleSetting
         isRTL={isRTL}
         label={t("streak.weekendPassLabel")}
         description={t("streak.weekendPassDescription")}
         value={streakState?.weekendPassEnabled || false}
         onChange={handleWeekendPassToggle}
       />
       {streakState?.weekendPassEnabled && (
         <p className="text-xs text-green-300 mt-2">
           {t("streak.weekendPassEnabled")}
         </p>
       )}
     </div>
   </SettingsSection>
   ```

7. **ParentGateMath overlay**: Render conditionally when `showParentGate` is true, same pattern as NotificationPermissionCard uses it.
