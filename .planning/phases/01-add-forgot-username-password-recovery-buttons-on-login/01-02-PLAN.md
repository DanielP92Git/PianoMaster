**Create `src/pages/ResetPasswordPage.jsx`:**

A dedicated page for setting a new password after clicking the reset link in the email. Must visually match the login page (same background, same glass card style).

Structure:
```jsx
import React, { useState, useEffect } from 'react';
import { useNavigate } from 'react-router-dom';
import { Eye, EyeOff, Loader2, Lock, AlertCircle } from 'lucide-react';
import { useUpdatePassword } from '../features/authentication/useUpdatePassword';
import { useTranslation } from 'react-i18next';
import supabase from '../services/supabase';
```

**Component `ResetPasswordPage`:**

State:
- `password` (string)
- `confirmPassword` (string)
- `showPassword` (boolean)
- `showConfirmPassword` (boolean)
- `validationError` (string | null)
- `isSessionReady` (boolean) -- tracks whether Supabase has detected the recovery token from the URL

**Session detection:**
- Supabase PKCE flow: When the user clicks the reset link, they are redirected to `/reset-password` with a token in the URL fragment. The Supabase client (`detectSessionInUrl: true` in supabase.js config) automatically picks up the token and establishes a recovery session.
- Use `useEffect` to listen for auth state changes:
  ```jsx
  useEffect(() => {
    const { data: { subscription } } = supabase.auth.onAuthStateChange((event) => {
      if (event === 'PASSWORD_RECOVERY') {
        setIsSessionReady(true);
      }
    });
    // Also check if already in a session (page reload case)
    supabase.auth.getSession().then(({ data: { session } }) => {
      if (session) setIsSessionReady(true);
    });
    return () => subscription?.unsubscribe();
  }, []);
  ```
- If `!isSessionReady` after 3 seconds, show an error state: "This reset link has expired or is invalid. Please request a new one." with a link back to `/login`.

**Validation (on submit, before calling updatePassword):**
- If `password.length < 6`: set `validationError` to `t('auth.resetPassword.passwordTooShort')`, return.
- If `password !== confirmPassword`: set `validationError` to `t('auth.resetPassword.passwordMismatch')`, return.
- Clear `validationError` and call `updatePassword({ password })`.

**Layout:**
- Same full-screen background as login: `h-screen bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900 flex items-center justify-center p-4`
- Glass card: `bg-white/10 backdrop-blur-lg rounded-2xl shadow-2xl border border-white/20 w-full max-w-md p-6 md:p-8`
- Lock icon at top: `<Lock className="w-10 h-10 text-indigo-400 mx-auto mb-3" />`
- Title: `t('auth.resetPassword.title')` -- `text-xl font-bold text-white text-center mb-6`
- Two password fields, each with Eye/EyeOff toggle (reuse the EXACT same pattern from LoginForm.jsx lines 118-143):
  - "New password" field with `showPassword` toggle
  - "Confirm password" field with `showConfirmPassword` toggle
  - Same input classes: `w-full px-3 py-2 pr-10 text-sm rounded-lg border-2 border-white/20 bg-white/15 backdrop-blur-sm focus:bg-white/25 focus:ring-2 focus:ring-indigo-500 focus:border-indigo-400/50 transition-all duration-300 text-white placeholder-white/70`
  - Same Eye/EyeOff button pattern with `absolute right-2 top-1/2 -translate-y-1/2`
- Validation error display: if `validationError`, show `<div className="flex items-center gap-2 text-red-300 text-sm"><AlertCircle className="w-4 h-4 shrink-0" />{validationError}</div>`
- Submit button: `t('auth.resetPassword.submitButton')` / `t('auth.resetPassword.updating')` when pending. Same `bg-indigo-600 hover:bg-indigo-700` style. Full width. Disabled when `isPending` or both fields empty.
- "Back to login" link at bottom: `text-white/60 hover:text-white/90 text-sm text-center` linking to `/login`.

**Error state (expired/invalid link):**
- If session not detected within timeout: Show `AlertCircle` icon (amber), error message `t('auth.resetPassword.errorExpiredLink')`, and a "Back to login" button that navigates to `/login`.
