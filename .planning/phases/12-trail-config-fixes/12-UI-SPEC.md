## Phase UI Classification

**This phase introduces zero new UI surface.** All three requirements (TCFG-01, TCFG-02, TCFG-03) are internal logic fixes:

- TCFG-01: `getPattern()` function signature change + 4 game call-site updates (pure JS logic)
- TCFG-02: String substitution in data files (`'easy'` → `'beginner'`) (pure data fix)
- TCFG-03: Test assertion replacement in 2 test files (pure test logic)
- D-06: New regression test file (pure test logic)
- D-07: Build script enhancement (pure validation logic)

No new components, screens, modals, overlays, or states are added. No existing component visual output changes. The rhythm games will generate different patterns when `rhythmPatterns` is specified in trail node config, but no element of the rendered UI changes shape, color, or interaction.

**Design contract scope:** Confirm existing tokens apply. No new tokens needed.
