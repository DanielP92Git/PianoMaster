Implement BPM-adaptive timing calculator and formal IDLE/ONSET/SUSTAIN/COOLDOWN state machine in useMicNoteInput, replacing implicit frame-counting candidacy logic.

Purpose: The current detection pipeline uses hardcoded timing parameters (onFrames=4, offMs=140, minInterOnMs=80) that work at ~80 BPM quarter notes but break at extremes — eighth notes at 120 BPM get swallowed, quarter notes at 60 BPM trigger false note-off during sustain. The implicit state machine via variable counting has pitch flicker at note boundaries. This plan introduces dynamic timing and an explicit FSM to handle all note durations at 60-120 BPM.

**Step 1: Create `src/hooks/detectionTimingParams.js`**

Create a new file with a single exported pure function:

```javascript
const FRAME_MS = 16.7; // ~60fps rAF

export function computeTimingParams(bpm = 80, noteBeats = 1) {
  const safeBpm = Math.max(40, Math.min(200, bpm));
  const safeBeats = Math.max(0.25, noteBeats);
  const noteDurationMs = (safeBeats / safeBpm) * 60000;

  const onFrames = Math.max(2, Math.min(5, Math.round((noteDurationMs * 0.25) / FRAME_MS)));
  const offMs = Math.max(60, Math.min(500, Math.round(noteDurationMs * 0.4)));
  const minInterOnMs = Math.max(30, Math.min(250, Math.round(noteDurationMs * 0.3)));
  const changeFrames = Math.max(2, onFrames + 1);

  return { onFrames, offMs, minInterOnMs, changeFrames };
}
```

Key design decisions from research:
- `onFrames` minimum is 2 (not 1) to prevent single-frame false positives from noise
- `offMs` minimum is 60ms to handle piano string beating patterns (amplitude dips every 50-100ms)
- `minInterOnMs` minimum is 30ms as absolute floor for fastest passages
- `changeFrames` is always `onFrames + 1` for slightly more stability on legato transitions
- BPM clamped to 40-200, noteBeats clamped to minimum 0.25 (sixteenth note)

**Step 2: Refactor `src/hooks/useMicNoteInput.js` with FSM**

Replace the implicit candidacy logic with an explicit state machine. The hook signature adds two new optional props: `bpm` (number, default null) and `shortestNoteDuration` (number in beats, default null). When both are provided, `computeTimingParams(bpm, shortestNoteDuration)` overrides the preset timing values.

Define state constants at the top of the file:
```javascript
const STATE = { IDLE: 'idle', ONSET: 'onset', SUSTAIN: 'sustain', COOLDOWN: 'cooldown' };
```

Refactor `stateRef.current` to include:
```javascript
{
  machineState: STATE.IDLE,
  currentNote: null,
  candidateNote: null,
  candidateFrames: 0,
  candidateStartedAt: -Infinity,
  lastPitchAt: -Infinity,
  prevPitchAt: -Infinity,
  lastEmitAt: -Infinity,
  lastFrequency: -1,
  lastAudioLevel: 0,
  cooldownNote: null,
  cooldownUntil: -Infinity,
}
```

Store computed timing params in a ref (`timingParamsRef`) that updates when `bpm` or `shortestNoteDuration` change. When `bpm` is null/undefined, fall back to the prop values (onFrames, offMs, etc.) for backward compatibility.

```javascript
const timingParamsRef = useRef({ onFrames, offMs, minInterOnMs, changeFrames });

// Update timing params when BPM context changes
useEffect(() => {
  if (bpm != null && shortestNoteDuration != null) {
    timingParamsRef.current = computeTimingParams(bpm, shortestNoteDuration);
  } else {
    timingParamsRef.current = { onFrames, offMs, minInterOnMs, changeFrames };
  }
}, [bpm, shortestNoteDuration, onFrames, offMs, minInterOnMs, changeFrames]);
```

Rewrite `handlePitchDetected` as a switch on `s.machineState`, always reading timing from `timingParamsRef.current` (never from closure-captured props — this avoids Pitfall 3 from research):

- **IDLE**: New pitch → transition to ONSET, set candidateNote/candidateFrames=1
- **ONSET**: Same pitch → increment candidateFrames; if >= params.onFrames → emit noteOn, transition to SUSTAIN. Different pitch → reset candidate to new pitch, candidateFrames=1. No pitch (handled in handleLevelChange) → back to IDLE.
- **SUSTAIN**: Same pitch → refresh lastPitchAt. Different pitch → emit noteOff for current, transition to ONSET for new pitch (NOT to COOLDOWN — COOLDOWN only blocks same-note re-trigger per Pitfall 4).
- **COOLDOWN**: If `note !== s.cooldownNote` OR `now >= s.cooldownUntil` → transition to ONSET for the new pitch. If same note and still in cooldown → ignore.

Rewrite `handleLevelChange` silence detection:
- In SUSTAIN: if `now - s.lastPitchAt >= params.offMs` → emit noteOff, transition to COOLDOWN (set `cooldownNote = prev`, `cooldownUntil = now + params.minInterOnMs`)
- In ONSET: if `now - s.lastPitchAt >= params.offMs` → transition to IDLE (no noteOff since noteOn was never emitted)
- In COOLDOWN: if `now >= s.cooldownUntil` → transition to IDLE

Update `resetInternalState` to set `machineState: STATE.IDLE` and clear `cooldownNote`/`cooldownUntil`.

Remove `onFrames`, `offMs`, `minInterOnMs`, `changeFrames` from the `handlePitchDetected` useCallback dependency array — they are now read from timingParamsRef.current. Add `emit` only.
