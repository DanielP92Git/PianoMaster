| Property | Value |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-icon padding |
| sm | 8px | Badge internal padding, icon-to-text gap inside tabs |
| md | 16px | Default element spacing, tab bar horizontal padding |
| lg | 24px | Section padding, tab bar vertical padding |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks between trail units |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions:
- Tab button minimum hit target: `min-h-[48px]` (existing pattern from `TrailMap.jsx` line 277) — 48px is the active tab touch target, overrides the grid for accessibility
- Touch targets on ComingSoon back button: `min-h-[44px] min-w-[44px]` (defined in `tailwind.config.js` as `minHeight.touch`)

## Color

This phase introduces one new color identity. All others are pre-existing.

| Role | Value | Usage |
|------|-------|-------|

| Accent — Bass (existing) | `from-purple-500 to-violet-600` | Bass tab active state, Bass node backgrounds |
| Accent — Rhythm (existing) | `from-emerald-500 to-teal-600` | Rhythm tab active state, Rhythm node backgrounds |

| Destructive | `kidsError.500` #ef4444 | Destructive actions only — not used in this phase |

**Ear Training tab active color spec (sourced from D-01 in CONTEXT.md):**
- Active tab background: `bg-gradient-to-br from-cyan-400 to-teal-500` with `border-cyan-300`
- Active tab text: `text-white`
- Active tab glow: `shadow-[0_0_15px_rgba(34,211,238,0.4)]` (matching the glow pattern in `nodeTypeStyles.js`)
- Inactive tab text: `text-white/60` (matches existing inactive tab pattern)
- Node ring color (for future Phase 10): `rgba(34,211,238,0.85)` (cyan-400)
- Node glow color (for future Phase 10): `rgba(6,182,212,0.45)` (cyan-500)

**Accent reserved for:**
- Active tab indicator (background + border of the selected tab button)
- Ear icon tint when Ear Training tab is active
- ComingSoon screen icon accent ring (matches the category color of the game being shown)

**Tab icons (sourced from D-02, D-03 in CONTEXT.md):**
- Treble: `Music` (Lucide) — musical note, unambiguous for treble clef learning
- Bass: `Music2` (Lucide) — alternative note variant, distinguishable from Treble
- Rhythm: `Drum` (Lucide) — drum kit, child-recognizable rhythm metaphor
- Ear Training: `Ear` (Lucide) — direct mapping, decided in D-02

Icon size inside tab buttons: 16px × 16px (`h-4 w-4`)

## Component Inventory

### 1. TRAIL_TAB_CONFIGS array (in `src/data/constants.js`)

Each entry shape:
```js
{
  id: string,          // 'treble' | 'bass' | 'rhythm' | 'ear_training'
  label: string,       // display label, used as i18n key fallback
  categoryKey: string, // key into NODE_CATEGORIES
  icon: ReactComponent,// lucide-react component
  colorActive: string, // Tailwind gradient classes for active state
  colorBorder: string, // Tailwind border class for active state
  colorGlow: string,   // Tailwind shadow class for active state
  bossPrefix: string,  // e.g. 'boss_treble' — for boss node filtering
}
```

**All four entries fully declared:**

| id | label | categoryKey | icon | colorActive | colorBorder | bossPrefix |
|----|-------|-------------|------|-------------|-------------|------------|
| treble | Treble | TREBLE_CLEF | Music | from-blue-500 to-indigo-600 | border-blue-400 | boss_treble |
| bass | Bass | BASS_CLEF | Music2 | from-purple-500 to-violet-600 | border-purple-400 | boss_bass |
| rhythm | Rhythm | RHYTHM | Drum | from-emerald-500 to-teal-600 | border-emerald-400 | boss_rhythm |
| ear_training | Ear Training | EAR_TRAINING | Ear | from-cyan-400 to-teal-500 | border-cyan-300 | boss_ear |

### 2. ComingSoon component (new, in `src/components/shared/ComingSoon.jsx`)

**Props:**
- `gameName: string` — display name of the game (e.g. "Rhythm Tap", "Pitch Comparison")
- `onBack: function` — callback to navigate back to the trail

**Layout:** Full-screen centered column on the trail forest background. Not a modal — a standalone page rendered via routing.

**Visual spec:**
- Background: inherits trail page background (`bg-gradient-to-br from-slate-900 via-indigo-950 to-slate-900`) — same dark context as other trail-launched game screens
- Card: `bg-white/10 backdrop-blur-md border border-white/20 rounded-2xl` — glass card pattern, max-width 400px, centered, padding 32px
- Icon: `Hourglass` (Lucide), 48px × 48px, `text-cyan-300` — neutral "waiting" metaphor, not category-specific (placeholder for all 5 types)
- Game name heading: 24px (text-2xl), weight 700, `text-white`
- Subtitle: "This game is coming soon! Check back after the next update." — 16px, `text-white/70`
- Back button: `bg-white/10 border border-white/20 hover:bg-white/20 text-white rounded-full px-6 py-3 text-sm font-bold` — ghost glass style, min-height 44px
- Back button label: "Back to Trail"

**Interaction:**
- `onBack` fires immediately on button tap — no confirmation required
- No loading state — static screen
- Supports RTL: text-align follows document direction

### 3. Updated TrailMap tab bar (refactored from hardcoded `TRAIL_TABS`)

- Tab bar renders via `.map()` over `TRAIL_TAB_CONFIGS`
- Each tab button: icon (16px) + label text + progress counter (existing 10px pattern)
- Icon renders above label text within the button column layout
- Active tab: gradient background + border + glow matching config entry
- Inactive tab: `bg-transparent border border-white/10 text-white/60 hover:text-white/80 hover:border-white/20`
- ARIA: `role="tablist"`, each button `role="tab"`, `aria-selected`, keyboard navigation RTL-aware (existing pattern preserved)

## Accessibility Notes

- Tab `role="tablist"` / `role="tab"` / `role="tabpanel"` pattern already implemented — preserve exactly
- ComingSoon back button must be focusable and operable by keyboard (not just touch)
- Icon-only tab rendering would fail WCAG 2.1 SC 1.1.1 — tabs must include visible text label alongside icon
- Reduced motion: ComingSoon screen has no animation. Tab glow uses `shadow-` (no `animation-` class) — safe for reduced-motion users
- RTL support: ComingSoon layout uses `flex-col items-center` — direction-agnostic. Back button text uses i18n key.
