    ```sql
    -- Phase 11: Reset exercise_progress and stars for all remapped rhythm nodes
    -- All rhythm nodes are being remapped from single EXERCISE_TYPES.RHYTHM to mixed exercise types.
    -- Stale single-exercise progress records would conflict with the new exercise type assignments.
    -- DEPLOY CONSTRAINT: This migration MUST run before updated rhythmUnit*.js files deploy to production.

    UPDATE public.student_skill_progress
    SET
      exercise_progress = '[]'::jsonb,
      stars = 0,
      best_score = NULL
    WHERE node_id LIKE 'rhythm_%'
       OR node_id LIKE 'boss_rhythm_%';
    ```

    This migration:
    - Resets `exercise_progress` JSONB to empty array (clears stale single-exercise progress)
    - Resets `stars` to 0 (stars will be re-earned with new exercise types)
    - Resets `best_score` to NULL
    - Targets all `rhythm_X_Y` nodes (48 non-boss) and `boss_rhythm_X` nodes (8 boss)
    - Uses LIKE patterns to match both naming conventions

    **CRITICAL deploy constraint (per STATE.md):** This migration must be applied via `supabase db push` or Supabase dashboard BEFORE the updated JS files deploy to production. The executor should note this in the SUMMARY for the deployment checklist.
