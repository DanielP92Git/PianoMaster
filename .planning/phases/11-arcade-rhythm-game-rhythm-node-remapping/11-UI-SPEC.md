## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tile gap between adjacent falling tiles |
| sm | 8px | In-tile padding, feedback pill padding |
| md | 16px | Header padding, game area horizontal padding (`px-4`) |
| lg | 24px | Section spacing, bottom safe area pad |
| xl | 32px | Game canvas top padding |
| 2xl | 48px | Hit zone height |
| 3xl | 64px | Tile height for quarter notes |

Exceptions:
- Touch targets (hit zone button): minimum 44px height — use `min-h-touch` Tailwind token (already defined in `tailwind.config.js`)
- Heart icons tappable area: 44px x 44px minimum (display-only, no interaction needed)
- Tile widths are 100% of the single lane (full viewport width minus horizontal padding)

Source: `tailwind.config.js` spacing tokens, `docs/DESIGN_SYSTEM.md`, CONTEXT.md D-01.

## Component Inventory

### ArcadeRhythmGame — Layout Structure

The game runs full-screen in landscape orientation (same as RhythmReadingGame). AppLayout hides sidebar/header for this route.

```
fixed inset-0
  bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900
  flex flex-col
  ├── <header> h-12 px-4 py-2 flex items-center justify-between
  │     text-white/80 text-sm font-medium
  │     [BackButton] [exercise counter] [lives hearts] [combo badge]
  ├── <main> flex-1 relative overflow-hidden
  │     [tile lane — full width, tiles descend via rAF transform]
  │     [hit zone line — absolute bottom, glowing horizontal line]
  │     [FloatingFeedback — absolute positioned, reuse existing component]
  │     [CountdownOverlay — fixed inset-0, reuse existing component]
  └── (VictoryScreen / GameOverScreen replace entire view on session end)
```

### Tile Lane

- Single column: `w-full` relative container, `overflow-hidden`
- Each tile: `absolute` positioned, translated via `ref.style.transform` (rAF, no React state updates per frame)
- Tile width: `w-full` (fills lane)
- Tile height by duration:
  - Whole note: 80px (`h-20`)
  - Half note: 64px (`h-16`)
  - Quarter note: 56px (`h-14`)
  - Eighth note: 44px (`h-11`)
- Tile border-radius: `rounded-xl` (12px)
- Ghost tile (rest): same height as corresponding duration, `bg-white/15 border border-dashed border-white/30`

### Hit Zone Line

- `absolute bottom-0 left-0 right-0` with height `h-12` (48px) as tappable zone
- Visual indicator: a 3px horizontal glowing line at vertical center of hit zone
- Default glow: `box-shadow: 0 0 12px 4px rgba(99, 102, 241, 0.8)` (indigo)
- On-fire glow: `box-shadow: 0 0 16px 6px rgba(249, 115, 22, 0.9)` (orange)
- The hit zone `<div>` is the full-screen tap target (CONTEXT.md D-01: tap anywhere)
- The visual line uses a separate absolutely-positioned child element

### Lives Display (Header)

- 3 heart icons in header row, right-aligned group
- Full heart: `<Heart className="h-5 w-5 fill-red-400 text-red-400" />` (reuse from NotesRecognitionGame)
- Empty heart: `<Heart className="h-5 w-5 text-white/30" />`
- On life loss: brief screen shake on the main container — CSS keyframe `@keyframes shake` applied as class for 400ms, skipped if `reducedMotion`

### Combo Badge (Header)

- Shown when combo >= 2
- `bg-yellow-400/20 border border-yellow-400/40 rounded-full px-2 py-0.5 text-yellow-300 text-sm font-bold`
- On-fire mode: badge changes to `bg-orange-400/20 border border-orange-400/40 text-orange-300` with flame icon prepended
- Combo threshold for on-fire activation: 5 (matches `ON_FIRE_THRESHOLD` in NotesRecognitionGame)

### FloatingFeedback (reused)

Reuse `src/components/games/rhythm-games/components/FloatingFeedback.jsx` without modification.
- Positioned above hit zone via `bottom: hitZoneHeight + 8px`
- PERFECT: `text-green-400 text-3xl font-bold`
- GOOD: `text-yellow-400 text-3xl font-bold`
- MISS: `text-red-400 text-3xl font-bold`

### CountdownOverlay (reused)

Reuse `src/components/games/rhythm-games/components/CountdownOverlay.jsx` without modification.
- Numbers (3, 2, 1): `text-yellow-400 text-3xl font-bold animate-pulse`
- GO: `text-green-400 text-3xl font-bold animate-pulse`
- First tiles are visible and slowly descending behind the overlay (opacity dimmed on overlay: `bg-black/30`)

### On-Fire Mode Visual State

Activated at combo >= 5 (CONTEXT.md D-06).
When active:
- Tile glow trail: each descending tile gets `drop-shadow-[0_0_12px_rgba(249,115,22,0.8)]` orange glow
- Hit zone line color transitions from indigo to orange (150ms transition)
- Combo badge switches to orange variant (see above)
- If `reducedMotion`: no glow animation, no trail — only color change on hit zone line and badge

## Motion Contract

All motion must check `reducedMotion` from `AccessibilityContext`.

| Interaction | Default animation | Reduced-motion fallback |
|-------------|-------------------|------------------------|
| Tile descent | `requestAnimationFrame` + `translateY` | Same (physics-based, cannot remove — gameplay) |
| Hit zone pulse | `animate-pulse` (2s cubic) | No pulse, static line |
| FloatingFeedback float-up | `translateY(-40px)` 800ms ease-out | `translateY(0)` — text appears/disappears only |
| PERFECT tile burst | CSS `@keyframes burst` particle spread | No burst — tile simply disappears |
| Screen shake on miss | CSS `@keyframes shake` 400ms | No shake |
| Countdown pulse | `animate-pulse` | No pulse, static number |
| On-fire tile trail | `drop-shadow` orange glow | No shadow — tile color only |
| Tile dissolve on GOOD | opacity 0 over 200ms | Immediate opacity 0 |

Source: CONTEXT.md D-05, D-06, D-07, D-08. Project convention from `NotesRecognitionGame.jsx`.

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Idle (before start) | Initial render | Tap instruction shown in hit zone area |
| Countdown | Game phase = READY | CountdownOverlay covers screen, tiles begin descending slowly |
| Playing | Countdown ends | Tiles descend at game speed, hit zone active |
| PERFECT hit | Tap within perfect window | Green FloatingFeedback, tile burst particles, combo++ |
| GOOD hit | Tap within good window | Yellow FloatingFeedback, tile dissolve, combo++ |
| MISS | Tile exits hit zone without tap | Red FloatingFeedback, tile grays out (`bg-white/15`), combo reset, life-- |
| On-fire | Combo >= 5 | Orange glow trail on tiles, orange hit zone line, orange combo badge |
| Life lost | MISS decrements to 0 lives | Screen shake (unless reducedMotion), heart icon drains |
| Game over | Lives = 0 | GameOverScreen replaces view |
| Session complete | 10 patterns scored | VictoryScreen replaces view |
| Audio interrupted | iOS AudioContext suspended | AudioInterruptedOverlay appears (existing component) |
