**3. New `frequencyToNote` function (ALGO-03):**
```javascript
function frequencyToNote(hz) {
  if (!hz || hz <= 0) return null;
  const midi = Math.round(12 * Math.log2(hz / 440) + 69);
  if (midi < MIN_MIDI || midi > MAX_MIDI) return null; // C3 to C6 guard
  const octave = Math.floor(midi / 12) - 1;
  const name = NOTE_NAMES[midi % 12];
  return `${name}${octave}`;
}
```
This is a module-level pure function, NOT a useCallback — it has no dependencies. Export it for reuse.

**6. Detection loop rewrite (ALGO-01, ALGO-02):**
Replace the autocorrelation `detectPitch` callback with pitchy-based detection:
```javascript
const detectLoop = () => {
  if (!currentAnalyser) return;

  // Get time-domain data (full fftSize, not frequencyBinCount)
  currentAnalyser.getFloatTimeDomainData(inputBufferRef.current);

  // Calculate RMS for audio level
  let sum = 0;
  const len = inputBufferRef.current.length;
  for (let i = 0; i < len; i++) {
    sum += inputBufferRef.current[i] * inputBufferRef.current[i];
  }
  const level = Math.sqrt(sum / len);
  setAudioLevel(level);
  if (onLevelChange) onLevelChange(level);

  // McLeod Pitch Method via pitchy (ALGO-01)
  const [pitch, clarity] = detectorRef.current.findPitch(
    inputBufferRef.current,
    currentSampleRate
  );

  // ALGO-02: Clarity gate — reject weak/ambiguous detections
  if (clarity >= clarityThreshold && pitch > 0) {
    const note = frequencyToNote(pitch);
    setDetectedFrequency(pitch);
    setDetectedNote(note);
    if (onPitchDetected && note) {
      onPitchDetected(note, pitch);
    }
  } else {
    setDetectedFrequency(-1);
    setDetectedNote(null);
    // Still call onLevelChange (already done above) — level is useful even without pitch
  }

  animationFrameRef.current = requestAnimationFrame(detectLoop);
};
```

**7. Dual-mode startListening (ARCH-02 backward compat):**

**CRITICAL — Call-time override support:**
`startListening` must accept an optional argument for call-time overrides:
```javascript
const startListening = useCallback(async (overrides = {}) => {
  const {
    analyserNode: callTimeAnalyser = null,
    sampleRate: callTimeSampleRate = null,
  } = overrides;

  // Resolve effective analyser: call-time arg > hook prop > null (fallback)
  const effectiveAnalyser = callTimeAnalyser || analyserNode;
  const effectiveSampleRate = callTimeSampleRate || sampleRate;
  // ...
```
This is required because the hook-level `analyserNode` prop is evaluated at render time, which may be null if the provider's `requestMic()` hasn't completed yet. Callers like NotesRecognitionGame (Plan 04) will call:
```javascript
const { analyser, audioContext: ctx } = await requestMic();
await startMicListening({ analyserNode: analyser, sampleRate: ctx.sampleRate });
```
The call-time argument takes priority over the hook-level prop. If neither is provided, fallback mode activates.

Mode A — Shared analyserNode available (from call-time arg OR hook prop):
- Skip getUserMedia and AudioContext creation entirely
- Use effectiveAnalyser directly
- Use effectiveSampleRate (or audioContextRef if available)
- Create PitchDetector for the provided analyser's fftSize
- Start detectLoop immediately

Mode B — No analyserNode at all (fallback, preserves existing tests):
- Call getUserMedia with DSP-disabled constraints (matching AudioContextProvider: echoCancellation: false, etc.)
- Create own AudioContext
- Create own AnalyserNode with fftSize 4096, smoothingTimeConstant 0.0
- Create PitchDetector
- Start detectLoop
- Store audioContext and stream in state for cleanup
