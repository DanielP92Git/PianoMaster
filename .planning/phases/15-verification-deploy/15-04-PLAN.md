## Issue 1: Auto-Advance Too Fast After Wrong Answer

**Root cause:** In RhythmDictationGame.jsx lines 318-331, after a wrong answer:
- T+300ms: correct card revealed + auto-replay starts via `playPattern(correctBeats, tempo, () => {})`
- T+2000ms: `advanceQuestion()` fires (hardcoded timeout)

The replay duration depends on tempo. At 90 BPM, a 4/4 measure takes ~2.67s. The 2000ms timeout fires BEFORE the replay finishes, cutting it short.

Additionally, `feedbackTimeoutRef.current` is overwritten at line 329 — the 2000ms timeout replaces the 300ms timeout ref, so clearTimeout in cleanup only clears the later one.

**Fix:** After the 300ms reveal, calculate the actual replay duration from `schedulePatternPlayback`'s return value. Set the advance timeout to fire AFTER the replay completes (replay duration + buffer).

Better: wait for the playPattern callback (which fires when replay finishes) instead of guessing timing:
```javascript
// At line 325, change:
playPattern(correctBeats, tempo, () => {});
// To:
playPattern(correctBeats, tempo, () => {
  // Advance 1s after replay finishes (breathing room)
  feedbackTimeoutRef.current = setTimeout(() => advanceQuestion(), 1000);
});
```
And REMOVE the hardcoded 2000ms timeout at lines 329-331.

## Issue 3: Sound Unification

**Root cause:** RhythmDictationGame uses `usePianoSampler().playNote('C4')` for pattern playback — this is oscillator synthesis (sine + triangle harmonics). MetronomeTrainer uses `audioEngine.createPianoSound()` which loads G4.mp3, a real piano sample.

**Fix:** In RhythmDictationGame:
1. Import `useAudioEngine` hook
2. Initialize: `const audioEngine = useAudioEngine(tempo, { sharedAudioContext: audioContextRef.current });`
3. In `schedulePatternPlayback` (rhythmTimingUtils.js), the `playNote` callback is passed in — so just pass `audioEngine.playNote` instead of `usePianoSampler().playNote`. But `schedulePatternPlayback` calls `playNote('C4', { startTime, duration })` — need to verify audioEngine's playNote accepts same signature.

Actually, checking the audioEngine: `createPianoSound()` is used in MetronomeTrainer for pattern playback (line 676). But `schedulePatternPlayback` in rhythmTimingUtils.js takes a generic `playNote` callback.

Best approach: Create a wrapper in RhythmDictationGame that uses audioEngine for playback:
```javascript
const enginePlayNote = useCallback((note, opts) => {
  audioEngine.createPianoSound(opts?.startTime);
}, [audioEngine]);
```

Then pass `enginePlayNote` to `playPattern` instead of `playNote` from usePianoSampler.

Actually, let me check what MetronomeTrainer passes to its pattern playback... It uses `audioEngine.createPianoSound(time)` directly in its own scheduling loop, not through `schedulePatternPlayback`. So the simplest fix is:

In `rhythmTimingUtils.js` `schedulePatternPlayback()`, the `playNote` parameter is a generic callback. RhythmDictationGame currently passes `usePianoSampler().playNote`. Just change what's passed — pass an audioEngine-based callback instead.

## Current schedulePatternPlayback (rhythmTimingUtils.js lines 234-251):
```javascript
export function schedulePatternPlayback(beats, tempo, audioContext, playNote, explicitStartTime) {
  const beatDuration = 60 / tempo;
  const sixteenthDuration = beatDuration / 4;
  const startTime = explicitStartTime ?? (audioContext.currentTime + 0.1);
  let offset = 0;
  beats.forEach((beat) => {
    if (!beat.isRest) {
      playNote('C4', {
        startTime: startTime + offset,
        duration: sixteenthDuration * beat.durationUnits * 0.8,
      });
    }
    offset += sixteenthDuration * beat.durationUnits;
  });
  return { startTime, totalDuration: offset };
}
```
