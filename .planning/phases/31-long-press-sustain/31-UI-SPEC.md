| State                            | Visual                                                                     |
| -------------------------------- | -------------------------------------------------------------------------- |
| Tap mode (default, quarter note) | Existing: `bg-white/10`, "TAP HERE" label, `onClick` path unchanged        |
| Hold mode idle                   | `bg-white/10`, "HOLD" badge visible, ring outline at 0% fill               |
| Hold mode active (pressing)      | `bg-white/15`, ring arc filling clockwise in `indigo-400`                  |

| Disabled                         | `opacity-50 cursor-not-allowed` (existing pattern, unchanged)              |

Event handling:

- Hold notes: `onPointerDown` → `setPointerCapture` → press start; `onPointerUp` / `onPointerCancel` → press end
- Tap notes: `onClick` path unchanged — no regression to existing quarter-note behavior
- `touch-action: none` on the button element to prevent scroll-cancel during hold

### New: `HoldRing.jsx`

Location: `src/components/games/rhythm-games/components/HoldRing.jsx`

Purpose: SVG ring progress indicator overlaid on TapArea. Driven by direct DOM ref mutation (not React state) at 60fps via `requestAnimationFrame` to avoid re-render jank on low-end tablets.

Visual spec:

- SVG canvas: 100x100px, `className="absolute inset-0 pointer-events-none"`
- Rotated -90deg so fill starts from 12-o'clock: `style={{ transform: 'rotate(-90deg)' }}`
- Track circle: `r=44`, `stroke="rgba(255,255,255,0.20)"`, `strokeWidth=4`, no fill
- Progress circle: `r=44`, `stroke="#818cf8"` (indigo-400, in-progress) or `"#4ade80"` (green-400, complete), `strokeWidth=4`, `strokeLinecap="round"`, `strokeDasharray=276` (circumference), `strokeDashoffset` updated via ref
- Progress formula: `strokeDashoffset = 276 * (1 - progress)`

Reduced-motion behavior (when `reducedMotion === true`):

- Skip ring animation entirely
- On press-start: set button background to `bg-white/20` (opacity highlight)
- On PERFECT complete: flash `bg-green-400/30` for 200ms then return to `bg-white/10`
- This matches the existing `beatPulse` reduced-motion pattern in `PulseQuestion.jsx` (line 598)

Performance constraint: Update `strokeDashoffset` via `ringCircleRef.current.setAttribute('stroke-dashoffset', value)` — NOT via React state. Store only `isComplete: boolean` as React state for the color switch.

### Modified: `PulseQuestion.jsx` — Stretched Beat Indicator (D-08)

Location: `src/components/games/rhythm-games/renderers/PulseQuestion.jsx`

When a beat has `durationUnits >= 8`, the beat dot in the beat row spans multiple columns:

### New: `holdScoringUtils.js`

Location: `src/components/games/rhythm-games/utils/holdScoringUtils.js`

Exports:

- `scoreHold(actualHoldMs, requiredHoldMs)` → `"PERFECT" | "GOOD" | "MISS"`
  - `>= 70%` → PERFECT
  - `40–69%` → GOOD
  - `< 40%` → MISS
- `isHoldNote(durationUnits)` → boolean (`durationUnits >= 8`)
- `calcHoldDurationMs(durationUnits, tempo)` → `(durationUnits / 4) * (60000 / tempo)`

## Interaction Contract

### Hold Note Lifecycle (full sequence)

1. **Before press:** TapArea shows "HOLD" badge + ring outline at 0% fill
2. **Press-down (`onPointerDown`):**
   - `setPointerCapture(e.pointerId)` — prevents missed release on finger slide-off
   - `pressStartTimeRef.current = audioEngine.getCurrentTime()`
   - Start `requestAnimationFrame` loop updating ring progress
   - Call `audioEngine.createPianoSound(pressTime, 0.8, noteFullDurationSeconds)` — sound plays for full note duration regardless of actual hold time
3. **While holding:** Ring arc fills clockwise from 0% to 100% in real time
4. **Release (`onPointerUp` / `onPointerCancel`):**
   - Stop rAF loop
   - Compute `holdMs = (releaseTime - pressStartTime) * 1000`
   - Call `scoreHold(holdMs, requiredHoldMs)` → tier
   - Show feedback label (PERFECT / "Almost, hold longer!" / MISS)
   - If PERFECT: green flash on ring (200ms), then reset
5. **After evaluation:** Reset `holdProgress` to 0, clear `pressStartTimeRef`, advance to next onset

### Quarter Note Path (unchanged)

Quarter notes (`durationUnits === 4`) bypass the hold path entirely. `TapArea` shows "TAP HERE" (existing label), `onClick` fires `handleTap()` as before. No regression.

### Pointer Capture Contract

All hold notes use `e.currentTarget.setPointerCapture(e.pointerId)` on `onPointerDown`. This ensures `onPointerUp` fires on the element even if the child's finger slides outside the boundary. `onPointerCancel` also triggers press-end (handles OS interruptions like notifications).
