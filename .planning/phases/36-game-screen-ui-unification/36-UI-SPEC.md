## Spacing Scale

Declared values match the CSS custom properties in `src/index.css` and `tailwind.config.js`:

| Token | Value | Usage                                                 |
| ----- | ----- | ----------------------------------------------------- |
| xs    | 4px   | Icon gaps (`gap-1`), inline chip padding              |
| sm    | 8px   | Compact HUD spacing (`gap-2`, `px-3 py-1.5` on pills) |
| md    | 16px  | Default element spacing, card inner padding (`p-4`)   |
| lg    | 24px  | Section padding, card generous padding (`p-6`)        |
| xl    | 32px  | Layout gaps                                           |
| 2xl   | 48px  | Major section breaks                                  |
| 3xl   | 64px  | Page-level spacing                                    |

Exceptions:

- **Touch targets:** `min-h-touch` (44px) on all interactive HUD buttons — enforced via `minHeight.touch` Tailwind token. Pause/Settings button: `px-3 py-2` minimum.
- **Progress bar height:** 16px (`h-4`) — not a standard spacing step; this is a fixed visual height.
- **Checkpoint dots:** 10px (`h-2.5 w-2.5`) — fractional size, visually tuned to the 16px bar.
- **Heart icons:** 20px mobile (`h-5 w-5`), 24px sm+ (`sm:h-6 sm:w-6`) — responsive icon sizing.
- **Flame badge (OnFireBadge):** 40px (`h-10 w-10`) inline, 96px/112px (`h-24 w-24 sm:h-28 sm:w-28`) for OnFireSplash full-screen overlay.

**Source:** `src/index.css` `:root` spacing vars, NotesRecognitionGame.jsx class inspection

## Typography

All HUD components render on the dark glassmorphism surface. No light-mode fallback needed for game screens.

**Declared sizes (4):** 12px · 14px · 16px · 30px  
**Declared weights (2):** semibold (600) · bold (700)

| Role            | Size                                     | Weight         | Line Height | Usage                                                             |
| --------------- | ---------------------------------------- | -------------- | ----------- | ----------------------------------------------------------------- |
| HUD label       | 12px (`text-xs`) → 14px sm (`text-sm`)   | semibold (600) | 1.5         | "XP", "Time", "Combo", progress text                              |
| HUD value       | 14px (`text-sm`) → 16px sm (`text-base`) | bold (700)     | 1.6         | Score number, combo count, timer digits                           |
| Feedback badge  | 14px (`text-sm`) → 16px sm (`text-base`) | bold (700)     | 1.5         | "FAST!", "New note unlocked!"                                     |
| Tier-up display | 30px (`text-3xl`) at all viewports       | bold (700)     | 1.2         | "DOUBLE XP!", "TRIPLE XP!" — size alone provides visual hierarchy |

Note on tier-up display: the reference implementation uses a responsive `text-3xl sm:text-4xl` pair, but the 30→36px delta is marginal and the large size at 30px already dominates the overlay. The contract declares `text-3xl` (30px) at all viewports to cap the declared size count at 4. Executor must NOT introduce `text-4xl` on tier-up elements.

Monospace font for numeric values: `font-mono tracking-wide` — preserves fixed-width digits so the score pill does not reflow as numbers change.

Progress counter: `text-xs font-semibold text-white/75` — "Question N of N".

**Source:** NotesRecognitionGame.jsx `TimerDisplay`, `ProgressBar`, `ScorePill` JSX inline, tier-up popup

## Color

### Game Screen Background (60%)

| State   | Value                                                | Tailwind equivalent                                              |
| ------- | ---------------------------------------------------- | ---------------------------------------------------------------- |
| Default | `linear-gradient(135deg, #312e81, #581c87, #4c1d95)` | `bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900` |
| On-fire | `linear-gradient(135deg, #713f12, #854d0e, #713f12)` | amber-900 variant — inline style only                            |

Transition between states: `transition: background 0.6s ease-in-out` — must be skipped when `reducedMotion` is active.

Background accent blurs (decorative, not interactive):

- Top-left: `h-72 w-72 bg-indigo-500/15 blur-3xl`
- Bottom-right: `h-96 w-96 bg-fuchsia-500/10 blur-3xl`
- Center: `h-[520px] w-[520px] bg-white/5 blur-3xl`

### Glass Surface (30%)

All HUD pill containers, StageCard wrappers, and overlay cards use the glass card pattern:

| Layer          | Classes                                               |
| -------------- | ----------------------------------------------------- |
| Primary glass  | `bg-white/10 backdrop-blur-md border border-white/20` |
| Nested element | `bg-white/5 border-white/10`                          |
| Hover state    | `hover:bg-white/20`                                   |

These surfaces carry ~30% of the visual weight (card containers, HUD row, answer grid background).

### Accent / Semantic (10%)

The 10% accent budget is split between:

| Semantic role     | Color                                                       | Reserved for                                                        |
| ----------------- | ----------------------------------------------------------- | ------------------------------------------------------------------- |
| Progress fill     | `from-indigo-400 via-violet-400 to-fuchsia-400`             | ProgressBar fill only                                               |
| Correct answer    | `bg-emerald-500/80 border-emerald-300/40`                   | Answer button correct state only                                    |
| Wrong answer      | `bg-rose-500/80 border-rose-300/40`                         | Answer button wrong state only                                      |
| Combo tier-2 (≥3) | `bg-amber-500/15 border-amber-400/30` + `text-amber-300`    | ScorePill + ComboPill tint at combo ≥ 3                             |
| Combo tier-3 (≥8) | `bg-yellow-500/20 border-yellow-400/40` + `text-yellow-300` | ScorePill + ComboPill tint at combo ≥ 8                             |
| Speed bonus badge | `bg-amber-400/20 text-amber-300`                            | SpeedBonusFlash text badge only                                     |
| New note badge    | `bg-emerald-400/20 text-emerald-300`                        | New-note unlock banner only (stays NotesRecognition-specific, D-04) |
| Tier-up popup     | `bg-gradient-to-br from-amber-500/90 to-yellow-500/90`      | TierUpPopup card only                                               |
| On-fire glow      | `drop-shadow-[0_0_16px_rgba(251,146,60,0.6)]`               | OnFireSplash image only                                             |
| Active hearts     | `fill-red-400 text-red-400`                                 | LivesDisplay active hearts only                                     |
| Spent hearts      | `text-white/30 opacity-30`                                  | LivesDisplay empty heart slots only                                 |

Accent reserved for: progress fill, answer feedback, combo/score tinting, speed bonus flash, tier-up popup, on-fire effects, lives hearts. Never apply combo-tint or on-fire coloring to structural layout elements.

### Text Colors

| Role           | Value           | Usage                       |
| -------------- | --------------- | --------------------------- |
| Primary        | `text-white`    | All HUD values and headings |
| Secondary      | `text-white/80` | HUD labels ("XP", "Time")   |
| Tertiary       | `text-white/75` | Progress counter text       |
| Disabled/spent | `text-white/30` | Spent heart icons           |

| Role        | Value                               | Usage                             |
| ----------- | ----------------------------------- | --------------------------------- |
| Destructive | `bg-rose-500/80 border-rose-300/40` | Wrong-answer button feedback only |

No destructive confirmation dialogs in this phase. The exit/back navigation (`BackButton`) is non-destructive — it navigates without a confirmation prompt.

## Visual Hierarchy

Primary focal point on the game screen: the note-display / answer area at center (largest interactive region, occupies the majority of vertical space).

Secondary: ProgressBar at top of the screen — spans full width, always visible, communicates session advancement.

Tertiary: HUD stat pills (score, combo, lives, timer) — compact, fixed height, visually recessive relative to the answer area.

Decorative: Background blur orbs and on-fire background shift — never compete with the primary focal point.

This hierarchy must be preserved when shared HUD components are adopted into other games. The answer/content area of each game remains the primary focal point; HUD elements must not grow in prominence.

## Animation Contract

All animations must check `reducedMotion` from `useAccessibility()` (AccessibilityContext) internally. This is an encapsulated concern — parent games never pass a `reducedMotion` prop (D-10).

| Animation             | Normal behavior                                         | Reduced-motion behavior             |
| --------------------- | ------------------------------------------------------- | ----------------------------------- |
| ProgressBar fill      | `motion.div scaleX` via `useMotionTokens().soft` spring | No-op (Framer `initial={false}`)    |
| Floating +score       | `y: -28, opacity: 0` over 0.55s                         | `opacity: 0` only (no translate)    |
| Combo shake (wrong)   | `x: [0,-6,6,-4,4,0]` 0.22s tween                        | Skip `animate` prop                 |
| Combo scale (correct) | `scale: [1, 1.18, 1]` 0.22s tween                       | Skip `animate` prop                 |
| Heart exit            | `scale: [1,1.4,0] opacity: [1,1,0]` 0.3s                | `undefined` transition              |
| Heart enter           | `scale: 0, opacity: 0` → default                        | Skip initial                        |
| OnFireBadge           | fade + scale in/out 0.3s, `animate-pulse` active        | `animate-pulse` disabled, fade only |
| OnFireSplash          | scale bounce `[1, 1.15, 1]` + fade 0.6s                 | No scale, opacity change only       |
| SpeedBonusFlash       | scale `[0.8→1→0.8]` + fade 0.35s                        | `opacity: 0` only                   |
| TierUpPopup fly       | scale `[0.5, 1, 1, 0.3]` + position fly to target 1.2s  | opacity `[1, 1, 0]` over 1.2s       |
| On-fire bg shift      | `transition: background 0.6s ease-in-out`               | Skip transition entirely            |

CSS utility: `motion-reduce:transition-none` is applied on pill `transition-colors` classes as a secondary guard.

**Source:** NotesRecognitionGame.jsx Framer Motion `animate`/`transition` props, `useMotionTokens()` usage

## RTL / Hebrew Parity

HUD components must render correctly in RTL layout (`dir="rtl"` when `i18n.language === "he"`).

| Rule                      | Implementation                                                                                                    |
| ------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Pills and icon-text pairs | Use `dir="ltr"` on numeric-only elements (score, combo count, timer digits) — numbers read L-to-R in both locales |
| Progress bar              | Always fills left-to-right regardless of locale (`origin-left` is invariant)                                      |
| Heart row                 | Flex row order is visual; `flex-row-reverse` not needed — hearts are symmetric                                    |
| Floating +score           | Absolute centered (`left-1/2 -translate-x-1/2`); locale-neutral                                                   |
| Tier-up popup             | Centered fixed overlay; locale-neutral                                                                            |
| BackButton / nav          | Already RTL-aware in its existing implementation                                                                  |

**Source:** CLAUDE.md (Accessibility section), 36-SPEC.md Constraints, NotesRecognitionGame.jsx `isRTL` usage

## StageCard Pattern

HUD rows and content cards wrap in the `StageCard` pattern (extracted inline component in NotesRecognitionGame):

```
rounded-3xl border border-white/20 bg-white/10 p-4 shadow-none
```

The inner content layer removes backdrop-blur from StageCard itself to avoid GPU flicker artifacts (noted in source code comment). Backdrop-blur is applied per-pill instead.
