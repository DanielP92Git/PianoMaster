  <action>
Apply three targeted one-line fixes. Do NOT extract shared utilities — two files, three lines, each self-contained.

**Fix 1 — TrailNodeModal.jsx line 172:**
Change:
```javascript
const enableFlats = notePool.some(n => n.includes('b'));
```
To:
```javascript
const enableFlats = notePool.some(n => /^[A-G]b\d/.test(n));
```
Regex: `^[A-G]` = starts with note letter, `b` = flat symbol, `\d` = octave digit. Matches Bb3, Eb4, Ab3, Db3. Does NOT match B3, B4.

**Fix 2 — NotesRecognitionGame.jsx line 396 (filterAutoGrowCandidates):**
Change:
```javascript
return candidatePool.filter(pitch => !/[#b]/.test(pitch));
```
To:
```javascript
return candidatePool.filter(pitch => !/[#]|[A-G]b/.test(pitch));
```
Separates sharp detection (`[#]`) from flat detection (`[A-G]b`). The `[A-G]b` pattern requires a note letter before lowercase 'b', so 'B3' does not match (no letter before uppercase B followed by lowercase b).

**Fix 3 — NotesRecognitionGame.jsx line 903 (currentPoolHasAccidentals fallback):**
Change:
```javascript
(currentNode.noteConfig?.notePool || []).some(p => /[#b]/.test(p));
```
To:
```javascript
(currentNode.noteConfig?.notePool || []).some(p => /[#]|[A-G]b/.test(p));
```
Same pattern as Fix 2. Ensures consistency across all three detection sites. This fallback is only reached for non-trail launches, but fixing it eliminates the last instance of the ambiguous pattern.

After all three fixes, run the full test suite to verify no regressions.
