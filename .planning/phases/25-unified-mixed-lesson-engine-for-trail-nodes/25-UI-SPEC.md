| Token | Value | Usage                                                 |
| ----- | ----- | ----------------------------------------------------- |
| xs    | 4px   | Icon gaps, progress dot spacing                       |
| sm    | 8px   | Card inner padding, inline gap between elements       |

### 2. Progress Bar

**Contract:**

- Track: `h-2 rounded-full bg-white/15 overflow-hidden` ŌĆö same glass track as existing progress bars (CLAUDE.md)
- Fill: `h-2 rounded-full bg-green-400` ŌĆö width set as inline style: `width: {(currentIndex / total) * 100}%`
- Width transition: `transition-[width] duration-300 ease-in-out` ŌĆö skipped when `reducedMotion` is true
- Fraction label: `text-sm font-bold text-white/70` at right end of top bar ŌĆö `"3/10"` format (never "3 of 10")
- ARIA: `role="progressbar" aria-valuenow={currentIndex} aria-valuemin={0} aria-valuemax={total} aria-label="Lesson progress"`

Source: CONTEXT.md D-10

### 3. Crossfade Question Wrapper

**Contract:**

- Container: `relative w-full` holding the currently active question renderer
- Transition: CSS `opacity` from 0 ŌåÆ 1 over 300ms (`animate-fadeIn` from tailwind.config.js keyframes) on question mount
- Outgoing question: fades out over 150ms, then incoming question fades in over 300ms (two-phase: out then in)
- Reduced-motion: skip all opacity transitions, instant DOM swap. Color feedback (card states) still applies.
- Implementation: controlled by `questionKey` state in engine ŌĆö when key changes, React remounts the renderer, triggering the enter animation

Source: CONTEXT.md D-11, D-14, tailwind.config.js `fadeIn` keyframe

## Interaction Contract

### Feedback Timing (from CONTEXT.md D-13 ŌĆö locked)

| Outcome | Card state change                                          | Sound                | Delay before advance |
| ------- | ---------------------------------------------------------- | -------------------- | -------------------- |
| Correct | Selected ŌåÆ `correct`, others ŌåÆ `dimmed`                    | `playCorrectSound()` | 800ms                |
| Wrong   | Selected ŌåÆ `wrong`, correct ŌåÆ `correct`, others ŌåÆ `dimmed` | `playWrongSound()`   | 1200ms               |

After delay: crossfade transition fires, then next question appears.

### Question Advance Sequence

1. Child taps a card ŌåÆ engine enters FEEDBACK state ŌåÆ `disabled=true` passed to renderer
2. Cards update to correct/wrong/dimmed states
3. Feedback delay timer runs (800ms or 1200ms)
4. Crossfade out: current renderer opacity 0 ŌåÆ 150ms
5. Engine increments `currentIndex`, updates `questionKey` ŌåÆ React remounts renderer
6. Crossfade in: new renderer opacity 0 ŌåÆ 1 ŌåÆ 300ms
7. `disabled=false`, child can tap again

Reduced-motion path: skip steps 4 and 6. Instant swap at step 5.

### Progress Bar Advance

Progress bar fill width = `(currentIndex / totalQuestions) * 100%`. Updates immediately when `currentIndex` increments (not after crossfade completes). Fraction text updates simultaneously.

### Landscape Detection

Same media query pattern as standalone games:

```javascript
const mql = window.matchMedia("(orientation: landscape)");
```

Updates `isLandscape` state. Passed as prop to both renderers.

## Accessibility Contract

| Requirement            | Implementation                                                                                                       |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------- |
| Screen reader progress | `<div aria-live="polite" className="sr-only">` containing feedbackMessage string                                     |
| Progress bar ARIA      | `role="progressbar" aria-valuenow aria-valuemin aria-valuemax aria-label`                                            |

| Touch targets          | DurationCard min-height 44px (`min-h-touch`) ŌĆö enforced by tailwind.config.js                                        |
| Disabled state         | `disabled=true` prop on renderer ŌåÆ `pointer-events-none` on cards during feedback phase                              |
