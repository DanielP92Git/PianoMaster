## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage                                                 |
| ----- | ----- | ----------------------------------------------------- |
| xs    | 4px   | Icon gaps, progress dot spacing                       |
| sm    | 8px   | Card inner padding, inline gap between elements       |
| md    | 16px  | Default element spacing, progress bar padding         |
| lg    | 24px  | Section padding, card content area                    |
| xl    | 32px  | Layout gaps between question area and cards           |
| 2xl   | 48px  | Major section breaks (portrait top offset)            |
| 3xl   | 64px  | Page-level spacing (not typically used in game views) |

Exceptions:

- Touch targets for DurationCard: minimum 44px height (`min-h-touch` from tailwind.config.js)
- Progress bar height: 8px (`h-2`) — below scale by design for compact header region

Source: tailwind.config.js (minHeight.touch = 44px), CLAUDE.md glass card pattern

## Color

| Role            | Value                                                            | Usage                                                       |
| --------------- | ---------------------------------------------------------------- | ----------------------------------------------------------- |
| Dominant (60%)  | `bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900` | Full-viewport game background — same as all game routes     |
| Secondary (30%) | `bg-white/10 backdrop-blur-md border border-white/20`            | Question prompt panel, card grid container, top bar backing |
| Accent (10%)    | `bg-green-400` / `bg-green-500/20 border-green-400`              | Progress bar fill, correct-answer card highlight            |
| Destructive     | `bg-red-500/20 border-red-400`                                   | Wrong-answer card highlight only                            |

Accent reserved for:

1. Progress bar fill (`bg-green-400`) — the advancing fill of the lesson progress bar
2. Correct card state (`bg-green-500/20 border-2 border-green-400 shadow-[0_0_12px_rgba(74,222,128,0.4)]`) — feedback on correct tap
3. Correct-answer reveal on wrong tap — the same green card state applied to the correct card after a wrong answer

Destructive reserved for:

1. Wrong card state (`bg-red-500/20 border-2 border-red-400`) — feedback on wrong tap only

Additional semantic colors (carried from Phase 24 DurationCard — do not change):

- Default card: `bg-white/10 border border-white/20` with `hover:bg-white/20 hover:border-white/40`
- Dimmed card: `opacity-40 pointer-events-none bg-white/10 border border-white/20`

Progress bar track: `bg-white/15` — established in CONTEXT.md D-10

### 2. Progress Bar

**Contract:**

- Track: `h-2 rounded-full bg-white/15 overflow-hidden` — same glass track as existing progress bars (CLAUDE.md)
- Fill: `h-2 rounded-full bg-green-400` — width set as inline style: `width: {(currentIndex / total) * 100}%`
- Width transition: `transition-[width] duration-300 ease-in-out` — skipped when `reducedMotion` is true
- Fraction label: `text-sm font-semibold text-white/70` at right end of top bar — `"3/10"` format (never "3 of 10")
- ARIA: `role="progressbar" aria-valuenow={currentIndex} aria-valuemin={0} aria-valuemax={total} aria-label="Lesson progress"`

Source: CONTEXT.md D-10

### 3. Crossfade Question Wrapper

**Contract:**

- Container: `relative w-full` holding the currently active question renderer
- Transition: CSS `opacity` from 0 → 1 over 300ms (`animate-fadeIn` from tailwind.config.js keyframes) on question mount
- Outgoing question: fades out over 150ms, then incoming question fades in over 300ms (two-phase: out then in)
- Reduced-motion: skip all opacity transitions, instant DOM swap. Color feedback (card states) still applies.
- Implementation: controlled by `questionKey` state in engine — when key changes, React remounts the renderer, triggering the enter animation

Source: CONTEXT.md D-11, D-14, tailwind.config.js `fadeIn` keyframe

## Interaction Contract

### Feedback Timing (from CONTEXT.md D-13 — locked)

| Outcome | Card state change                                          | Sound                | Delay before advance |
| ------- | ---------------------------------------------------------- | -------------------- | -------------------- |
| Correct | Selected → `correct`, others → `dimmed`                    | `playCorrectSound()` | 800ms                |
| Wrong   | Selected → `wrong`, correct → `correct`, others → `dimmed` | `playWrongSound()`   | 1200ms               |

After delay: crossfade transition fires, then next question appears.

### Question Advance Sequence

1. Child taps a card → engine enters FEEDBACK state → `disabled=true` passed to renderer
2. Cards update to correct/wrong/dimmed states
3. Feedback delay timer runs (800ms or 1200ms)
4. Crossfade out: current renderer opacity 0 → 150ms
5. Engine increments `currentIndex`, updates `questionKey` → React remounts renderer
6. Crossfade in: new renderer opacity 0 → 1 → 300ms
7. `disabled=false`, child can tap again

Reduced-motion path: skip steps 4 and 6. Instant swap at step 5.

### Progress Bar Advance

Progress bar fill width = `(currentIndex / totalQuestions) * 100%`. Updates immediately when `currentIndex` increments (not after crossfade completes). Fraction text updates simultaneously.

### Landscape Detection

Same media query pattern as standalone games:

```javascript
const mql = window.matchMedia("(orientation: landscape)");
```

Updates `isLandscape` state. Passed as prop to both renderers.

## Accessibility Contract

| Requirement            | Implementation                                                                                                       |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------- |
| Screen reader progress | `<div aria-live="polite" className="sr-only">` containing feedbackMessage string                                     |
| Progress bar ARIA      | `role="progressbar" aria-valuenow aria-valuemin aria-valuemax aria-label`                                            |
| Card ARIA              | Each DurationCard receives `ariaLabel` prop (already implemented in DurationCard.jsx)                                |
| Reduced motion         | `useMotionTokens().reduce` — skip crossfade opacity transitions, instant DOM swap                                    |
| RTL support            | Hebrew: `dir="ltr"` wrapper on SVG icon elements (music notation is always LTR); text UI inherits document direction |
| Touch targets          | DurationCard min-height 44px (`min-h-touch`) — enforced by tailwind.config.js                                        |
| Disabled state         | `disabled=true` prop on renderer → `pointer-events-none` on cards during feedback phase                              |

Source: CLAUDE.md accessibility section, DurationCard.jsx, SyllableMatchingGame.jsx `dir="ltr"` pattern (verified)

## Animation Contract

| Animation              | Class / Implementation                                                       | Reduced-Motion         |
| ---------------------- | ---------------------------------------------------------------------------- | ---------------------- |
| Page enter             | `animate-fadeIn` (tailwind.config.js keyframe: opacity 0→1, 300ms)           | Omit class entirely    |
| Question crossfade out | inline `opacity-0 transition-opacity duration-150`                           | Skip — instant         |
| Question crossfade in  | `animate-fadeIn` on remounted renderer (opacity 0→1, 300ms)                  | Skip — instant         |
| Progress bar width     | `transition-[width] duration-300 ease-in-out` inline on fill div             | Skip — omit transition |
| Card correct flash     | `shadow-[0_0_12px_rgba(74,222,128,0.4)]` on correct card (from DurationCard) | Color still shows      |
| Card state transition  | `transition-all duration-300` on correct/wrong states (from DurationCard)    | Instant (no-op)        |
