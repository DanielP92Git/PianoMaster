## Spacing Scale

Declared values (multiples of 4px, aligned with project `tailwind.config.js` CSS custom property tokens):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline icon-text padding |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`p-4`, `gap-4`) |
| lg | 24px | Section padding (`p-6`), form field vertical rhythm |
| xl | 32px | Layout gaps between form sections (`pt-8`) |
| 2xl | 48px | Separation below Logout before feedback trigger (`pt-12`) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets (trigger button, submit button): minimum 44px height — maps to `minHeight.touch` token in `tailwind.config.js`
- Textarea minimum height: 120px (3 visible lines at 16px line-height 1.5)

**Source:** `tailwind.config.js` spacing tokens, `AppSettings.jsx` existing Logout section uses `pt-12 pb-8`

## Color

This phase lives entirely inside `AppSettings.jsx`, which renders on the app's global purple gradient background (`bg-gradient-to-br from-indigo-900 via-purple-900 to-violet-900`). All components follow the glassmorphism pattern, not the white-card pattern from `docs/DESIGN_SYSTEM.md` (the white-card pattern is documented as a migration target but AppSettings itself remains on glass).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgba(255,255,255,0.10)` — `bg-white/10` | Form container glass background, trigger button resting state |
| Secondary (30%) | `rgba(255,255,255,0.05)` — `bg-white/5` | Textarea background, select background |
| Accent (10%) | `#4f46e5` (indigo-600) — `bg-indigo-600` | Submit button only |
| Success semantic | `rgba(16,185,129,0.30)` — `text-green-400` / `bg-green-500/20` | Success state icon and container |
| Destructive / error | `rgba(239,68,68,0.10)` — `bg-red-500/10 border-red-500/30 text-red-200` | Inline error banner |

**Text colors** (from CLAUDE.md glass card pattern):
- Primary: `text-white`
- Secondary: `text-white/70`
- Tertiary / meta: `text-white/60`
- Disabled / cooldown: `text-white/40`
- Error: `text-red-200`
- Success: `text-green-400`

Accent (`bg-indigo-600`) reserved for: **submit button only**. Trigger button uses glass (`bg-white/10`) not accent.

**Source:** CLAUDE.md glass card pattern, `SettingsSection.jsx`, RESEARCH.md code examples

## Component Inventory

All components are built from Tailwind utilities. No shadcn or Radix primitives are used.

### FeedbackForm.jsx (new)

Self-contained four-state component in `src/components/settings/`:

**State: `idle`**
- Centered trigger button: `inline-flex items-center gap-2 px-5 py-2.5 rounded-xl bg-white/10 border border-white/20 text-white/80 hover:bg-white/20 hover:text-white transition-all duration-200 text-sm font-medium`
- `MessageSquare` icon (w-4 h-4) + label text
- Wrapped in `div.pt-6.pb-8.flex.justify-center`
- Placed after the Logout `border-t border-white/20` div

**State: `gated`**
- `ParentGateMath` component renders (reused unchanged)
- Accepts `onConsent`, `onCancel`, `isRTL` props
- ParentGateMath renders its own fixed overlay at z-50

**State: `form`**
- Container: `bg-white/10 backdrop-blur-md border border-white/20 rounded-xl shadow-lg p-6 space-y-4`
- Header: `MessageSquare` icon (w-5 h-5 text-indigo-300) + heading `text-lg font-bold text-white` + subtitle `text-sm text-white/70`
- Error banner (conditional, see below): renders above form fields
- Feedback type select: `w-full bg-white/5 border border-white/20 rounded-lg px-3 py-2.5 text-white text-sm focus:outline-none focus:ring-2 focus:ring-indigo-400/50 focus:border-indigo-400/50 transition-colors`
- Label: `block text-sm font-medium text-white/80 mb-1.5`
- Textarea: same base classes as select, min-height 120px, resize-none, max 1000 chars
- Character counter: `text-xs text-white/40` at rest; `text-amber-300` when trimmed length >= 900; `text-red-300` when trimmed length >= 980
- Submit button: `w-full px-4 py-3 bg-indigo-600 hover:bg-indigo-700 disabled:opacity-50 disabled:cursor-not-allowed text-white font-semibold rounded-lg transition-colors flex items-center justify-center gap-2 min-h-[44px]`
- Submit loading: `Loader2` (w-4 h-4 animate-spin) + "Sending..." text
- Honeypot: absolute-positioned, 1px × 1px, opacity 0, pointer-events none, aria-hidden true, tabIndex -1

**State: `success`**
- Container: `text-center space-y-3 py-6`
- `CheckCircle` icon: `w-12 h-12 text-green-400 mx-auto`
- Title: `text-white font-semibold` (18px)
- Subtitle: `text-white/60 text-sm`
- Cooldown message: `text-white/40 text-xs` with countdown formatted as `M:SS`

**Error banner (within `form` state)**
- Container: `rounded-xl border border-red-500/30 bg-red-500/10 px-4 py-3 flex items-start gap-3`
- `AlertTriangle` icon: `w-4 h-4 text-red-300 flex-shrink-0 mt-0.5`
- Text: `flex-1 text-sm text-red-200`
- Dismiss button: `X` icon `w-4 h-4 text-red-400 hover:text-red-200`
- Retry button (network error only): `ml-2 underline hover:no-underline text-red-300` inline after error text

### Animations

- Form reveal after gate: `animate-fadeIn` (0.3s ease-in-out, defined in `tailwind.config.js`)
- Submit button disabled state transitions immediately (no animation)
- Reduced motion: Remove `animate-fadeIn` when `prefers-reduced-motion: reduce` — use `transition-none` fallback

| Element | LTR | RTL |
|---------|-----|-----|

## Form Validation Display

Client-side validation messages (shown as inline text below the field, not as banners):

| Field | Rule | Display |
|-------|------|---------|
| Message textarea | min 10 trimmed characters | Disable submit button; counter turns red-300 near limit |
| Message textarea | max 1000 trimmed characters | Counter turns amber-300 at 900+, red-300 at 980+ |
| Feedback type | required | Submit disabled if no selection (default to 'bug' pre-selected) |

Pre-select "Bug Report" as default on form open. This ensures the type field is never empty on submit.

**Source:** CONTEXT.md D-07, D-08; RESEARCH.md Pitfall 4

## State Machine Visual Map

```
[idle]
  trigger button centered below Logout section
  → click → [gated]

[gated]
  ParentGateMath overlay (z-50, fixed)
  → onConsent → [form] (with animate-fadeIn)
  → onCancel → [idle]

[form]
  glass card with type select + textarea + submit
  error banner (orthogonal — appears within form state)
  → success 200 → [success]
  → 429/5xx/network → stays [form] + shows error banner

[success]
  CheckCircle + thank you message + countdown
  → countdown reaches 0 → [idle]
```

## Anti-Patterns Blocked by This Contract

| Anti-Pattern | Why Blocked |
|--------------|-------------|
| Toast notifications for form errors | D-13: inline banners only |
| Raw error codes visible to user | D-15: translate to friendly copy |
| localStorage for cooldown | D-16: React state only, per-session |
| `display: none` for honeypot | RESEARCH.md Pitfall 2: use absolute-position + opacity 0 |
| `SettingsSection` wrapper for the form | D-04/D-06: standalone centered layout |
| Checking `parentConsentGranted` from push subscription | D-01: gate always fires fresh |
| `bg-white` card style | AppSettings uses glassmorphism; do not use white-card pattern |
